  <action>
Create `.github/workflows/deploy.yml` in the racecontrol repo root (NOT in pwa/).

Workflow specification:
- **name:** "Deploy to VPS"
- **trigger:** `on: push: branches: [main]`
- **concurrency:** `group: deploy, cancel-in-progress: true` (prevent overlapping deploys)
- **Single job:** `deploy` running on `ubuntu-latest`
- **Steps:**
  1. `actions/checkout@v4` — needed so the workflow file itself is valid
  2. `appleboy/ssh-action@v1.2.0` — SSH into VPS and run deploy commands
     - `host: ${{ secrets.VPS_HOST }}`
     - `username: root`
     - `key: ${{ secrets.VPS_SSH_KEY }}`
     - `script_stop: true` — CRITICAL: this makes the action fail if ANY command in the script returns non-zero, preventing deploy on failed build
     - `script:` (multi-line):
       ```
       cd /opt/racingpoint
       git -C racecontrol pull origin main
       git -C racingpoint-admin pull origin main
       cp racecontrol/cloud/compose.yml .
       cp racecontrol/cloud/Caddyfile .
       docker compose build
       docker compose up -d
       docker compose ps
       ```
     - Note: `docker compose build` and `docker compose up -d` are separate steps (not `--build`) so a build failure stops before `up`. The `script_stop: true` flag ensures this.

Do NOT add any test/lint steps — this is a pure deploy pipeline. Tests are not set up for this project.
Do NOT use Docker registry — images build on VPS directly.
  </action>
  <verify>
    <automated>cat .github/workflows/deploy.yml | head -50 && grep -c "script_stop: true" .github/workflows/deploy.yml && grep -c "secrets.VPS_SSH_KEY" .github/workflows/deploy.yml && grep -c "secrets.VPS_HOST" .github/workflows/deploy.yml</automated>
  </verify>
