   ```yaml
   # REMOVE:

     NEXT_PUBLIC_API_URL: https://api.racingpoint.cloud
   # REPLACE WITH:
   args:
     NEXT_PUBLIC_RC_URL: https://api.racingpoint.cloud
     NEXT_PUBLIC_GATEWAY_URL: https://api.racingpoint.cloud
     NEXT_PUBLIC_GATEWAY_API_KEY: rp-gateway-2026-secure-key
   ```
   The admin Dockerfile accepts NEXT_PUBLIC_RC_URL (not NEXT_PUBLIC_API_URL). NEXT_PUBLIC_GATEWAY_URL is used client-side for analytics/bookings/health fetches — on cloud it should point to the cloud API. The gateway API key uses the same default as the Dockerfile.

2. **Add server-side environment vars** — Add an `environment` section to the admin service:
   ```yaml
   environment:
     - RC_URL=http://host.docker.internal:8080
     - RACECONTROL_URL=http://host.docker.internal:8080
     - GATEWAY_URL=http://host.docker.internal:8080
     - PORT=3300
   ```
   RC_URL is used by `/api/rc/[...path]/route.ts` (the main API proxy). RACECONTROL_URL is used by `/api/waivers/route.ts`. GATEWAY_URL is used server-side by analytics/calendar/health routes. PORT=3300 overrides the Dockerfile's PORT=3000 so the app listens on 3300 (matching Caddy and expose config). Using host.docker.internal because racecontrol runs on the VPS host, not in Docker.

3. **Add admin to Caddy depends_on** — In the caddy service, add admin to depends_on alongside pwa:
   ```yaml
   depends_on:
     pwa:
       condition: service_healthy
     admin:
       condition: service_healthy
   ```
