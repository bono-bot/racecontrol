Create `cloud/health-check.sh` — a self-contained bash script that runs on the VPS via cron every 2 minutes.

**Cooldown mechanism:**
- Use a state directory `/tmp/rc-health-alerts/` to track last alert time per failure key
- Before sending any alert, check if the same failure key was alerted within the last 30 minutes (1800 seconds)
- Failure key format: `pm2_{process_name}`, `pm2_crashloop_{process_name}`, `disk`, `memory`

**PM2 process monitoring:**
- Run `pm2 jlist` and parse JSON output with `jq`
- Check each process: if status is "errored" or "stopped" → alert with process name and status
- Track restart counts: store previous restart count in `/tmp/rc-health-state/` per process. If delta >3 within the check window (compare timestamps) → crash loop alert
- For crash loop detection: store `{process_name}.restarts` files with format `{timestamp} {restart_count}`. On each run, compare current restart count to the stored value. If current - stored > 3 AND current_time - stored_time < 600 (10 min) → crash loop. Update stored value after check.

**System resource monitoring:**
- Disk: `df -h / | awk 'NR==2{print $5}' | tr -d '%'` — alert if >90
- Memory: `free | awk '/Mem:/{printf "%.0f", $3/$2 * 100}'` — alert if >90

**Alert delivery — WhatsApp via Evolution API:**
- Instance name: discover from Evolution API or use default "racingpoint"
- Endpoint: `POST http://localhost:53622/message/sendText/{instance_name}`
- Headers: `Content-Type: application/json`, `apikey: {from env or config}`
- Body: `{"number": "UDAY_PHONE", "text": "ALERT_MESSAGE"}`
- Uday's phone number: read from env var `UDAY_WHATSAPP` (set in cron env or script config section at top)
- Evolution API key: read from env var `EVOLUTION_API_KEY`
- Alert message format: `[Racing Point VPS Alert]\n{issue}\nTime: {IST timestamp}`
- NOTE: The Evolution API key and Uday's WhatsApp number must be configured as variables at the top of the script. Use placeholder values that Bono/Uday will fill in: `EVOLUTION_API_KEY="CHANGE_ME"` and `UDAY_WHATSAPP="91XXXXXXXXXX"`

**Alert delivery — comms-link to James:**
- After WhatsApp alert, also send to comms-link WebSocket
- Use: `cd /root/racingpoint/comms-link && COMMS_PSK="85d1d06c806b3cc5159676bbed35e29ef0a60661e442a683c2c5a345f2036df0" COMMS_URL="ws://localhost:8765" node send-message.js "[VPS Health] {issue}"`

Create `cloud/setup-swap.sh` — a one-time setup script to add 2GB swap on the VPS.

Script contents:
```bash
#!/usr/bin/env bash
set -euo pipefail

# Racing Point VPS — 2GB Swap Setup
# Run once: bash /root/racingpoint/racecontrol/cloud/setup-swap.sh

if swapon --show | grep -q '/swapfile'; then
  echo "Swap already configured:"
  swapon --show
  exit 0
fi

echo "Creating 2GB swap..."
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# Make persistent across reboots
if ! grep -q '/swapfile' /etc/fstab; then
  echo '/swapfile none swap sw 0 0' >> /etc/fstab
fi

echo "Swap configured:"
swapon --show
free -h
```
