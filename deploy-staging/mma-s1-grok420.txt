**1. Why is the watchdog not restarting rc-agent?**

The watchdog is not triggering because **rc-sentry.exe is running in the wrong context for the watchdog logic you wrote**.

Even though rc-sentry.exe is technically in Session 1 (Console), it was launched via HKLM Run key **after** explorer.exe and the user shell had already started. Your `session1_spawn.rs` code (the part that does `WTSQueryUserToken` + `CreateProcessAsUser`) was **never executed** on boot.

Most implementations of this pattern have the watchdog/restart logic **only active when the process was spawned via the session1_spawn path** (i.e. there is usually a flag, environment variable, or different code path). Because rc-sentry was started â€œnormallyâ€? via the Run key, the watchdog is either:
- In a â€œpassive/monitoring-onlyâ€? mode, or
- The health check code path that leads to `spawn_in_session1()` is skipped.

The cmd.exe remnant in Session 0 is the classic symptom of the HKLM Run key race condition when both .bat files run at the same time.

- rc-sentry.exe then keeps rc-agent.exe alive.

**rc-sentry.exe changes needed:**
- When launched with `--service` (or when running as SYSTEM), **immediately** call `spawn_in_session1("rc-sentry.exe", "--watchdog")` so that a copy of rc-sentry runs in Session 1 with the watchdog active.
- The Session 1 instance (the one with `--watchdog`) does the 5-second health polling + `spawn_in_session1()` for rc-agent.exe.
- The service instance should only monitor that the Session 1 rc-sentry is alive and restart it if it dies.

**HKLM Run key cleanup:**
- Remove `start-rcagent.bat` completely from HKLM\Software\Microsoft\Windows\CurrentVersion\Run.
- Remove `start-rcsentry.bat` from HKLM Run (or make it only run on the physical console for fallback).

**3. Auto-login configuration (correct way on Windows 11)**

```powershell
# Enable auto-login properly (passwordless)
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon /t REG_SZ /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUserName /t REG_SZ /d "user" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultDomainName /t REG_SZ /d "SIM1" /f
# Leave DefaultPassword empty or set to a dummy value

# Disable the welcome screen / first-logon animation that breaks Session 1 timing

# Make sure explorer starts cleanly
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v Shell /t REG_SZ /d "explorer.exe" /f
```
