      - Write to {evo_install_dir}/Saved/Config/WindowsNoEditor/GameUserSettings.ini
      - Unreal INI format: [/Script/Engine.GameUserSettings] section with key=value pairs
      - If launch_args empty or no parseable fields: skip config write, return Ok (default game config used)

      - Return first valid path
    - check_iracing_ready() -> Result<(), String>
      - Check if iRacing is installed (iRacingSim64DX11.exe or iRacingService.exe present)
      - Launch iRacing silently via iRacingService, wait up to 30s
      - Poll window titles via sysinfo for "iRacing" window. If window title contains "Login" or "Subscription" or "Error" within 30s: return Err("iRacing subscription check failed — account may be inactive")
      - If game window appears normally: return Ok(()) (game is playable)
      - Fallback: if window title detection is unreliable, check if iRacingSim64DX11.exe PID exists after 30s as basic launch success
  </behavior>

    **AC EVO config adapter (GAME-04):**

    Add `write_evo_config()` and `find_evo_install_dir()` as public functions in `assetto_corsa_evo.rs`.

    The Unreal GameUserSettings.ini format for AC EVO:
    ```ini
    [/Script/AssettoCorsaEVO.ACEVOGameUserSettings]
    SelectedCar=car_name
    SelectedTrack=track_name
    WeatherPreset=weather_name
    TimeOfDay=time_value
    ```

    Parse launch_args as JSON `{"car": "...", "track": "...", "weather": "...", "time_of_day": "..."}`. Write INI using simple string formatting (no INI library needed — just key=value under a section header). Read existing file first and merge/replace only the keys we set, preserving user's graphics/audio settings.

    If the Saved/Config/WindowsNoEditor directory doesn't exist, create it. If GameUserSettings.ini doesn't exist, create with only our section.

    **iRacing subscription check (GAME-05):**

    Create `crates/rc-agent/src/iracing_checks.rs` with `check_iracing_ready()`.

    Strategy (per CONTEXT.md): Don't try web scraping or API. Instead:
    1. Check iRacingService.exe or iRacingService64.exe is running (it runs as a service when iRacing is installed)
    2. If not running, that's OK — the launch will start it
    3. After iRacing launches (in the ws_handler, post-launch), poll for 30s looking at window titles
    4. If within 30s the main iRacing window appears with "iRacing Simulator" title: subscription OK
    5. If window shows login/error dialog (window title heuristic): subscription problem

    For pre-launch: check_iracing_ready() verifies iRacing files exist on disk and the service can be started. The subscription verification happens post-launch (similar to GAME-07 wait_for_game_window pattern).

    Register `pub mod iracing_checks;` in lib.rs.

    - write_evo_config with empty args returns Ok

    - check_iracing_ready basic structure (mocked)
