    - check_steam_ready(sim_type: SimType, config: &GameExeConfig) -> Result<(), String>
      - If config.use_steam or config.steam_app_id.is_some(): verify steam.exe running via sysinfo
      - If Steam not running: attempt start from "C:\Program Files (x86)\Steam\steam.exe" with -silent flag, wait 10s, recheck
      - If still not running after retry: return Err("Steam is not running...")
      - Check for Steam update processes (steamwebhelper updating, SteamOverlayUpdate) — if found, return Err("Steam has pending updates...")
      - If sim_type is AC: skip (Game Doctor handles AC Steam checks already)
      - Non-Steam games (exe_path only, no steam_app_id): return Ok(()) immediately
    - wait_for_game_window(sim_type: SimType, timeout_secs: u64) -> Result<u32, String>
      - Poll every 2s for the sim_type's expected exe name via sysinfo process scan
      - If game exe found within timeout: return Ok(pid)
      - If only Steam windows after timeout_secs (default 60): return Err("Game failed to launch - only Steam dialog visible")
      - Uses all_game_process_names() mapping to find the right exe for each SimType
    - check_dlc_installed(sim_type: SimType, launch_args: &str, config: &GameExeConfig) -> Result<(), String>
      - Parse launch_args for car/track content identifiers
      - For Steam games: check appmanifest_{app_id}.acf for InstalledDepots section
      - For AC: check content/cars/{car_name} and content/tracks/{track_name} directories exist
      - If content not found: return Err("Content not installed: {details}")
    - GAME-02: Update all_game_process_names() to include verified exe names:
      - Add "F1_2025.exe" alongside "F1_25.exe" (both possible names)
