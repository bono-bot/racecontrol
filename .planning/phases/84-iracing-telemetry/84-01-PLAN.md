  <action>
    Create `crates/rc-agent/src/sims/iracing.rs` with the following structure:

    **Structs:**
    - `IracingAdapter` with fields: pod_id, connected, shm_handle (Option<ShmHandle> cfg(windows)), var_offsets (VarOffsets), last_session_uid (i32), last_session_info_update (i32), last_lap_count (i32), sector_times ([Option<u32>; 3]), pending_lap (Option<LapData>), current_track (String), current_car (String), current_session_type (SessionType), first_read (bool)
    - `VarOffsets` struct with i32 offsets for: is_on_track, lap_completed, lap_last_lap_time, session_unique_id, speed, throttle, brake, gear, rpm, lap_dist_pct, session_state. All default to -1 (not found).
    - `IrsdkHeader` struct matching the 112-byte layout from RESEARCH.md (status, session_info_update, session_info_len, session_info_offset, num_vars, var_header_offset, num_buf, buf_len)
    - Reuse AC's `ShmHandle` pattern (define locally in this file, same struct with _handle, ptr, _size)

    **Shared memory (all #[cfg(windows)]):**
    - `open_iracing_shm()` -> Result<ShmHandle>: Opens `Local\IRSDKMemMapFileName` via OpenFileMappingW + MapViewOfFile (FILE_MAP_READ). Bail if handle is null (iRacing not running).
    - `read_header(ptr: *const u8) -> IrsdkHeader`: Read header fields from fixed offsets (per RESEARCH.md layout)
    - `is_iracing_active(status: i32) -> bool`: `status & 1 != 0`
    - `find_var_offset(shm_ptr, header, name)` -> Option<(i32, i32)>: Scan irsdk_varHeader array (144 bytes each, name at offset 16, 32 chars). Return (offset_in_row, type).
    - `build_var_offsets(shm_ptr, header)` -> VarOffsets: Call find_var_offset for each needed variable name (b"IsOnTrack", b"LapCompleted", b"LapLastLapTime", b"SessionUniqueID", b"Speed", b"Throttle", b"Brake", b"Gear", b"RPM", b"LapDistPct", b"SessionState")
    - `read_latest_row_offset(shm_ptr, header)` -> Option<i32>: Double-buffer tick-lock — find highest tickCount in varBuf[0..numBuf], read row, verify tick unchanged (retry up to 3 times)
    - `read_var_i32(ptr, row_offset, var_offset)` -> i32, `read_var_f32(...)` -> f32, `read_var_bool(...)` -> bool: Unsafe reads at ptr + row_offset + var_offset

    **YAML session parsing:**
    - `extract_yaml_value(yaml: &str, key: &str) -> Option<String>`: Find "key:" in string, extract the rest of line (trim whitespace + quotes). This is NOT serde_yaml — manual scan only.
    - `parse_session_type(yaml_val: &str) -> SessionType`: "Race"/"Sprint Race" -> Race, "Qualify"/"Lone Qualify"/"Open Qualify" -> Qualifying, "Time Trial" -> Hotlap, _ -> Practice
    - `parse_session_yaml(&mut self, shm_ptr, header)`: Read YAML string from shm at session_info_offset, extract TrackDisplayName, CarScreenName, SessionType

    **SimAdapter impl:**
    - `connect()`: Open shm, read header, verify is_iracing_active, build var_offsets, parse session YAML, set first_read=true, set last_lap_count from current LapCompleted (first-packet safety). On non-Windows: bail!("iRacing shared memory requires Windows")
    - `is_connected()`: self.connected
    - `read_telemetry()`: Read header, check connected bit, read latest buffer row, extract speed/throttle/brake/gear/rpm/steer, detect session UID change (reset state + re-parse YAML), detect lap (LapCompleted > last_lap_count AND LapLastLapTime > 0 AND !first_read), construct TelemetryFrame. On first read: set last_lap_count without emitting lap, set first_read=false.
    - `poll_lap_completed()`: self.pending_lap.take() (same take semantics as F1 25)
    - `session_info()`: Return SessionInfo with current_track, current_car, current_session_type if connected
    - `disconnect()`: UnmapViewOfFile + CloseHandle (cfg(windows)), set connected=false
    - Non-Windows stubs: connect returns Err, read_telemetry returns Ok(None), etc.

    **IsOnTrack method (for Plan 02 integration):**
    - `pub fn read_is_on_track(&self) -> Option<bool>`: Read IsOnTrack variable from current buffer row. Returns None if not connected.

    **Pre-flight check:**
    - `pub fn check_iracing_shm_enabled() -> bool`: Use dirs_next::document_dir() to find Documents/iRacing/app.ini. Read file, scan lines for "irsdkEnableMem=1" (trimmed, case-sensitive). tracing::warn if not found. Return bool.
    - For testability: `pub fn check_iracing_shm_enabled_at(path: &std::path::Path) -> bool` — same logic but accepts explicit path (tests use this with tempfile).

    **Unit tests (in #[cfg(test)] mod tests):**
    All tests are pure Rust — no iRacing process needed. Tests exercise parsing logic, state transitions, and pre-flight.

    1. `test_connect_no_shm`: Create adapter, call connect(). On non-Windows: assert Err. On Windows without iRacing: assert Err.
    2. `test_session_transition_resets_lap`: Create adapter, manually set last_lap_count=5, sector_times=[Some(10000), Some(11000), None], last_session_uid=42. Call internal reset method with new UID=43. Assert last_lap_count=0, sector_times=[None;3], last_session_uid=43.
    3. `test_lap_completed_event`: Create adapter, set connected=true, last_lap_count=1, first_read=false, current_track/car/session_type. Call internal lap detection logic with lap_completed=2, last_lap_time=62.5f32. Assert pending_lap is Some with lap_time_ms=62500, sim_type=IRacing, lap_number=2.
    4. `test_first_packet_safety`: Create adapter with first_read=true. Feed lap_completed=3, last_lap_time=55.0. Assert NO pending_lap (just stores last_lap_count=3).
    5. `test_preflight_missing_ini`: Call check_iracing_shm_enabled_at with nonexistent path. Assert returns false.
    6. `test_preflight_ini_enabled`: Write tempfile with "[Miscellaneous]\nirsdkEnableMem=1\n". Call check_iracing_shm_enabled_at. Assert returns true.
    7. `test_session_type_mapping`: Assert parse_session_type for all YAML values: Race, Qualify, Lone Qualify, Time Trial, Practice, Warmup, unknown.

    **Key constraints:**
    - No .unwrap() in production code (standing rule). Use anyhow::bail! or return Err.
    - LapLastLapTime is in SECONDS (f32), multiply by 1000 for lap_time_ms (u32). Do NOT confuse units.
    - Use `LapCompleted` (finished laps), NOT `Lap` (started laps). RESEARCH.md is explicit about this.
    - Do NOT use serde_yaml. iRacing YAML is ISO-8859-1 non-standard. Manual key scan only.
    - Variable lookup is by name (dynamic), NOT by fixed offset. Offsets change between SDK versions.
    - Sector splits: set all three to None for v1 (no sector variables in iRacing real-time telemetry).
    - `IracingAdapter::new(pod_id: String)` constructor — simpler than AC (no sim_ip/sim_port needed).
