1. Import `fetchApi` from `@/lib/api` (reuse existing pattern with auth + timeout)
2. Define TypeScript interfaces matching Rust structs EXACTLY:
   - `TimePoint` — `ts: number` + `value: number`
   - `QueryResponse` — `metric: string` + `pod: number | null` + `resolution: string` + `points: TimePoint[]`
   - `SnapshotEntry` — `name: string` (NOT metric_name) + `pod: number | null` (NOT pod_id string) + `value: number` + `updated_at: number` (unix epoch i64, NOT ISO string)
   - `SnapshotResponse` — `metrics: SnapshotEntry[]`
   - `NamesResponse` — `names: string[]` (wrapped object, NOT bare array)
3. Export three functions:
   - `fetchMetricNames(): Promise<string[]>` — GET `/metrics/names`, unwrap `resp.names`
   - `fetchMetricSnapshot(pod?: number): Promise<SnapshotEntry[]>` — GET `/metrics/snapshot?pod=N`, unwrap `resp.metrics`
   - `fetchMetricQuery(metric: string, from: number, to: number, pod?: number): Promise<QueryResponse>` — GET `/metrics/query?metric=X&from=Y&to=Z&pod=N`

**Create `web/src/app/metrics/page.tsx`** — Metrics dashboard:

1. `"use client"` directive
2. Wrap in `DashboardLayout` (same as fleet page)

9. Sparkline charts: one `recharts` `<AreaChart>` per metric name, data from query response `points` array, X-axis = `ts` formatted as time, Y-axis = `value`
10. Use Tailwind classes matching existing dashboard style (dark theme: bg-zinc-900, cards bg-zinc-800, text-white)
