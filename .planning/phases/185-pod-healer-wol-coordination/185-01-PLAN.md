<objective>
Wire ProcessOwnership registry, recovery-intent.json with 2-min TTL, and GRACEFUL_RELAUNCH sentinel checking into racecontrol's pod_healer and recovery subsystems -- so no two recovery authorities can claim the same process simultaneously, and intentional restarts are distinguishable from crashes at the server level.

Purpose: Prevents the infinite restart loops documented in standing rules where rc-sentry, self_monitor, pod_healer, and rc-watchdog fight each other over the same process.
Output: ProcessOwnership in AppState, RecoveryIntentStore with TTL, pod_healer checks ownership+intent+sentinel before acting.

   pub struct RecoveryIntentStore {
       intents: Vec<RecoveryIntent>,
   }
   impl RecoveryIntentStore {
       pub fn new() -> Self { Self { intents: Vec::new() } }
       pub fn register(&mut self, intent: RecoveryIntent) { self.cleanup_expired(); self.intents.push(intent); }
       pub fn has_active_intent(&self, pod_id: &str, process: &str) -> Option<&RecoveryIntent> {

       pub fn cleanup_expired(&mut self) { self.intents.retain(|i| !i.is_expired()); }
   }

   {
       let ownership = state.process_ownership.lock().unwrap_or_else(|e| e.into_inner());

   Note: rc-agent.exe is registered to RcSentry (the pod-side authority). PodHealer should NOT restart rc-agent.exe directly -- it should only do WoL (machine-level, not process-level). Adjust the ownership check: PodHealer should check if it owns the "wol" action for the pod, OR if there's no active intent from another authority. The key insight: ProcessOwnership is for process-level restarts. pod_healer's TierOneRestart (which does `rc-agent restart via pod-agent`) should check ownership. WoL is a machine-level action not covered by process ownership.

   For TierOneRestart step specifically, add before the HTTP exec call:

           if owner != RecoveryAuthority::PodHealer {
               tracing::info!(target: "pod_healer", "Pod {} rc-agent.exe owned by {:?} -- skipping Tier 1 restart, advancing to AI escalation", pod.id, owner);

       }
   }
   ```

       let intents = state.recovery_intents.lock().unwrap_or_else(|e| e.into_inner());

           tracing::info!(

               "Pod {} has active recovery intent from {:?} ({}), skipping -- TTL expires in {}s",
               pod.id, active.authority, active.reason,

           return;
       }
   }

   Before acting in run_graduated_recovery, query rc-sentry /files on the pod to check for the sentinel:

   // COORD-03: Check GRACEFUL_RELAUNCH sentinel via rc-sentry
   let sentry_url = format!("http://{}:8091/files?path=C%3A%5CRacingPoint%5CGRACEFUL_RELAUNCH", pod.ip_address);
   match state.http_client.get(&sentry_url).timeout(Duration::from_secs(3)).send().await {
       Ok(resp) if resp.status().is_success() => {
           tracing::info!(
               target: "pod_healer",
               "Pod {} GRACEFUL_RELAUNCH sentinel present -- intentional restart in progress, skipping recovery",
               pod.id
           );
           let decision = RecoveryDecision::new("server", "rc-agent.exe", RecoveryAuthority::PodHealer, RecoveryAction::SkipCascadeGuardActive, "graceful_relaunch_sentinel_present");

           return;
       }
       _ => {} // Sentinel absent or sentry unreachable -- proceed
   }
   ```
