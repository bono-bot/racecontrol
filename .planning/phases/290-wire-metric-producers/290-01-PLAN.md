  <action>
Create `crates/racecontrol/src/metrics_producers.rs` with a single public function:

```rust
pub fn spawn_metric_producers(state: Arc<AppState>, metrics_tx: MetricsSender)
```

This function spawns ONE tokio task that runs a 30-second interval loop. Each tick:

1. **WS connections** (METRIC_WS_CONNECTIONS): Read `state.agent_senders` (snapshot+drop lock pattern), count entries. Send one sample with `pod_id: None`, value = count as f64.

2. **Game session count** (METRIC_GAME_SESSION_COUNT): Read `state.game_launcher.active_games` (snapshot+drop), count entries. Send one sample with `pod_id: None`, value = count as f64.

3. **Pod health scores** (METRIC_POD_HEALTH_SCORE): Read `state.pod_fleet_health` (snapshot+drop). For each pod that is `http_reachable`, send a sample with `pod_id: Some(pod_id)` and value = the pod's health score (check the PodFleetHealthStore struct for the exact field name -- might be a computed value from uptime or a stored field). If no explicit health score field exists, use `1.0` for reachable pods and `0.0` for unreachable as a binary health indicator.

4. **Billing revenue** (METRIC_BILLING_REVENUE): Query `SELECT COALESCE(SUM(total_amount_paise), 0) FROM billing_sessions WHERE date(created_at) = date('now')` from `state.db`. Send one sample with `pod_id: None`, value = result / 100.0 (convert paise to rupees). Use `sqlx::query_scalar`. If query fails, log warn and skip.

For timestamps, use `chrono::Utc::now().to_rfc3339()`.

For sending, use `metrics_tx.try_send(sample).ok()` -- do NOT await `.send()` (non-blocking, drop if channel full).

No `.unwrap()` anywhere. Log lifecycle: info on start, warn on individual errors, error only if the entire loop panics.

Add `pub mod metrics_producers;` to `crates/racecontrol/src/lib.rs`.

Add unit test `metrics_producers_builds` that just confirms the module compiles (call the function signature in a type-check context or test a helper).
