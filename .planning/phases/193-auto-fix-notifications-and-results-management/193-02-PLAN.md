  <action>
Create audit/lib/notify.sh with the following structure:

1. Header comment explaining the three notification channels and that all are failure-safe (notification failure never aborts audit).

2. COMMS_LINK_DIR constant:
COMMS_LINK_DIR="C:/Users/bono/racingpoint/comms-link"

3. _build_summary_text() helper:
Reads $RESULT_DIR/audit-summary.json via jq. Constructs a compact text message:
"Fleet Audit ($AUDIT_MODE) -- $verdict
Counts: $pass PASS, $fail FAIL, $warn WARN, $quiet QUIET
Severity: $p1_count P1, $p2_count P2, $p3_count P3
Fixes applied: $fix_count"

If $RESULT_DIR/delta.json exists and has_previous is true, append delta section:
"Delta: $regressions regressions, $improvements improvements, $new_issues new issues"
If regressions > 0, append: "REGRESSIONS DETECTED -- review required"

This satisfies NOTF-05 (delta summary in notification).
Return the text via printf.

4. _notify_bono_ws() function (NOTF-01):
Takes message text as arg. Checks COMMS_PSK and COMMS_URL env vars are set. If not, echo warning and return 0 (don't fail). Runs:
cd "$COMMS_LINK_DIR" && COMMS_PSK="$COMMS_PSK" COMMS_URL="$COMMS_URL" node send-message.js "$message" 2>/dev/null
Wrap in subshell with timeout (15s). On failure, echo warning but return 0.

5. _notify_bono_inbox() function (NOTF-02):
Takes message text as arg. Checks COMMS_LINK_DIR/INBOX.md exists. Generates IST timestamp header: "## YYYY-MM-DD HH:MM IST -- from james (audit)". Appends header + message to INBOX.md. Then:
cd "$COMMS_LINK_DIR" && git add INBOX.md && git commit -m "audit: fleet audit summary ($AUDIT_MODE)" && git push 2>/dev/null
Wrap everything in a subshell. On any failure, echo warning but return 0.

6. _notify_whatsapp_uday() function (NOTF-03):
Takes message text as arg. Sends WhatsApp to Uday via Bono relay. Use the comms-link relay exec endpoint:
Write JSON payload to temp file: {"command":"whatsapp_send","params":{"number":"919XXXXXXXXXX","message":"$message"}}
Note: Uday's exact number should be read from env var UDAY_WHATSAPP if set, otherwise skip WhatsApp with a warning. The number format must include country code 91.
curl -s -m 15 -X POST http://localhost:8766/relay/exec/run -H "Content-Type: application/json" -d @tmpfile 2>/dev/null
On any failure (relay down, no number configured), echo warning but return 0.

7. send_notifications() entry function (NOTF-04):
FIRST LINE: if [[ "${NOTIFY:-false}" != "true" ]]; then return 0; fi -- off-by-default gate.
Echo "--- Notifications ---"
Build summary text: local summary; summary=$(_build_summary_text)
If summary is empty (jq failed, no audit-summary.json), create a minimal fallback: "Fleet Audit ($AUDIT_MODE) completed. Results in $RESULT_DIR"
Call _notify_bono_ws "$summary" (WS channel)
Call _notify_bono_inbox "$summary" (INBOX.md channel)
Call _notify_whatsapp_uday "$summary" (WhatsApp channel)
Echo "--- Notifications Complete ---"
ALWAYS return 0 -- notification failures must never affect audit exit code.
Export with export -f send_notifications.

All notification calls are wrapped so that failures (network down, comms-link unavailable, etc.) print warnings to stderr but never abort the audit or change exit code.
