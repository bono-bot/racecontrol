  <action>
Create audit/lib/fixes.sh with the following structure:

1. Header and APPROVED_FIXES whitelist (FIX-08):
Define APPROVED_FIXES=("clear_stale_sentinels" "kill_orphan_powershell" "restart_rc_agent"). This is the explicit whitelist array. No fix function runs unless its name is in this array. Add a helper _is_approved_fix() that iterates the array and returns 0 if found, 1 otherwise.

2. IP-to-pod-number mapping:
Create _ip_to_pod_number() function. Maps: .89=1, .33=2, .28=3, .88=4, .86=5, .87=6, .38=7, .91=8. Used by is_pod_idle to find the pod in fleet health JSON.

3. is_pod_idle() gate function (FIX-02):
Takes pod IP as argument. Uses a module-level cached variable _FLEET_HEALTH_CACHE. If cache is empty, fetches fleet health from $FLEET_HEALTH_ENDPOINT with 8s timeout via curl. Looks up the pod by pod_number (from _ip_to_pod_number). Returns 0 (idle) if active_billing_session and billing_active are both false/null. Returns 1 (busy) if either is true. On ANY failure (curl timeout, jq parse error, pod not found in response), returns 1 (fail-safe: assume busy, never risk touching an active session).

4. check_pod_sentinels() function (FIX-03):
Takes pod IP as argument. Uses safe_remote_exec to check if C:\RacingPoint\OTA_DEPLOYING exists on the pod via: "if exist C:\\RacingPoint\\OTA_DEPLOYING echo OTA_ACTIVE". If output contains OTA_ACTIVE, return 1 (skip fixes). Return 0 otherwise. Note: MAINTENANCE_MODE is NOT a skip condition here because clearing it IS one of the fixes (FIX-04).

5. Fix function: clear_stale_sentinels (FIX-04):
Guard with _is_approved_fix "clear_stale_sentinels" at entry. Check for MAINTENANCE_MODE, GRACEFUL_RELAUNCH, rcagent-restart-sentinel.txt via safe_remote_exec using: "if exist C:\\RacingPoint\\MAINTENANCE_MODE echo MM & if exist C:\\RacingPoint\\GRACEFUL_RELAUNCH echo GR & if exist C:\\RacingPoint\\rcagent-restart-sentinel.txt echo RS". Before state = which files found. If none found, return early (nothing to fix). Delete found files via: "del /Q C:\\RacingPoint\\MAINTENANCE_MODE C:\\RacingPoint\\GRACEFUL_RELAUNCH C:\\RacingPoint\\rcagent-restart-sentinel.txt 2>nul". Re-check after deletion. Call emit_fix "sentinel" "$pod_ip" "clear_stale_sentinels" "$before" "$after".

6. Fix function: kill_orphan_powershell (FIX-05):
Guard with _is_approved_fix. Count powershell.exe via safe_remote_exec: "tasklist /FI \"IMAGENAME eq powershell.exe\" /NH 2>nul | findstr /I powershell | find /C /V \"\"". Parse the count. If count <= 1, return (no orphans). If count > 1, kill all: "taskkill /F /IM powershell.exe 2>nul" (the watchdog will auto-restart one). Before state = "powershell_count=$count". After state = "killed_all, watchdog_will_restart". Call emit_fix.

7. Fix function: restart_rc_agent (FIX-06):
Guard with _is_approved_fix. Check rc-agent health: http_get "http://${pod_ip}:8090/health" 5. If reachable (non-empty response), return (nothing to fix). If NOT reachable, check rc-sentry: http_get "http://${pod_ip}:8091/health" 5. If rc-sentry is also down, skip (cannot restart without rc-sentry). If rc-sentry IS up, restart via rc-sentry exec: safe_remote_exec "$pod_ip" 8091 "schtasks /Run /TN StartRCAgent" 10. Before state = "rc-agent_down,rc-sentry_up". After state = "schtasks_triggered". Call emit_fix.

8. run_auto_fixes() entry function (FIX-01):
FIRST LINE: if [[ "${AUTO_FIX:-false}" != "true" ]]; then return 0; fi -- this is the off-by-default gate. Then: echo "--- Auto-Fix Engine ---". Reset _FLEET_HEALTH_CACHE="" to force fresh fetch. Initialize counters: fixes_applied=0, pods_skipped=0. For each pod_ip in $PODS: (a) is_pod_idle check -- if busy, emit_fix "autofix" "$pod_ip" "SKIP_ACTIVE_SESSION" "billing_active" "skipped" and increment pods_skipped, continue. (b) check_pod_sentinels -- if OTA active, emit_fix "autofix" "$pod_ip" "SKIP_OTA_DEPLOYING" "ota_deploying" "skipped" and continue. (c) For each fix_name in APPROVED_FIXES: call "$fix_name" "$pod_ip" and increment fixes_applied on success. Print summary. Echo "--- Auto-Fix Complete ---". Export run_auto_fixes with export -f.

All functions must use export -f for subshell compatibility. Follow the established pattern from core.sh.
