  <action>
Create `crates/rc-sentry-ai/src/mjpeg.rs` with:

1. **MjpegState struct** containing:
   - `frame_buf: FrameBuffer`
   - `cameras: Vec<CameraConfig>` (cloned from config at startup)
   - `service_port: u16` (to build absolute stream URLs in camera list response)

2. **GET /api/v1/cameras** handler:
   - Returns JSON array of objects: `{ name, role, stream_url, status }`.
   - `stream_url` = `"/api/v1/cameras/{name}/stream"` (relative, dashboard constructs full URL).
   - `status` derived from FrameBuffer: call `frame_buf.get(&name)` -- if Some and timestamp < 10s ago = "connected", < 30s = "reconnecting", else "disconnected". If None = "offline".
   - Iterate `self.cameras` to enumerate, not FrameBuffer keys (shows configured cameras even if no frames yet).

3. **GET /api/v1/cameras/:name/stream** MJPEG handler:
   - Extract `:name` path param. Return 404 JSON `{"error":"camera not found"}` if name not in `cameras` list.
   - Set response headers: `Content-Type: multipart/x-mixed-replace; boundary=frame`, `Cache-Control: no-cache, no-store`, `Connection: keep-alive`.
   - Use `axum::body::Body::from_stream()` with a `futures::stream::unfold` that:
     a. Reads frame from `frame_buf.get(&name)`.
     b. If Some, format as MJPEG part: `--frame\r\nContent-Type: image/jpeg\r\nContent-Length: {len}\r\n\r\n{jpeg_bytes}\r\n`.
     c. If None (no frame yet), send a 1x1 transparent JPEG placeholder or skip.
     d. Sleep for `1000 / fps` ms (use camera's configured fps, capped to max 10 FPS to avoid overwhelming browser).
     e. Yield `Ok::<_, std::io::Error>(bytes)` for stream compatibility.
   - The stream runs until the client disconnects (TCP connection drops, unfold returns None on send error).
   - CRITICAL: Only call `frame_buf.get()` (read lock), never `update()`. This ensures zero contention with the AI detection pipeline.

4. **CORS layer** on the MJPEG router:
   - Add `tower_http::cors::CorsLayer` allowing:
     - Origins: `Any` (LAN-only service, dashboard at :3200 needs cross-origin img access).
     - Methods: GET.
     - Headers: any.
   - This is required because the dashboard at :3200 loads img src from :8096.

5. **pub fn mjpeg_router(state: Arc&lt;MjpegState&gt;) -> axum::Router** that builds the route tree.

IMPORTANT: FrameData.data is raw decoded pixels (not JPEG). The FrameBuffer stores decoded frame data. To serve MJPEG, the frames must be re-encoded to JPEG. Use the `image` crate (already in Cargo.toml) to encode `FrameData.data` to JPEG. Check how stream.rs decodes frames to understand the pixel format. If FrameData.data is already JPEG bytes from go2rtc snapshot, then pass through directly. Read `crates/rc-sentry-ai/src/stream.rs` to determine the format.
