  <action>
1. Add `winrt-toast = "0.1"` to `crates/rc-sentry-ai/Cargo.toml` under [dependencies].

2. Add `pub mod toast;` to `crates/rc-sentry-ai/src/alerts/mod.rs`.

3. Create `crates/rc-sentry-ai/src/alerts/toast.rs`:
   - `pub async fn run(mut alert_rx: broadcast::Receiver<AlertEvent>)`
   - Loop receiving AlertEvent from alert_rx broadcast
   - For each event, spawn_blocking to call the winrt-toast API (it's synchronous COM):
     ```rust
     use winrt_toast::{Toast, Text, Header};
     use winrt_toast::content::text::TextPlacement;
     ```
   - For `AlertEvent::Recognized`: toast title = "Person Detected", body line 1 = person_name, body line 2 = camera + timestamp (IST formatted)
   - For `AlertEvent::UnknownPerson`: toast title = "Unknown Person", body line 1 = "Unrecognized face detected", body line 2 = camera + timestamp (IST)
   - Set audio to system default: `toast.audio(winrt_toast::content::audio::Audio::new(winrt_toast::content::audio::KnownSound::Default, false))`
   - Use AUM ID "Racing Point Sentry" (or Windows.SystemToasts.* if custom AUM fails) — try `winrt_toast::Toast::new()` first, if it needs an AUM, use a constant like `"RacingPoint.Sentry"`
   - Handle errors with tracing::warn (toast failure should never crash the service)
   - Handle Lagged/Closed same as attendance engine pattern
   - Convert UTC timestamp to IST (chrono::FixedOffset::east_opt(5*3600+1800)) for display

4. Important: winrt-toast is Windows-only. Wrap the module body in `#[cfg(target_os = "windows")]` and provide a no-op stub for non-Windows:
   ```rust
   #[cfg(not(target_os = "windows"))]
   pub async fn run(mut alert_rx: broadcast::Receiver<AlertEvent>) {
       tracing::info!("toast notifications not available on this platform");
       // drain the channel to avoid backpressure
       loop {
           match alert_rx.recv().await {
               Err(broadcast::error::RecvError::Closed) => break,
               _ => continue,
           }
       }
   }
   ```
