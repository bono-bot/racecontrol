
Inside:
a) Query matching events:
```sql
SELECT id, reference_time_ms FROM hotlap_events
WHERE track = ? AND car_class = ? AND sim_type = ?
  AND status IN ('active', 'upcoming')
  AND (starts_at IS NULL OR datetime(starts_at) <= datetime('now'))
  AND (ends_at IS NULL OR datetime(ends_at) >= datetime('now'))
```
Use `.fetch_all(pool).await.unwrap_or_default()`

b) For each matching event:
- Check existing entry: `SELECT lap_time_ms FROM hotlap_event_entries WHERE event_id = ? AND driver_id = ?`
- If existing entry is faster or equal, skip (continue)
- Compute badge from reference_time_ms:
  - If reference_time_ms is None: badge = None (SQL NULL)
  - ratio = lap_time_ms as f64 / ref_ms as f64
  - ratio <= 1.02 -> "gold", <= 1.05 -> "silver", <= 1.08 -> "bronze", else -> "none"
- UPSERT: `INSERT INTO hotlap_event_entries (...) VALUES (...) ON CONFLICT(event_id, driver_id) DO UPDATE SET ...`
  - Include: id (new UUID), event_id, driver_id, lap_id, lap_time_ms, sector1_ms, sector2_ms, sector3_ms (cast f32 to i64), badge, result_status='finished', entered_at
- After upsert, call recalculate_event_positions(pool, &event_id)
- Log with tracing::info!

c) Sector conversion: `sector.map(|v| v as i64)` for i64 storage.

**2. `recalculate_event_positions()`**:

```rust
async fn recalculate_event_positions(pool: &sqlx::SqlitePool, event_id: &str) {
```

Uses a two-step approach (SQLite doesn't support UPDATE with window functions in all versions):
a) Query ranked entries:
```sql
SELECT id, lap_time_ms FROM hotlap_event_entries
WHERE event_id = ? AND result_status = 'finished'
ORDER BY lap_time_ms ASC
```
b) Find leader_ms (first row's lap_time_ms)
c) For each entry, UPDATE position, gap_to_leader_ms, and within_107_percent:
- position = 1-indexed row number
- gap_to_leader_ms = lap_time_ms - leader_ms
- within_107_percent: use integer math `lap_time_ms * 100 <= leader_ms * 107` -> 1 else 0

This avoids the window function approach which is complex to do as UPDATE in SQLite.
