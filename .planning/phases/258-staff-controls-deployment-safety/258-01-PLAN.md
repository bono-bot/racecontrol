**STAFF-01: Discount approval flow.**
1. Add `discount_approval_threshold_paise: i64` config constant (default 5000 = Rs.50). Place near existing pricing constants in billing.rs or a config module.
2. Create `POST /billing/{id}/discount` handler in staff_routes (cashier+ access). Request body: `{ "discount_paise": i64, "reason_code": String, "manager_approval_code": Option<String> }`. Logic:
   - If `discount_paise > discount_approval_threshold_paise` AND `manager_approval_code` is None or invalid, return 403 with `"Discount above threshold requires manager approval code"`.
   - Validate `manager_approval_code` by looking up staff table for a manager/superadmin with matching PIN hash (reuse existing PIN validation from `staff_validate_pin`).
   - Apply discount: UPDATE billing_sessions SET discount_paise = discount_paise + ?, discount_reason = ? WHERE id = ? AND status IN ('active', 'paused_manual', 'paused_game_pause', 'paused_disconnect').
   - Insert audit_log entry: action='discount_applied', actor_id=staff_claims.user_id, details=JSON with discount_paise, reason_code, manager_approved (bool).
   - Return 200 with applied discount amount.
3. Below-threshold discounts proceed without manager approval but still require reason_code and audit log entry.

**STAFF-02: Mark as already complete (SEC-05).** No implementation needed. Self-topup block exists at routes.rs:7028 with tests at routes.rs:19218.

**STAFF-03: Daily override report.**
4. Create `GET /admin/reports/daily-overrides` handler in manager_routes section. Query parameters: `date` (optional, defaults to today IST).
5. SQL query joins:
   - billing_sessions WHERE discount_paise > 0 AND date(started_at) = ?date — staff discounts
   - wallet_transactions WHERE type = 'manual_refund' AND date(created_at) = ?date — manual refunds
   - audit_log WHERE action IN ('discount_applied', 'tier_change', 'manual_refund') AND date(created_at) = ?date — all override actions
6. Return JSON array of override entries, each with: `{ timestamp, action_type, actor_id, actor_name, target_driver, amount_paise, reason, details }`.

**STAFF-04: Cash drawer reconciliation.**
7. Create `GET /admin/reports/cash-drawer` in manager_routes. Returns: `{ system_cash_total_paise: i64, date: String }` — sum of wallet_transactions WHERE method='cash' AND date(created_at) = today.
8. Create `POST /admin/reports/cash-drawer/close` in manager_routes. Request body: `{ physical_count_paise: i64 }`. Logic:
   - Compute system total (same query as GET).
   - Calculate discrepancy = physical_count_paise - system_cash_total_paise.
   - INSERT into audit_log: action='cash_drawer_close', details=JSON with system_total, physical_count, discrepancy, actor_id.
   - Return `{ system_total_paise, physical_count_paise, discrepancy_paise, status: "over"|"under"|"balanced" }`.

Wire all new routes into the existing staff_routes() function:
- `/billing/{id}/discount` in the main staff section (cashier+ access)
- `/admin/reports/daily-overrides`, `/admin/reports/cash-drawer`, `/admin/reports/cash-drawer/close` in the manager_routes `.merge()` block (line 530 area)
