# Phase 294 Plan 01: Intelligence Report v2

## Objective

Enhance `weekly_report.rs` with 4 new WhatsApp report sections using data from the
stores built in Phases 290-292: per-model accuracy rankings, KB promotion count, cost
savings from Tier 1 hardened rules, and model accuracy trends.

## Context

- @crates/rc-agent/src/weekly_report.rs — existing report to enhance
- @crates/rc-agent/src/model_eval_store.rs — EvalRecord + ModelEvalStore::query_all()
- @crates/rc-agent/src/eval_rollup.rs — EvalRollup + compute_rollup()
- @crates/rc-agent/src/kb_promotion_store.rs — KbPromotionStore::all_candidates()
- @crates/rc-agent/src/model_reputation_store.rs — ModelReputationStore::load_all_outcomes()
- @crates/rc-agent/src/main.rs — weekly_report::spawn() call needs new store args

## Tasks

### Task 1 — Enhance WeeklyReport struct + collect_report() with store data [type=auto]

Extend the `WeeklyReport` struct with 4 new fields and update `collect_report()` to
populate them from `ModelEvalStore`, `KbPromotionStore`, and `ModelReputationStore`.

**New fields:**
- `model_accuracy_rankings: Vec<(String, f64, u32)>` — (model_id, accuracy, run_count), sorted desc by accuracy
- `kb_promotions_this_week: u32` — candidates that advanced stage in the past 7 days
- `hardened_rule_savings_usd: f64` — hardened_applications * estimated_model_cost ($0.001/call default)
- `model_trends: Vec<(String, String)>` — (model_id, "improving"/"declining"/"stable") from reputation store

**collect_report() changes:**
- Accept new optional store Arc refs (eval_store, promo_store, rep_store) — wrapped in Option
  so it degrades gracefully if stores unavailable
- For rankings: open ModelEvalStore, query_all(from=7-days-ago, to=now), compute_rollup(),
  sort by accuracy desc — up to top 5 models
- For KB promotions: open KbPromotionStore, all_candidates(), count those where
  stage_entered_at is within the past 7 days AND stage != "observed"
- For cost savings: count hardened candidates * 0.001 USD estimated cost
- For trends: load reputation rows, compute trend from current accuracy vs a threshold:
  >70% = improving-zone, <30% = declining-zone, compare status field

**Done criteria:**
- `WeeklyReport` struct compiles with 4 new fields
- `collect_report()` accepts store arcs and populates fields without panics
