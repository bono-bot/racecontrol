Create a Task Scheduler task that starts WSL2 Salt services when the bono user logs in. Per user decision: at-login trigger (not at-startup SYSTEM), because WSL2 requires an active user session. James auto-logs in, so Salt is available within seconds of boot.

**Step 1 — Create the startup batch script**

Write `C:\Users\bono\racingpoint\racecontrol\deploy-staging\start-salt.bat` with this exact content:
```bat
@echo off
REM Start WSL2 Salt services after login
REM Phase 36 — INFRA-05 auto-start
REM Delay 5s to let WSL2 initialize (Pitfall 4: WSL2 not ready at logon)
timeout /t 5 /nobreak > nul
wsl -d Ubuntu-24.04 -e bash -c "sudo service salt-master start && sudo service salt-api start"
```

After writing the file, fix line endings for Windows:
```bash
sed -i 's/$/\r/' /c/Users/bono/racingpoint/racecontrol/deploy-staging/start-salt.bat
```

**Step 2 — Register the Task Scheduler task via PowerShell**

Run in elevated PowerShell:
```powershell
$action = New-ScheduledTaskAction -Execute "cmd.exe" `
    -Argument '/c "C:\Users\bono\racingpoint\racecontrol\deploy-staging\start-salt.bat"'
$trigger = New-ScheduledTaskTrigger -AtLogOn -User "bono"
$settings = New-ScheduledTaskSettingsSet -ExecutionTimeLimit (New-TimeSpan -Minutes 2) -StartWhenAvailable
$principal = New-ScheduledTaskPrincipal -UserId "bono" -LogonType Interactive -RunLevel Highest
Register-ScheduledTask -TaskName "SaltMasterStart" -Action $action -Trigger $trigger -Settings $settings -Principal $principal -Force
```

Key decisions in this config:
- `-Execute "cmd.exe"` with `/c` flag — more reliable than calling `wsl` directly from Task Scheduler
- `-AtLogOn -User "bono"` — fires when bono logs in (per locked decision)
- `-RunLevel Highest` — elevated for the wsl + sudo calls
- `-ExecutionTimeLimit 2 minutes` — the batch has a 5s delay + service start; 2 min is generous
- `-StartWhenAvailable` — if login happens during boot and the trigger is missed, re-fires when available
- `-Force` — overwrite if task already exists

**Step 3 — Verify the task was created**

```powershell
Get-ScheduledTask -TaskName "SaltMasterStart" | Format-List TaskName, State, Description
Get-ScheduledTaskInfo -TaskName "SaltMasterStart" | Format-List LastRunTime, LastTaskResult, NextRunTime
```

The task should show State: `Ready`.
