## Credit Format Contract

This is the core output of Phase 35. Defines exactly how credits appear on every surface.

### Format Rules

| Context | Format | Example | Notes |
|---------|--------|---------|-------|
| Session cost (admin history, modal, overlay) | `"X cr"` | `"1050 cr"` | Integer floor division from paise (`Math.floor(paise / 100)`) |
| Per-minute rate (pricing table, tier prompts) | `"X cr/min"` | `"25 cr/min"` | Always integer, no decimals |
| Kiosk tier price chip | `"X credits"` | `"700 credits"` | Long form acceptable for customer-facing kiosk — wider audience |
| Kiosk wallet balance | `"X credits"` | `"1050 credits"` | Long form — wallet topup context |
| Overlay HUD | `"X cr"` | `"45 cr"` | GDI text space-constrained, short form mandatory |
| Summary totals (daily report) | `"X cr"` | `"3150 cr"` | Consistent with history table |

### Consistency Rule

- Admin panel: always `"X cr"` (short form) — staff context, compact tables
- Kiosk: always `"X credits"` (long form) — customer context, touch UI
- Overlay (Rust GDI): always `"X cr"` — pixel budget is limited
- Never use `"₹"`, `"Rs."`, `"₹X"`, or `"Rs. X"` on any surface

### Shared Utility

Both admin Next.js files define `formatCredits` identically:
```ts
const formatCredits = (paise: number) => `${Math.floor(paise / 100)} cr`;
```
This is a file-local function (not shared from a util module). That is acceptable for Phase 35 — do not refactor into a shared util unless scope requires it.

## Per-Minute Rates Table Design (Admin Pricing Page)

The table is already implemented. This section documents the final design contract for audit.

### Table Structure

```
┌──────────────┬─────────────┬─────────────┬────────┬─────────┐
│ Tier Name    │ Up To (min) │ Rate (cr/min)│ Active │ Actions │
├──────────────┼─────────────┼─────────────┼────────┼─────────┤
│ Standard     │ 30 min      │ 25 cr/min   │ [●]    │ Edit    │
│ Extended     │ 60 min      │ 20 cr/min   │ [●]    │ Edit    │
│ Marathon     │ Unlimited   │ 15 cr/min   │ [●]    │ Edit    │
└──────────────┴─────────────┴─────────────┴────────┴─────────┘
```

- "Unlimited" shown when `threshold_minutes === 0`
- Rate column: `font-mono font-bold text-neutral-300`
- Active toggle: `rp-red` when active, `rp-card` when inactive — matches existing tier toggle pattern
- Inline edit: clicking "Edit" replaces the row with text/number inputs, Save/Cancel actions inline
- Save action: calls `PUT /billing/rates/{id}` with `{ tier_name, threshold_minutes, rate_per_min_paise: rateEditRate * 100 }`
- No delete button on rates (rates are managed as a fixed 3-tier set, only deactivation via toggle)

### Section Header Copy

```
Per-Minute Rates
Non-retroactive tiered pricing (credits/min)     [N active tiers]
```
