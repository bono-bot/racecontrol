In the `migrate()` function in `db/mod.rs`, insert the following two table creations AFTER the `cafe_orders` indexes block (after line ~2462) and BEFORE the "Seed default categories" block.

**Table 1: cafe_promos**
```sql
CREATE TABLE IF NOT EXISTS cafe_promos (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    promo_type TEXT NOT NULL CHECK(promo_type IN ('combo', 'happy_hour', 'gaming_bundle')),
    config TEXT NOT NULL DEFAULT '{}',
    is_active BOOLEAN NOT NULL DEFAULT 0,
    start_time TEXT,
    end_time TEXT,
    stacking_group TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT
)
```

- `config` stores JSON string. Schema varies by promo_type:
  - combo: `{"items": [{"id": "...", "qty": 1}], "bundle_price_paise": 14900}`
  - happy_hour: `{"discount_percent": 20, "applies_to": "category", "target_ids": ["..."]}`
  - gaming_bundle: `{"session_duration_mins": 60, "cafe_item_ids": ["..."], "bundle_price_paise": 99900}`
- `start_time` / `end_time`: HH:MM strings in IST (e.g. "15:00", "18:00") for happy_hour windows. NULL for non-time-based promos.
- `stacking_group`: promos sharing the same non-null group string are mutually exclusive (only one can apply per order). NULL means no exclusivity.

**Table 2: cafe_promo_stacking** — not needed as a separate table; `stacking_group` on `cafe_promos` is sufficient (per Claude's Discretion in CONTEXT.md). Skip this table.

Add one index:
```sql
CREATE INDEX IF NOT EXISTS idx_cafe_promos_active ON cafe_promos(is_active)
```

Use `sqlx::query("...").execute(pool).await?` (non-idempotent CREATE TABLE IF NOT EXISTS). The index also uses `await?`.

No `.unwrap()` anywhere.

3. Handlers — all follow the `(StatusCode, Json<T>)` / `(StatusCode, Json<serde_json::Value>)` return pattern from cafe.rs:

- **`list_cafe_promos`** — `GET /cafe/promos`
  `SELECT * FROM cafe_promos ORDER BY created_at DESC`
  Returns `(StatusCode::OK, Json(Vec<CafePromo>))`

- **`create_cafe_promo`** — `POST /cafe/promos`
  Validate promo_type with `validate_promo_type`. Return 400 if invalid.
  Serialize `req.config` to string with `serde_json::to_string`.
  INSERT with `Uuid::new_v4().to_string()` as id.
  Re-fetch inserted row and return `(StatusCode::CREATED, Json(CafePromo))`.

- **`update_cafe_promo`** — `PUT /cafe/promos/{id}`
  Build UPDATE SQL dynamically only for `Some` fields. Use `updated_at = datetime('now')` always.
  If no fields provided return 400 `{"error": "no fields to update"}`.
  Re-fetch and return `(StatusCode::OK, Json(CafePromo))`.
  Return 404 if id not found.

- **`delete_cafe_promo`** — `DELETE /cafe/promos/{id}`
  `DELETE FROM cafe_promos WHERE id = ?`
  Return `StatusCode::NO_CONTENT` (no body) on success.
  Return 404 if rows_affected == 0.

- **`toggle_cafe_promo`** — `POST /cafe/promos/{id}/toggle`
  `UPDATE cafe_promos SET is_active = NOT is_active, updated_at = datetime('now') WHERE id = ?`
  Re-fetch and return `(StatusCode::OK, Json(CafePromo))`.
  Return 404 if id not found.

**No `.unwrap()` anywhere.** Use `?` operator or map errors to `(StatusCode::INTERNAL_SERVER_ERROR, Json(json!({"error": e.to_string()})))`.
