  <action>
    1. Add `imageproc = "0.26"` to `[dependencies]` in rc-sentry-ai/Cargo.toml.

    2. Add `pub mod alignment;` to recognition/mod.rs (Plan 01 creates mod.rs -- if running in parallel, executor adds this line).

    3. Create `crates/rc-sentry-ai/src/recognition/alignment.rs` with:

       - `ARCFACE_REF: [[f32; 2]; 5]` constant: `[[38.2946, 51.6963], [73.5318, 51.5014], [56.0252, 71.7366], [41.5493, 92.3655], [70.7299, 92.2041]]`

       - `pub fn estimate_similarity_transform(src: &[[f32; 2]; 5]) -> [f32; 6]`:
         Solve for similarity transform [a, -b, tx, b, a, ty] using least-squares over all 5 point pairs.
         Build 10x4 system (2 equations per point): `dx = a*sx - b*sy + tx`, `dy = b*sx + a*sy + ty`.
         Compute A^T*A (4x4) and A^T*b (4x1), solve with `solve_4x4()`.
         Return `[a, -b, tx, b, a, ty]` as f32.

       - `fn solve_4x4(ata: &[[f64; 4]; 4], atb: &[f64; 4]) -> [f64; 4]`:
         Gaussian elimination with partial pivoting on the 4x4 augmented matrix.

       - `pub fn align_face(frame_rgb: &[u8], frame_w: u32, frame_h: u32, landmarks: &[[f32; 2]; 5]) -> image::RgbImage`:
         1. Call `estimate_similarity_transform(landmarks)` to get affine params.
         2. Convert affine [a, -b, tx, b, a, ty] to `imageproc::geometric_transformations::Projection` (3x3 matrix with last row [0,0,1]). The Projection needs the INVERSE transform (from output to input), so invert the 2x3 affine.
         3. Build full-frame RgbImage from raw rgb bytes.
         4. Call `imageproc::geometric_transformations::warp(&img, &projection, interpolation::Bilinear, image::Rgb([0,0,0]))`.
         5. Crop/resize result to 112x112 and return.

         IMPORTANT: The Projection maps from destination to source. Compute the inverse of the similarity transform for this.

    4. Add `#[cfg(test)] mod tests` with the behavior tests. For the align_face test, create a synthetic 640x480 image with `image::RgbImage::from_fn(640, 480, |x, y| ...)` and use ARCFACE_REF scaled to image coords as landmarks.

    2. Create `crates/rc-sentry-ai/src/recognition/arcface.rs` following the EXACT same pattern as `detection/scrfd.rs`:

       - `pub struct ArcfaceRecognizer { session: Arc<Mutex<Session>> }`

       - `pub fn new(model_path: &str) -> anyhow::Result<Self>`:
         - Use `Session::builder()` with `ep::CUDA::default().build().error_on_failure()`.
         - Log all input/output node names and shapes (like SCRFD does).
         - Verify expected shapes at load: input should be [1, 3, 112, 112], output should be [1, 512].
         - Wrap session in `Arc::new(Mutex::new(session))`.

       - `pub fn clone_shared(&self) -> Self`: Arc clone (same as SCRFD).

       - `pub fn preprocess(aligned_rgb: &image::RgbImage) -> ndarray::Array4<f32>`:
         - Assert dimensions are 112x112.
         - Create NCHW tensor [1, 3, 112, 112].
         - Normalize: `(pixel - 127.5) / 127.5` (maps [0,255] to [-1.0, 1.0]).
         - Note: NOT the same as SCRFD normalization which uses /128.0. ArcFace glintr100.onnx uses /127.5.

       - `pub async fn extract_embedding(&self, tensor: ndarray::Array4<f32>) -> anyhow::Result<[f32; 512]>`:
         - Clone Arc, spawn_blocking (same pattern as ScrfdDetector::detect).
         - Lock session, create Tensor from array, run inference.
         - Extract first output tensor, convert to Vec<f32>.
         - Verify length == 512.
         - L2-normalize: compute norm = sqrt(sum(x^2)), divide each element by norm.
         - Return `[f32; 512]`.

    3. Add unit test for `preprocess`: verify output tensor shape is [1, 3, 112, 112] and values are in [-1.0, 1.0] range.

    NOTE: Integration tests requiring the actual ONNX model file are in Plan 03 (pipeline integration). This task tests preprocessing only. The model file (glintr100.onnx) must be present at C:\RacingPoint\models\ for runtime use.
