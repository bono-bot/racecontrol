  <action>
    1. Add `mod recognition;` to main.rs (after existing `mod detection;`).

    2. Create `crates/rc-sentry-ai/src/recognition/mod.rs`:
       ```
       pub mod types;
       pub mod quality;
       pub mod clahe;
       ```

    3. Create `crates/rc-sentry-ai/src/recognition/types.rs` with:
       - `RejectReason` enum: `TooSmall { width: u32, height: u32 }`, `TooBlurry { laplacian_var: f64 }`, `ExcessiveYaw { estimated_yaw: f64 }`
       - `RecognitionResult` struct: `person_id: i64`, `person_name: String`, `confidence: f32`, `camera: String`, `timestamp: chrono::DateTime<chrono::Utc>`
       - `GalleryEntry` struct: `person_id: i64`, `person_name: String`, `embedding: [f32; 512]`
       - All types derive Debug, Clone, serde::Serialize where appropriate.

    4. Create `crates/rc-sentry-ai/src/recognition/quality.rs` with:
       - `QualityGates` struct with fields: `min_face_size: u32` (default 80), `min_laplacian_var: f64` (default 100.0), `max_yaw_degrees: f64` (default 45.0)
       - `QualityGates::check(&self, face: &DetectedFace, frame_gray: &[u8], frame_w: u32, frame_h: u32) -> Result<(), RejectReason>`
       - `check_size`: compute bbox width = `(bbox[2] - bbox[0]) as u32`, height = `(bbox[3] - bbox[1]) as u32`. Reject if either < min_face_size.
       - `check_blur`: extract face crop from frame_gray using bbox coords (clamp to frame bounds). Compute Laplacian variance using 3x3 kernel `[0,1,0; 1,-4,1; 0,1,0]`. Reject if variance < min_laplacian_var.
       - `check_pose`: call `estimate_yaw(face.landmarks)`. Reject if yaw > max_yaw_degrees.
       - `estimate_yaw(landmarks: &[[f32; 2]; 5]) -> f64`: use eye-nose ratio method from RESEARCH.md. left_dist = |nose_x - left_eye_x|, right_dist = |right_eye_x - nose_x|. ratio = min/max. yaw = (1.0 - ratio) * 90.0.
       - `laplacian_variance(gray: &[u8], width: u32, height: u32) -> f64`: from RESEARCH.md code example.

    5. Add `#[cfg(test)] mod tests` in quality.rs with all the behavior tests listed above.

    6. In Cargo.toml, add NO new dependencies for this task (quality gates use only std + image which is already present). The `clahe` crate is added in Task 2.
