1. Create `decoder.rs` -- H.264 NAL to RGB via openh264:
   ```rust
   use openh264::decoder::Decoder;

   pub struct FrameDecoder {
       decoder: Decoder,
   }

   impl FrameDecoder {
       pub fn new() -> anyhow::Result<Self> {
           let decoder = Decoder::new()?;
           Ok(Self { decoder })
       }

       /// Decode H.264 NAL unit(s) to RGB pixels.
       /// Returns None if frame cannot be decoded (e.g., waiting for keyframe).
       pub fn decode(&mut self, nal_data: &[u8]) -> anyhow::Result<Option<DecodedFrame>> {
           // Feed NAL to decoder -- stateful, must keep decoder alive across frames
           let yuv = match self.decoder.decode(nal_data)? {
               Some(yuv) => yuv,
               None => return Ok(None),
           };
           let (width, height) = yuv.dimension_rgb();
           let mut rgb = vec![0u8; width * height * 3];
           yuv.write_rgb8(&mut rgb);
           Ok(Some(DecodedFrame {
               rgb,
               width: width as u32,
               height: height as u32,
           }))
       }
   }

   pub struct DecodedFrame {
       pub rgb: Vec<u8>,
       pub width: u32,
       pub height: u32,
   }
   ```
   CRITICAL: One FrameDecoder per camera stream -- H.264 is stateful (P-frames depend on previous frames). Creating a new decoder per frame will only decode I-frames.

2. Create `scrfd.rs` -- SCRFD ONNX inference:

   Struct `ScrfdDetector` wrapping `Arc<ort::session::Session>`.

   **Constructor `ScrfdDetector::new(model_path: &str) -> anyhow::Result<Self>`:**
   - Create ort::Session with CUDA EP using `ep::CUDA::default().build().error_on_failure()`
   - Log all input/output node names and shapes at info level for runtime verification
   - Wrap session in Arc for thread-safe sharing

   **Method `preprocess(rgb: &[u8], width: u32, height: u32) -> (ndarray::Array4<f32>, f32)`:**
   - Calculate scale = 640.0 / max(width, height) as f32
   - Resize to (new_w, new_h) maintaining aspect ratio using image::imageops::FilterType::Lanczos3
   - Create 640x640 canvas, paste resized at (0,0), zero-pad remainder
   - Normalize: (pixel - 127.5) / 128.0 per channel
   - Return NCHW tensor shape [1, 3, 640, 640] and the scale factor

   **Method `detect(&self, tensor: ndarray::Array4<f32>, det_scale: f32, conf_threshold: f32) -> anyhow::Result<Vec<DetectedFace>>`:**
   - Run inference: `session.run(ort::inputs!["input.1" => tensor.view()]?)?`
   - NOTE: The input name "input.1" should be verified at runtime. If the model uses a different name, use `session.inputs[0].name` instead.
   - Post-process across 3 FPN levels with strides [8, 16, 32]:
     a. Generate anchor centers for each stride level (grid of (x*stride + stride/2, y*stride + stride/2), 2 anchors per location)
     b. For each stride: extract scores, bboxes, keypoints tensors from session outputs (map by tensor dimension: dim[-1]==1 for scores, ==4 for bbox, ==10 for kps)
     c. Filter by conf_threshold (default 0.5)
     d. `distance2bbox(anchor, distance)`: anchor_x - left, anchor_y - top, anchor_x + right, anchor_y + bottom (all multiplied by stride)
     e. `distance2kps(anchor, distance)`: for each of 5 landmarks, anchor_x + dx*stride, anchor_y + dy*stride
     f. Divide all coordinates by det_scale to recover original image space
   - Apply greedy NMS with IoU threshold 0.4:
     a. Sort by confidence descending
     b. For each face, suppress all lower-confidence faces with IoU > 0.4
     c. IoU = intersection_area / union_area

   **Helper functions (private):**
   - `fn distance2bbox(anchor: (f32, f32), dist: &[f32], stride: f32) -> [f32; 4]`
   - `fn distance2kps(anchor: (f32, f32), dist: &[f32], stride: f32) -> [[f32; 2]; 5]`
   - `fn nms(faces: &mut Vec<DetectedFace>, iou_threshold: f32)`
   - `fn iou(a: &[f32; 4], b: &[f32; 4]) -> f32`

   Use `tokio::task::spawn_blocking` in the detect method since ONNX inference is GPU-bound and blocks the thread.
