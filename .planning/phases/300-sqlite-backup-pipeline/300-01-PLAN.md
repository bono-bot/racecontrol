        loop {
            interval.tick().await;

            }
        }
    });
}

    2. Create `backup_pipeline.rs` with these components:

    **pub fn spawn(state: Arc<AppState>)** — follows scheduler.rs pattern exactly:
    - Log "backup_pipeline task started" at INFO
    - If `!state.config.backup.enabled`, log "backup pipeline disabled" and return without spawning
    - On first tick, scan `backup_dir` for most recent backup file to initialize `BackupStatus.last_backup_at` (prevents false staleness alert on startup — Pitfall 5 from RESEARCH.md)
    - tokio interval loop at `config.backup.interval_secs` (default 3600)
    - Each tick calls `backup_tick(&state, &mut last_alert_fired).await`
    - Wrap in `if let Err(e)` with `tracing::error!(target: "backup_pipeline", ...)`

    **async fn backup_tick(state: &Arc<AppState>, last_alert_fired: &mut Option<Instant>) -> anyhow::Result<()>:**
    - Create backup_dir if not exists: `std::fs::create_dir_all`
    - Generate timestamp: `chrono::Utc::now().with_timezone(&chrono_tz::Asia::Kolkata).format("%Y-%m-%dT%H-%M-%S")`
    - Determine if this is a weekly backup: check if IST day-of-week is Sunday (chrono Weekday::Sun)
    - For each DB pool (main `state.db` + telemetry `state.telemetry_db` if Some):
      - Daily backup path: `{backup_dir}/{prefix}-{timestamp}.db` where prefix is "racecontrol" or "telemetry"
      - Run `VACUUM INTO '{path}'` via sqlx::query (the path must use forward slashes even on Windows — SQLite handles this)
      - If Sunday, also create weekly: `{backup_dir}/{prefix}-weekly-{year}-W{week}.db` (copy from daily, or run VACUUM INTO again)
      - Log time taken; warn if > 30 seconds
    - Call `rotate_backups` for each prefix
    - Update `state.backup_status` (write lock, set last_backup_at, size, file, count, staleness)
    - Call `check_staleness` with debounce

    **fn rotate_backups(backup_dir: &str, prefix: &str, daily_retain: usize, weekly_retain: usize) -> anyhow::Result<()>:**
    - List files in backup_dir matching `{prefix}-*.db` (daily) and `{prefix}-weekly-*.db` (weekly)
    - Sort by name (which is chronological due to ISO timestamp format)
    - Delete oldest daily files beyond `daily_retain`
    - Delete oldest weekly files beyond `weekly_retain`
    - Log each deletion at DEBUG, summary at INFO

    **async fn check_staleness(state: &Arc<AppState>, last_alert_fired: &mut Option<Instant>):**
    - Read backup_status for staleness_hours
    - If staleness_hours > config.backup.staleness_alert_hours as f64:
      - Check debounce: if last_alert_fired is None OR elapsed > 2 * 3600 seconds
      - Fire WhatsApp: `whatsapp_alerter::send_whatsapp(&state.config, &msg).await`
      - Message format: `"[BACKUP] No successful backup in {hours:.1} hours -- last at {time} | {ist_now_string()}"`
      - Set `last_alert_fired = Some(Instant::now())`

    **Helper: fn compute_staleness(backup_dir: &str) -> Option<f64>:**
    - Scan directory for newest file by mtime
    - Return hours since that mtime, or None if no files
