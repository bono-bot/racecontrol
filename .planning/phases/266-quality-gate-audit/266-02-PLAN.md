  <files>.planning/phases/266-quality-gate-audit/266-02-TOUCH-VERIFICATION.md</files>
  <action>
    Run the following checks from the repo root and document results in 266-02-TOUCH-VERIFICATION.md under "## Cross-App Consistency Check".

    **1. Status color token consistency**
    ```
    grep -rn "status-ready\|status-occupied\|status-warning\|status-offline\|status-maintenance" web/src/ kiosk/src/
    ```
    Both apps must use the same CSS variable names for status colors (defined in packages/shared-tokens/tokens.css). If web uses `var(--status-ready)` and kiosk uses `var(--color-ready)` — that is a mismatch. Document every name used in each app. If names differ, fix the kiosk references to match web (web is authoritative for shared token names).

    **2. Timer format consistency**
    ```
    grep -rn "MM:SS\|mm:ss\|toFixed\|padStart.*2\|String.*time\|formatTime\|formatDuration" web/src/ kiosk/src/ | grep -v node_modules
    ```
    Verify both apps format session timers as MM:SS (not M:SS, not HH:MM:SS for sub-hour durations). If a format utility exists in web but not kiosk (or vice versa), extract it to shared-tokens or duplicate it explicitly so both are identical. Document the format function location in each app.

    **3. Racing Red accent consistency**
    ```
    grep -rn "#E10600\|var(--rp-red)\|Racing Red\|racing-red" web/src/ kiosk/src/
    grep -rn "#[Ee]10600" web/src/ kiosk/src/
    ```
    Confirm #E10600 is the only red accent value in use. Any other red value is a bug. Document hits.

    **4. Build verification**
    Run from repo root:
    ```
    cd web && npm run build 2>&1 | tail -20
    cd ../kiosk && npm run build 2>&1 | tail -20
    ```
    Both must exit 0 with no TypeScript errors. Record "BUILD PASS" or "BUILD FAIL — [error]" for each.

    **5. Static file smoke test from James machine**
    Run these curl commands (James machine is 192.168.31.27, server is 192.168.31.23):
    ```
    curl -s -o /dev/null -w "%{http_code}" http://192.168.31.23:3200/_next/static/css/app.css
    curl -s -o /dev/null -w "%{http_code}" http://192.168.31.23:3300/kiosk/_next/static/css/app.css
    curl -s -o /dev/null -w "%{http_code}" http://192.168.31.23:3200/leaderboard-display
    ```
    First two must return 200 (static files served). Third must return 200 (leaderboard-display is unauthenticated). Any 404 or 302 is a deployment failure — document and raise as blocker.

    Fix any issues found. Document the final state of each check as PASS or FAIL with evidence.
  </action>
