<objective>
Create a best-effort AC EVO telemetry adapter that reuses the AC1 shared memory struct layout (ACC format), with graceful degradation when EVO data is unavailable or incomplete.

Purpose: AC EVO (Early Access, UE5) likely uses the same `acpmf_physics`/`acpmf_graphics`/`acpmf_static` shared memory as AC1/ACC. This adapter attempts to read telemetry and emit laps, but never crashes or blocks billing if telemetry is absent.

Output: Working `AssettoCorsaEvoAdapter` implementing `SimAdapter`, registered in main.rs, with unit tests covering all three requirements.
</objective>

    Create `crates/rc-agent/src/sims/assetto_corsa_evo.rs` as a NEW file (do NOT modify assetto_corsa.rs).

    The adapter is ~80% copy-paste from assetto_corsa.rs with these key differences:

    1. **Struct:** `AssettoCorsaEvoAdapter` with same fields as AC1 plus warn-once booleans:
       - `warned_no_shm: bool` — set on first connect failure
       - `warned_empty_graphics: bool` — set when graphics struct reads all zeros

    2. **ShmHandle:** Duplicate the 4-line struct + Send/Sync impls (private to this file, same as iRacing/LMU pattern).

    3. **Struct offsets:** Copy physics/graphics offset constants from AC1 unchanged.

    4. **connect():** Try to open all three shared memory maps (`Local\acpmf_physics`, `Local\acpmf_graphics`, `Local\acpmf_static`). Per research Pattern 2:
       - If physics opens: `connected = true`, log info
       - If physics fails: `connected = false`, warn once via `warned_no_shm`, return Ok(()) (NOT Err)
       - Graphics and static are individually optional — failing is fine, just skip those handles
       - Snapshot `completed_laps` on connect to avoid stale-lap-count false positive (Pitfall 4)

    5. **read_telemetry():** If not connected or handles are None, return Ok(None) (NOT Err — Pitfall 5). When physics is available, read speed, RPM, gear, gas, brake into TelemetryFrame. If graphics is available AND non-zero, include lap count. If graphics reads all zeros, warn once (Pattern 4).

    6. **poll_lap_completed():** Zero-guard all lap detection (Pattern 3). Only emit LapData when:
       - `completed_laps > last_lap_count && last_lap_count > 0`
       - `lap_ms > 0` (never emit lap_time_ms = 0)
       - sim_type = SimType::AssettoCorsaEvo (TEL-EVO-03)

    7. **read_is_on_track():** Return `Some(speed > 5.0 || rpm > 500)` from physics data (Pattern 5). If physics handle is None, return None.

    8. **disconnect():** Close all handles (UnmapViewOfFile + CloseHandle), set connected = false, reset warn flags.

    9. **sim_type():** Return `SimType::AssettoCorsaEvo`.

    10. **Non-windows cfg:** All shared memory operations gated with `#[cfg(windows)]`. Non-windows: connect returns Ok with connected=false, read returns Ok(None).

    11. **mod.rs:** Add `pub mod assetto_corsa_evo;` to sims/mod.rs (required for cargo test compilation — same lesson from Phase 85).

    12. **Unit tests:** All tests in `#[cfg(test)] mod tests` block inside assetto_corsa_evo.rs. Tests operate on adapter state without needing actual shared memory (test connect on non-Windows returns Ok, test struct field defaults, etc.).
