**Public entry point:**

```rust
pub async fn run(state: &AppState, ffb: &dyn FfbBackend) -> PreFlightResult
```

Implementation:
- Wrap all checks in `timeout(Duration::from_secs(5), async { tokio::join!(...) })`.
- Each individual check function internally uses `timeout(Duration::from_secs(2), ...)` for its own budget.
- If the 5s hard timeout fires, return `MaintenanceRequired` with a "pre-flight timeout" failure.
- After collecting results, filter for `CheckStatus::Fail` results.
- For each Fail: call the corresponding auto-fix function (if one exists), then re-run that single check.
- If any check is still Fail after auto-fix: return `MaintenanceRequired { failures }`.
- If all Pass or Warn: return `Pass`.
- Log results via `tracing::info!` for Pass, `tracing::warn!` for MaintenanceRequired.

**Check functions (all private):**

a) `async fn check_hid(ffb: &dyn FfbBackend) -> CheckResult`
- Call `ffb.zero_force()`.
- `Ok(true)` -> Pass, detail "Wheelbase HID connected"
- `Ok(false)` -> Fail, detail "Wheelbase HID not detected (VID:0x1209 PID:0xFFB0)"
- `Err(e)` -> Fail, detail format!("Wheelbase HID error: {}", e)
- No auto-fix possible (hardware). No spawn_blocking needed — zero_force is a quick HID write.

b) `async fn check_conspit() -> CheckResult`
- Wrap in `spawn_blocking` (CRITICAL — sysinfo blocks 100-300ms):
  ```rust
  spawn_blocking(|| {
      let mut sys = System::new();
      sys.refresh_processes(ProcessesToUpdate::All, true);
      let found = sys.processes().values().any(|p| {
          p.name().to_string_lossy().eq_ignore_ascii_case("ConspitLink.exe")
      });
      if !found {
          return CheckResult { name: "conspit_link", status: CheckStatus::Fail, detail: "ConspitLink.exe not running".into() };
      }
      // Stage 2: check config file
      let config_path = std::path::Path::new(r"C:\ConspitLink\config.json");
      if !config_path.exists() {
          return CheckResult { name: "conspit_link", status: CheckStatus::Warn, detail: "ConspitLink running but config.json missing".into() };
      }
      match std::fs::read_to_string(config_path) {
          Ok(content) => {
              if serde_json::from_str::<serde_json::Value>(&content).is_ok() {
                  CheckResult { name: "conspit_link", status: CheckStatus::Pass, detail: "ConspitLink running, config valid".into() }
              } else {
                  CheckResult { name: "conspit_link", status: CheckStatus::Warn, detail: "ConspitLink running but config.json is invalid JSON".into() }
              }
          }
          Err(e) => CheckResult { name: "conspit_link", status: CheckStatus::Warn, detail: format!("ConspitLink running but config.json unreadable: {}", e) },
      }
  }).await.unwrap_or_else(|e| CheckResult { name: "conspit_link", status: CheckStatus::Fail, detail: format!("spawn_blocking panicked: {}", e) })
  ```

c) `async fn check_orphan_game(state: &AppState) -> CheckResult`
- Read `state.game_process` — need to handle this carefully. AppState has `game_process: Option<GameProcess>` as a pub(crate) field. Since `run()` takes `&AppState`, we can read it.
- Check: if `state.game_process.is_some()` AND `state.heartbeat_status.billing_active.load(Ordering::Relaxed) == false`:
  - Extract PID: `state.game_process.as_ref().and_then(|gp| gp.pid)`
  - If PID exists: spawn_blocking to run `taskkill /F /PID {pid}` via `std::process::Command::new("taskkill").args(["/F", "/PID", &pid.to_string()]).output()`
  - Return Fail with detail "Orphaned game process (PID {pid}) killed" — this IS the auto-fix (kill IS the fix). After kill, return Pass since the fix is complete.
  - If no PID: return Warn with detail "Orphaned game_process record but no PID — clearing state"
- If game_process is None OR billing_active is true: return Pass.
- IMPORTANT: SYS-01 auto-fix IS the kill itself. Unlike ConspitLink where fix = restart, here fix = kill. So on success, return Pass immediately.

**Auto-fix function:**

```rust
async fn fix_conspit() -> bool
```
- `spawn_blocking` wrapper:
  - Use `hidden_cmd` pattern (replicate the 5-line function locally since game_process::hidden_cmd is private):
    ```rust
    let mut cmd = std::process::Command::new(r"C:\ConspitLink\ConspitLink.exe");
    #[cfg(windows)]
    {
        use std::os::windows::process::CommandExt;
        cmd.creation_flags(0x08000000); // CREATE_NO_WINDOW

  - After spawn, `std::thread::sleep(Duration::from_secs(2))` inside spawn_blocking.
  - Re-scan process list for "ConspitLink.exe".
  - Return true if found, false if not.
- Wrap in `timeout(Duration::from_secs(3), ...)` per CONTEXT.md auto-fix timeout.

**Auto-fix loop in run():**
After concurrent checks complete, for each Fail result:
- If name == "conspit_link": call `fix_conspit()`, if true -> re-run `check_conspit()` and replace result.
- If name == "hid": no fix available, keep Fail.
- Orphan game: the kill in check_orphan_game IS the fix, so it either returns Pass (killed successfully) or Fail (could not kill). No separate fix function needed.

CoreToAgentMessage::BillingStarted {
    billing_session_id, driver_name, allocated_seconds, ..
} => {
    tracing::info!("Billing started: {} for {} ({}s)", billing_session_id, driver_name, allocated_seconds);

    // Pre-flight gate (PF-01): check hardware before starting session
    if state.config.preflight.enabled {
        let ffb_ref: &dyn crate::ffb_controller::FfbBackend = state.ffb.as_ref();
        match pre_flight::run(state, ffb_ref).await {
            pre_flight::PreFlightResult::Pass => {
                tracing::info!("Pre-flight passed, proceeding with session");
            }
            pre_flight::PreFlightResult::MaintenanceRequired { failures } => {
                tracing::warn!("Pre-flight FAILED: {:?}", failures);
                let failure_strings: Vec<String> = failures.iter().map(|f| f.detail.clone()).collect();
                let pod_id = state.config.pod.number.to_string();
                let msg = rc_common::protocol::AgentMessage::PreFlightFailed {
                    pod_id,
                    failures: failure_strings,
                    timestamp: chrono::Utc::now().to_rfc3339(),
                };
                if let Ok(json) = serde_json::to_string(&msg) {
                    let _ = ws_tx.send(tokio_tungstenite::tungstenite::Message::Text(json.into())).await;
                }
                // Do NOT set billing_active, do NOT show active session
                // Phase 98 will add MaintenanceRequired lock screen state here
                return Ok(HandleResult::Continue);
            }
        }
    }

    // --- All code below only runs on Pass (or preflight disabled) ---
    state.heartbeat_status.billing_active.store(true, std::sync::atomic::Ordering::Relaxed);
    conn.blank_timer_armed = false;
    let billing_session_id_clone = billing_session_id.clone();
    let _ = state.failure_monitor_tx.send_modify(|s| {
        s.billing_active = true;
        s.active_billing_session_id = Some(billing_session_id_clone);
    });
    conn.current_driver_name = Some(driver_name.clone());
    conn.session_max_speed_kmh = 0.0;
    conn.session_race_position = None;
    if allocated_seconds == 0 || allocated_seconds >= 10800 {
        state.overlay.activate_v2(driver_name.clone());
    } else {
        state.overlay.activate(driver_name.clone(), allocated_seconds);
    }
    state.lock_screen.show_active_session(driver_name, allocated_seconds, allocated_seconds);
    tokio::task::spawn_blocking(|| ac_launcher::minimize_background_windows());
}
```
