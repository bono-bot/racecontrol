## Typography

Source: observed across pricing/page.tsx and KioskPodCard.tsx. Exact values used in production.

Two weights only: 400 (body, sub-heading) and 700 (heading, label). Weight differentiates headings and small-size labels from body text.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Table rows, form field values, pod card data |
| Label | 12px (text-xs) | 700 (bold) | 1.4 | Column headers (uppercase tracking-wider), form labels, state badges — bold compensates for small size |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 | Page title ("Pricing Tiers", "Per-Minute Rates") |
| Sub-heading | 14px (text-sm) | 400 (normal) | 1.4 | Section headings within cards ("Add New Tier", "Add Rate") — differentiated by context and placement, not weight |

Font: Montserrat for all text. No Enthocentric in these UIs (Enthocentric is reserved for marketing/branding headers, not operational dashboards).

## Component Inventory

Existing components this phase extends (do NOT redesign — extend only):

### 1. KioskPodCard — `kiosk/src/components/KioskPodCard.tsx`

**Changes required:**
- `derivePodState()`: Add `"loading"` branch — when `gameInfo?.game_state === "loading"` and billing session exists, return new KioskPodState `"loading"`
- `StateLabel`: Add `"loading"` key to both `styles` and `labels` maps
  - `styles.loading`: `"text-amber-400 bg-amber-400/10"`
  - `labels.loading`: `"Loading..."` (generic fallback) or `"Loading {game}..."` if sim_type available on gameInfo
- Loading timer: count-up display below the badge showing elapsed seconds since `game_state` became `"loading"`. Format: `M:SS` (same `formatTime()` function). Color: `text-amber-400`. Font: `font-mono`. Padding: `px-2 py-1` (4px vertical — on-grid).
- `KioskPodState` TypeScript union: add `"loading"` literal

**Loading timer behavior:**
- Start counting when game_state transitions to `"loading"`
- Count up (not down) — staff needs to see how long loading has taken, not a countdown
- Reset to zero when game_state transitions to `"running"` (billing started)
- Timer persists across WebSocket reconnects — use `gameInfo?.loading_started_at` timestamp if provided by server, otherwise estimate from first `"loading"` state received

### 2. Billing Pricing Page — `web/src/app/billing/pricing/page.tsx`

**Changes required:**
- Per-Minute Rates table: add "Game" column between "Threshold" and "Rate" columns
- "Game" column value: display `sim_type` value as a human-readable label, or "All games" when `sim_type` is null
- Inline edit mode for a rate row: replace "Game" cell with a `<select>` dropdown
- Dropdown options: `All games` (value: `""` / null), `F1 25` (value: `"f1_25"`), `iRacing` (value: `"iracing"`), `LMU` (value: `"le_mans_ultimate"`), `AC EVO` (value: `"assetto_corsa_evo"`), `EA WRC` (value: `"ea_wrc"`)
- Add Rate form: add "Game" field as `<select>` with same options, defaulting to "All games"
- Display label mapping: `null` → `"All games"`, `"f1_25"` → `"F1 25"`, `"iracing"` → `"iRacing"`, `"le_mans_ultimate"` → `"LMU"`, `"assetto_corsa_evo"` → `"AC EVO"`, `"ea_wrc"` → `"EA WRC"`, `"assetto_corsa"` → `"AC"`

**Focal point — admin billing rates screen:**
Primary focal point: Per-Minute Rates table body — the tier name and rate columns are where the eye lands first. The Game column is secondary context. Column order (Tier → Threshold → Game → Rate) reflects this reading priority: staff identifies the tier, checks the threshold, reads the game filter, then reads the rate.

## Interaction Contracts

### Pod Card — Loading State Transition

```
game_state: "launching" → game_state: "loading" → game_state: "running"
       ↓                           ↓                         ↓
StateLabel: "Launching"    StateLabel: "Loading..."    StateLabel: "On Track"
(text-blue-400)            (text-amber-400)            (text-rp-red)
No timer shown             Timer counts up (M:SS)      Timer hidden, billing countdown shown
```

- Transition is driven by WebSocket `gameInfo.game_state` value — no local state machine needed in the card
- Loading timer starts from zero on each entry to `"loading"` state (or from `loading_started_at` if present)
- When billing session starts (`game_state` becomes `"running"`), the existing countdown timer resumes immediately — no gap or flash

### Admin Billing Rates — Game Column Edit Flow

```
View mode: "F1 25" label (text-sm text-neutral-200)
              ↓ (click Edit button on row)
Edit mode: <select> with current sim_type selected, border-rp-border focus:border-rp-red
              ↓ (click "Save Rate")
Save: PUT /api/v1/billing_rates/:id with { sim_type: "f1_25" | null }
              ↓
View mode: updated label shown
```

- "All games" option maps to `null` in the API payload (not empty string — avoid SQL type mismatch)
- Select uses same CSS class as existing edit inputs: `bg-rp-card border border-rp-border rounded px-2 py-1 text-sm text-neutral-200 focus:outline-none focus:border-rp-red`
- Width: `w-32` (sufficient for longest label "All games")

### Per-Game Rate Lookup Display

When multiple rates exist for the same tier but different games, the table shows one row per rate. Staff can read: "Standard / 30 min / 5 cr/min / F1 25". No grouping or collapsing — flat table rows, same as existing tier table pattern.

## States and Conditions

### KioskPodCard with GameState::Loading

| Condition | Display |
|-----------|---------|
| game_state = "loading", no sim_type | Badge: "Loading..." (amber), timer: M:SS count-up |
| game_state = "loading", sim_type = "f1_25" | Badge: "Loading F1 25..." (amber), timer: M:SS count-up |
| game_state = "loading", sim_type = other | Badge: "Loading {game abbr}..." (amber), timer: M:SS count-up |
| game_state transitions to "running" | Badge immediately changes to "On Track" (rp-red), timer replaced by billing countdown |
| billing session exists + game_state = "loading" | Show driver name in card header (billing was deferred, not started) |
| billing session exists + game_state = "loading" + 5+ min elapsed | Staff alert visible (handled by server — pod card shows warning badge if BillingWarning prop is set) |

### Admin Billing Rates — sim_type Column States

| Condition | Display |
|-----------|---------|
| rate.sim_type = null | "All games" in neutral text (text-neutral-400) |
| rate.sim_type = valid sim key | Mapped display name (text-neutral-200) |
| rate.sim_type = unknown value | Raw value string in text-amber-400 (migration guard) |
| Row in edit mode | `<select>` with current value pre-selected |
| Save in progress | "Saving..." text on Save Rate button, input disabled |
| Save failed | "Save failed." inline below row in text-red-400 text-xs |
