```rust
struct CircuitBreaker {
    consecutive_failures: u32,

impl CircuitBreaker {
    fn new() -> Self { Self { consecutive_failures: 0, last_failure: None } }
    fn is_open(&self) -> bool {
        if self.consecutive_failures < CIRCUIT_BREAKER_THRESHOLD { return false; }
        match self.last_failure {
            Some(t) => t.elapsed().as_secs() < CIRCUIT_BREAKER_COOLDOWN_SECS,
            None => false,
        }
    }
}
// CIRCUIT_BREAKER_THRESHOLD = 3 (consecutive), COOLDOWN = 300s (5 min)
```

```
pub struct BlastRadiusLimiter {
    active_fixes: DashMap<String, ActiveFix>,  // node_id -> ActiveFix
    max_concurrent: usize,  // default 2
}

pub struct ActiveFix {
    pub node_id: String,
    pub started_at: chrono::DateTime<chrono::Utc>,
    pub action: String,
}

pub struct FixGuard {
    node_id: String,
    limiter: Arc<BlastRadiusLimiter>,
}
```

- `BlastRadiusLimiter::new(max_concurrent: usize)` — constructor
- `BlastRadiusLimiter::try_acquire(&self, node_id: &str, action: &str) -> Option<FixGuard>` — returns None if max_concurrent reached OR this node already has an active fix. Inserts into DashMap on success.
- `BlastRadiusLimiter::active_count(&self) -> usize` — for monitoring
- `BlastRadiusLimiter::is_node_under_fix(&self, node_id: &str) -> bool`
- `FixGuard` implements `Drop` — removes the node_id from DashMap on drop (RAII cleanup). This is critical: if the fix task panics, the slot is still released.
- Add `dashmap` to rc-common's Cargo.toml dependencies (already in workspace deps).

**2. PerActionCircuitBreaker** (replaces existing consecutive-only breaker):
```
pub struct PerActionCircuitBreaker {
    actions: DashMap<String, ActionStats>,
    fail_rate_threshold: f64,    // 0.40 (40%)
    cooldown: Duration,           // 2 minutes
    min_samples: u32,             // 5 (don't trip on 1/2 failures)
}

struct ActionStats {
    successes: u32,
    failures: u32,
    last_failure: Option<Instant>,
    tripped_at: Option<Instant>,
}
```

- `PerActionCircuitBreaker::new(fail_rate_threshold: f64, cooldown: Duration, min_samples: u32)` — constructor
- `is_open(&self, action_key: &str) -> bool` — true if tripped and within cooldown
- `record_success(&self, action_key: &str)` — increment success, reset trip if was tripped
- `record_failure(&self, action_key: &str)` — increment failure, trip if fail_rate >= threshold AND total >= min_samples
- Action key format: `"{tier}:{action_type}"` (e.g., "1:kill_process", "2:kb_apply", "3:openrouter")

**3. IdempotencyTracker**:
```
pub struct IdempotencyTracker {
    seen: DashMap<String, Instant>,
    ttl: Duration,  // 30 minutes default
}
```

- `IdempotencyTracker::new(ttl: Duration)` — constructor
- `check_and_mark(&self, key: &str) -> bool` — returns true if this key has NOT been seen within TTL (i.e., action should proceed). Returns false if duplicate. Inserts/updates timestamp.
- `make_key(node_id: &str, rule_version: &str, incident_fingerprint: &str) -> String` — static helper, format: `"{node_id}:{rule_version}:{incident_fingerprint}"`
- `cleanup(&self)` — remove entries older than TTL. Called periodically by tier engine.
- Key format per research consensus: `node_id + rule_version + incident_fingerprint`

Add `pub mod safety;` to `crates/rc-common/src/lib.rs`.
