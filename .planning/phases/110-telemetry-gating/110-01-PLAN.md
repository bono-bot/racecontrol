<objective>
Gate shared memory telemetry adapters behind a 5-second deferred connect after game reaches Running state, and feature-flag AC EVO telemetry off by default.

Purpose: Anti-cheat kernel drivers (EAC, EOS, Javelin) initialize during game startup and scan for memory access patterns. Deferring shared memory connect by 5 seconds after the game is stable avoids triggering these scans. AC EVO's anti-cheat status is unknown, so its telemetry is disabled by default.

Output: Modified event loop with deferred connect logic, AC EVO feature flag in config, updated adapter creation path.
</objective>

In the telemetry tick, before calling `adapter.connect()`, check if the adapter uses shared memory (iRacing, LMU, AC EVO) and if `game_running_since` is less than 5 seconds ago. 1. **Find the state struct** used in the event loop (likely `ConnectionState` or `AppState` -- check where `state.adapter` lives). Add field:
   ```rust

   pub game_running_since: Option<std::time::Instant>,
   /// Whether the deferred-connect log has been emitted (avoid spam).
   pub shm_defer_logged: bool,
   ```

2. **Set `game_running_since`** at the point where game state transitions to `Running`. Search for where `GameState::Running` is set or where PlayableSignal is handled. Set `state.game_running_since = Some(Instant::now())` there. Reset to None when game exits (GameState::Idle/Stopping/Error).

3. **Gate `adapter.connect()` in the telemetry tick** (event_loop.rs ~line 165). Before the existing `if !adapter.is_connected()` block, add logic:
   ```rust
   // HARD-03: Defer shared memory connect by 5s after game reaches Running
   let is_shm_adapter = matches!(
       adapter.sim_type(),
       SimType::IRacing | SimType::LeMansUltimate | SimType::AssettoCorsaEvo | SimType::AssettoCorsaRally
   );
   if is_shm_adapter && !adapter.is_connected() {
       let defer_ok = match state.game_running_since {
           Some(since) if since.elapsed() >= Duration::from_secs(5) => true,
           Some(since) => {
               if !state.shm_defer_logged {
                   tracing::info!(
                       target: LOG_TARGET,
                       elapsed_ms = since.elapsed().as_millis() as u64,
                       "shm_connect deferred — waiting for anti-cheat init window"
                   );
                   state.shm_defer_logged = true;
               }
               false
           }
           None => false, // Game not running yet, don't connect
       };
       if !defer_ok {
           continue; // Skip this telemetry tick -- don't call connect()
       }
   }
   ```
   Keep the original connect logic intact for non-shm adapters (F1 25 UDP, AC original UDP).

4. **Reset shm_defer_logged** to false when game exits (same place game_running_since is reset).

5. **Do NOT modify the adapter connect() methods themselves.** The gating happens in the event loop, not inside the adapters. The adapters' connect() still calls OpenFileMappingW/MapViewOfFile directly -- the delay is enforced by the caller.
