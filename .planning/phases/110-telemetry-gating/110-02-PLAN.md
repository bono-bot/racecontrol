<objective>
Ensure UDP telemetry sockets are created only when their corresponding game is active and destroyed on game exit.

Purpose: Clean socket lifecycle prevents orphaned UDP bindings that could be flagged by anti-cheat systems monitoring network activity during post-game scans. The F1 25 adapter on port 20777 is the primary target. iRacing port 6789 is read via shared memory (not a separate UDP socket in rc-agent), so it is covered by Plan 01's deferred connect.

Output: UDP adapter lifecycle tied to game Running state, clean disconnect on game exit.
</objective>

impl SimAdapter for F125Adapter {
    fn connect(&mut self) -> Result<()> {
        let socket = UdpSocket::bind("0.0.0.0:20777")?;
        socket.set_nonblocking(true)?;
        self.socket = Some(socket);
        self.connected = true;
        Ok(())
    }

        self.socket = None;  // Drops the UdpSocket, releasing port 20777
        self.connected = false;
    }
}

**Goal:** F1 25 UDP socket exists ONLY while the game is in Running state. The socket is created when the adapter's `connect()` is called (already gated by telemetry tick), and destroyed when the game exits.

**Analysis of current behavior:**
- The telemetry tick calls `adapter.connect()` whenever `!adapter.is_connected()` -- this means the socket gets bound as soon as an F125Adapter exists, even before the game is in Running state.
- Game exit paths already call `adapter.disconnect()` in event_loop.rs and ws_handler.rs.
- The missing piece: gating connect() to only happen when game is Running.

**Implementation:**

1. **In the event loop telemetry tick** (event_loop.rs ~line 165), add a check for UDP adapters similar to the shm check in Plan 01. Before calling `adapter.connect()` for a UDP adapter, verify the game is in Running state:
   ```rust
   // HARD-04: UDP sockets only while game is Running
   let is_udp_adapter = matches!(adapter.sim_type(), SimType::F125);
   if is_udp_adapter && !adapter.is_connected() {
       // Only bind UDP socket when game is confirmed Running
       let game_running = state.game_process.as_mut()
           .map(|gp| gp.state == GameState::Running)
           .unwrap_or(false);
       if !game_running {
           continue; // Don't bind socket until game is active
       }
   }
   ```
   Note: The exact field access for game state may differ -- check how `state.game_process` or equivalent stores the current GameState. Adapt the check accordingly.

2. **Verify all game exit paths call adapter.disconnect().** Search for ALL places where game state transitions to Idle/Stopping/Error and confirm `adapter.disconnect()` is called. The key paths to verify:
   - event_loop.rs: game_check_interval detecting process exit
   - event_loop.rs: StopGame command handling
   - ws_handler.rs: StopGame and EndSession handlers

   If any path is missing disconnect(), add it. The adapter's `disconnect()` already drops the socket (`self.socket = None`), which releases the UDP port immediately.

3. **Add a log message** when the UDP socket is destroyed on game exit:
   In the F1 25 adapter's `disconnect()` method, the log already exists (`"F1 25 adapter disconnected"`). Enhance it to include port info:
