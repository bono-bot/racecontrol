<objective>
Implement server-side persistence for pod game inventory and combo validation results:

1. Create pod_game_inventory and combo_validation_flags DB tables via SQLite migrations
2. Create game_inventory.rs module with handlers for GameInventoryUpdate and ComboValidationReport WS messages
3. Add fleet_validity aggregation — compute valid/partial/invalid for each preset based on combo_validation_flags
4. Add auto-disable logic: when all pods report a preset as Invalid, set enabled=false in game_presets and send WhatsApp alert
5. Wire the two new AgentMessage arms in ws/mod.rs
6. Extend GET /api/v1/presets to include fleet_validity field per preset

Purpose: After Phase 316, pods send GameInventoryUpdate and ComboValidationReport on boot. Without this plan, the server silently drops these messages. This plan makes the server persist and act on them.

**2. upsert_combo_validation_flags(db: &SqlitePool, pod_id: &str, results: &[ComboValidationResult]) -> Result<(), sqlx::Error>**

  ```sql

  ```
- Bind: pod_id, preset_id, status as serialized string (format!("{:?}", result.status)), failure_reasons as JSON (serde_json::to_string_lossy), validated_at
- For failure_reasons: `serde_json::to_string(&result.failure_reasons).unwrap_or_else(|_| "[]".to_string())`
- Log INFO: "Upserted {} combo validation flags for pod {}", results.len(), pod_id

**3. compute_fleet_validity(db: &SqlitePool, preset_id: &str) -> Result<String, sqlx::Error>**
- Count total known pods and valid pods from combo_validation_flags:
  ```sql
  SELECT
    COUNT(*) as total,
    SUM(CASE WHEN status = 'Available' THEN 1 ELSE 0 END) as valid_count
  FROM combo_validation_flags WHERE preset_id = ?
  ```
- If total == 0: return "unknown"
- If valid_count == total: return "valid"
- If valid_count == 0: return "invalid"
- Else: return "partial"

**4. auto_disable_invalid_presets(db: &SqlitePool, config: &Config, preset_id: &str, preset_name: &str) -> Result<bool, sqlx::Error>**
where Config comes from `use crate::config::Config;`
- Call compute_fleet_validity for this preset_id
- If result == "invalid":
  - UPDATE game_presets SET enabled = 0 WHERE id = ? AND enabled = 1
  - If rows_affected > 0 (preset WAS enabled, now disabled):
    - Get the first failure reason: query combo_validation_flags for this preset WHERE status != 'Available' LIMIT 1
    - Log ERROR: "AUTO-DISABLED preset '{}' ({}): invalid on all pods. First failure: {}", preset_name, preset_id, reason
    - Return Ok(true) — caller spawns WhatsApp alert
  - If rows_affected == 0: preset already disabled, no alert needed, return Ok(false)
- If result != "invalid": return Ok(false)

**5. handle_game_inventory_update(state: &AppState, inventory: GameInventory)**
