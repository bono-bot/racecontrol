<objective>
Add a launch timeout watchdog that prevents permanent pod lockout by emitting DiagnosticTrigger::GameLaunchTimeout to the pod's tier engine when a game stalls in Launching state beyond the dynamic timeout threshold.

Purpose: The server already detects timeouts in check_game_health and transitions GameTracker to Error (GSTATE-01). Phase 318 extends this: server sends LaunchTimedOut to the agent, agent feeds it into the tier engine so Game Doctor runs and attempts recovery. Dynamic timeout is also pushed via AgentConfig.launch_timeout_config for future per-combo tuning.

Output: LaunchTimedOut CoreToAgentMessage variant, GameLaunchTimeout DiagnosticTrigger variant, launch_timeout_config field on AgentConfig, WS handler wiring in rc-agent, server emission in check_game_health.
</objective>

```rust
// LAUNCH-01: Notify agent so it can feed GameLaunchTimeout into its tier engine.
// No-lock-across-.await: snapshot the sender before sending.
let launch_timed_out_msg = rc_common::protocol::CoreToAgentMessage::LaunchTimedOut {
    sim_type,
    elapsed_secs: timeout_secs as u64,
};
let wrapped = rc_common::protocol::CoreMessage::wrap(
    rc_common::protocol::CoreToAgentMessage::LaunchTimedOut {
        sim_type,
        elapsed_secs: timeout_secs as u64,
    }
);
{
    let senders = state.agent_senders.read().await;
    if let Some(tx) = senders.get(&pod_id) {
        if let Err(e) = tx.send(wrapped).await {
            tracing::warn!("LAUNCH-01: Failed to send LaunchTimedOut to pod {}: {}", pod_id, e);
        }
    }
}
```

IMPORTANT per standing rules: do NOT hold the `senders` read lock across `.await`. The `{ let senders = ...; tx.send(...).await; }` block is wrong because `tx.send().await` is inside the lock scope. Instead:
```rust
// Snapshot the sender (clone) then drop the lock BEFORE awaiting.
let sender_opt = {
    let senders = state.agent_senders.read().await;
    senders.get(&pod_id).cloned()
};
if let Some(tx) = sender_opt {
    let msg = rc_common::protocol::CoreMessage::wrap(
        rc_common::protocol::CoreToAgentMessage::LaunchTimedOut {
            sim_type,
            elapsed_secs: timeout_secs as u64,
        }
    );
    if let Err(e) = tx.send(msg).await {
        tracing::warn!("LAUNCH-01: Failed to send LaunchTimedOut to pod {}: {}", pod_id, e);
    }
}
```
