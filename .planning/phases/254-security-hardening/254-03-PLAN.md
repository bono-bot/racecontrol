    **SEC-07: WSS TLS support for agent WebSocket connections.**

    The agents currently validate `wss://` URLs in config but the `connect_async` call has no TLS connector — a `wss://` URL would fail at runtime.

    1. **Cargo.toml:** Add `tokio-tungstenite` feature `native-tls` (or `rustls-tls-native-roots` if rustls is preferred — check what's already in the dependency tree). For LAN self-signed certs, `native-tls` is simpler because it can use the Windows certificate store.

    2. **config.rs:** Add optional TLS config fields to the `[core]` section:
       ```rust
       /// Path to custom CA cert for self-signed server certificates (optional).
       /// When set, agent accepts this CA in addition to system roots.
       /// For LAN wss:// with self-signed certs.
       #[serde(default)]
       pub tls_ca_cert_path: Option<String>,

       /// Skip TLS certificate verification (DANGEROUS — for LAN testing only).
       /// Default: false. Only set true in racecontrol.toml for venue LAN pods.
       #[serde(default)]
       pub tls_skip_verify: bool,
       ```

    3. **main.rs:** In the WS connection function, check if the URL starts with `wss://`:
       - If `wss://`: build a TLS connector using `native_tls::TlsConnector` (or rustls)
         - If `tls_ca_cert_path` is set, add the custom CA cert
         - If `tls_skip_verify` is true, disable cert verification (with WARN log)
         - Use `tokio_tungstenite::connect_async_tls_with_config()` with the connector
       - If `ws://`: use existing `connect_async()` unchanged (no regression)

    4. **Backward compatibility:** All existing `ws://` configs work unchanged. No behavioral change until someone edits the agent TOML to use `wss://`. This is configuration-level preparation per SEC-07 context note "WSS requires TLS certs. For LAN, self-signed is fine. Configure in racecontrol.toml."

    5. **Do NOT generate self-signed certs in this plan** — that's an operational step. The code must be READY for WSS; actual cert generation and config rollout is a deploy-time activity.

    6. Add a compile-time test that verifies the TLS connector builds without error (feature gate test).
