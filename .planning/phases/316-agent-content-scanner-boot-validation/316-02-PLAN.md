<objective>
Add boot-time AC preset validation to rc-agent: when the server sends PresetPush, rc-agent validates each AC preset against the pod filesystem (car dir, track dir, AI lines) and sends ComboValidationReport back. The validation is gated on receiving presets — it never runs against an empty list. Also wires the validation into the WS reconnect path.

Purpose: The server and kiosk currently have no way to know which AC presets are actually launchable on each pod. Boot validation sends this data proactively before any customer session starts, enabling Phase 317 to aggregate fleet availability and Phase 320 to filter the kiosk.

Output: validate_ac_combo and validate_ac_combos functions in content_scanner.rs, plus a PresetPush handler branch in ws_handler.rs that triggers validation and sends ComboValidationReport.
</objective>

```rust
/// Find the Assetto Corsa install root directory.
/// Prefers the inventory-discovered path (from scan_steam_library results),
/// falls back to the hardcoded default path.
///
/// Returns the path to the AC root (the directory containing content/, system/, etc.)
/// NOT the content/ subdirectory.
pub fn find_ac_base_path() -> Option<std::path::PathBuf> {
    // Primary: check the default Steam path
    let default = std::path::Path::new(
        r"C:\Program Files (x86)\Steam\steamapps\common\assettocorsa"
    );
    if default.is_dir() {
        return Some(default.to_path_buf());
    }
    // Fallback: check non-default Steam libraries
    let alt_roots = [
        r"D:\Steam\steamapps\common\assettocorsa",
        r"E:\Steam\steamapps\common\assettocorsa",
        r"D:\SteamLibrary\steamapps\common\assettocorsa",
        r"E:\SteamLibrary\steamapps\common\assettocorsa",
    ];
    for p in &alt_roots {
        if std::path::Path::new(p).is_dir() {
            return Some(std::path::Path::new(p).to_path_buf());
        }
    }
    None
}
```

**validate_ac_combo(pod_id: &str, preset: &GamePreset, ac_base: Option<&Path>) -> ComboValidationResult**

Algorithm:
1. If `preset.game != "assettoCorsa"` — this function should not be called for non-AC presets, but if it is, return `ComboValidationResult { status: ComboAvailabilityStatus::Unknown, failure_reasons: vec!["not_ac_preset".to_string()], ... }`.
2. If `ac_base` is None OR the path does not exist — return `ComboValidationResult { status: ComboAvailabilityStatus::GameNotInstalled, failure_reasons: vec!["AC not installed".to_string()], checked_paths: vec![ac_base.map(|p| p.display().to_string()).unwrap_or_default()], ... }`.
3. Build `checked_paths: Vec<String>` as you check each path.
4. Check car: if `preset.car.is_some()`, check `{ac_base}/content/cars/{car}` is a dir. Push the path to `checked_paths`. If missing, push `format!("car '{}' not found", car)` to `failure_reasons`.
5. Check track: if `preset.track.is_some()`, check `{ac_base}/content/tracks/{track}` is a dir. If missing, push `format!("track '{}' not found", track)` to `failure_reasons`.
6. Check AI lines: if track dir exists, check `{ac_base}/content/tracks/{track}/ai/` has at least one file (reuse the existing `check_has_ai` logic — the track config is `""` for default layout). If dir exists but empty (or missing), push `"ai lines missing for track '{track}'"` to `failure_reasons`.
7. If `failure_reasons.is_empty()`: `status = Available`. Else: `status = Invalid`.
8. Return `ComboValidationResult { preset_id: preset.id.clone(), preset_name: preset.name.clone(), pod_id: pod_id.to_string(), status, failure_reasons, checked_paths, validated_at: Utc::now().to_rfc3339() }`.

**validate_ac_combos(pod_id: &str, presets: &[GamePresetWithReliability]) -> Vec<ComboValidationResult>**
- Resolve `ac_base = find_ac_base_path()`.
- Filter to AC presets: `preset.preset.game == "assettoCorsa"`.
- For each AC preset, call `validate_ac_combo(pod_id, &p.preset, ac_base.as_deref())`.
- Log INFO: `"Combo validation complete: {}/{} AC presets checked ({} available, {} invalid)"`.
- Return the results vec.

**Handler implementation:**

```rust
CoreToAgentMessage::PresetPush(payload) => {
    tracing::info!(
        target: LOG_TARGET,
        "Presets received: {} preset(s) from server",
        payload.presets.len()
    );

    // COMBO-01 / COMBO-02: Trigger AC combo validation now that we have presets.
    // Run in spawn_blocking — filesystem walk, no async I/O needed.
    // Send result via ws_exec_result_tx so the main WS loop serializes and sends it.
    if !payload.presets.is_empty() {
        let pod_id = state.pod_id.clone();
        let presets = payload.presets.clone();
        let result_tx = state.ws_exec_result_tx.clone();
        tokio::spawn(async move {
            let pod_id_inner = pod_id.clone();
            let results = tokio::task::spawn_blocking(move || {
                crate::content_scanner::validate_ac_combos(&pod_id_inner, &presets)
            })
            .await
            .unwrap_or_default();

            tracing::info!(
                target: LOG_TARGET,
                "Combo validation complete: {}/{} AC combos validated",
                results.iter().filter(|r| matches!(r.status, rc_common::types::ComboAvailabilityStatus::Available)).count(),
                results.len()
            );

            if !results.is_empty() {
                let report = rc_common::protocol::AgentMessage::ComboValidationReport {
                    pod_id: pod_id.clone(),
                    results,
                };
                if result_tx.send(report).await.is_err() {
                    tracing::warn!(
                        target: LOG_TARGET,
                        "Failed to queue ComboValidationReport — ws_exec_result_tx closed"
                    );
                }
            }
        });
    } else {
        tracing::warn!(
            target: LOG_TARGET,
            "PresetPush received with 0 presets — skipping combo validation"
        );
    }
}
```

**Sequencing gate guarantee:** This handler only runs when the server actually sends PresetPush. The validate_ac_combos call is inside the handler, so it is structurally impossible for combo validation to run before presets are received. The log line "Presets received" appears before "Combo validation complete" because the tracing::info at the start of the handler fires before the async spawn completes. This satisfies Success Criterion 4.

**Important notes for executor:**
- `state.ws_exec_result_tx` is `mpsc::Sender<AgentMessage>` — use `.send(msg).await` NOT `.try_send()` to avoid dropping the message if the channel is momentarily full.
- The `unwrap_or_default()` on spawn_blocking join is intentional — if the blocking task panics (extremely unlikely for pure filesystem reads), return empty vec rather than propagating the panic.
- Do NOT add `.unwrap()` anywhere in the new code.
- The handler goes in the `match msg { ... }` block where FlagSync, ConfigPush, FullConfigPush handlers already live. It is a new arm, not a modification to existing arms.
