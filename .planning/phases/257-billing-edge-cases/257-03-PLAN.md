    **BILL-07 — Multiplayer session billing synchronization:**

    The multiplayer billing infrastructure already exists (group_sessions, group_session_members, MultiplayerBillingWait, check_and_stop_multiplayer_server). This task adds the missing piece: synchronized pause/resume across all pods in a group when one pod crashes.

    1. In `billing.rs`, add `pause_multiplayer_group(state, group_session_id, reason)`:
       - Query all `pod_id` values from `group_session_members` where `group_session_id = ?`.
       - For each pod's active billing timer, set status to `PausedGamePause` (with pause_reason = CrashRecovery).
       - Log: `"BILL-07: Pausing all {} pods in multiplayer group {} — reason: {}"`.
       - Broadcast `DashboardEvent::MultiplayerGroupPaused { group_session_id, pod_ids }`.

    2. Add `resume_multiplayer_group(state, group_session_id)`:
       - Query all member pod_ids.
       - For each pod timer in PausedGamePause with CrashRecovery reason, set status back to Active.
       - Log: `"BILL-07: Resuming all pods in multiplayer group {}"`.

    3. In the crash detection handler (where server receives GameStateUpdate::CrashDetected from agent):
       - Check if the crashed pod is part of a multiplayer group (query group_session_members).
       - If yes, call `pause_multiplayer_group()` instead of just pausing the single pod.
       - On crash recovery success (game relaunched), call `resume_multiplayer_group()`.

    4. Add BILL-07 comment markers on all synchronized pause/resume code.

    5. Add a billing_event `"multiplayer_group_paused"` and `"multiplayer_group_resumed"` logged to billing_events table for audit.
  </action>

  <action>
    1. In `db/mod.rs`, add `dispute_requests` table migration:
       ```sql
       CREATE TABLE IF NOT EXISTS dispute_requests (
         id TEXT PRIMARY KEY,
         billing_session_id TEXT NOT NULL,
         driver_id TEXT NOT NULL,
         reason TEXT NOT NULL,
         status TEXT NOT NULL DEFAULT 'pending',
         created_at TEXT NOT NULL DEFAULT (datetime('now')),
         resolved_at TEXT,
         resolved_by TEXT,
         resolution_reason TEXT,
         refund_amount_paise INTEGER
       )
       ```
       Add UNIQUE index: `CREATE UNIQUE INDEX IF NOT EXISTS idx_dispute_session ON dispute_requests(billing_session_id) WHERE status != 'denied'` — one active dispute per session.

    2. In `routes.rs`, add to customer routes:
       **POST /customer/dispute:**
       - Extract driver_id from customer JWT (same pattern as pwa_game_request).
       - Validate billing_session_id exists and belongs to this driver.
       - Validate session status is 'completed' or 'ended_early' (can't dispute active sessions).
       - Insert into dispute_requests with status='pending'.
       - Broadcast `DashboardEvent::DisputeCreated { dispute_id, driver_name, session_id }`.
       - Return `{ ok: true, dispute_id }`.

    3. In `routes.rs`, add to staff/manager routes (RBAC: manager+ per SEC-04):
       **GET /admin/disputes:**
       - Query dispute_requests JOIN billing_sessions JOIN drivers.
       - Return array of disputes with: dispute_id, driver_name, session_id, pod_id, session_duration, amount_charged, reason, status, created_at.
       - Optional query param `?status=pending` to filter.

       **GET /admin/disputes/{id}/details:**
       - Return full dispute context: dispute info + billing_events for that session + billing_session details (start/end time, allocated vs used seconds, tier, cost).
       - This gives staff the full audit trail to make an informed decision.

       **POST /admin/disputes/{id}/resolve:**
       - Body: `{ "action": "approve" | "deny", "reason": "..." }`
       - Validate dispute exists and is pending.
       - If approve:
         - Compute refund amount using `compute_refund()` from billing.rs (the existing unified refund path per FATM-06).
         - Credit the wallet using `credit_in_tx()` (the existing atomic credit per FATM-03).
         - Update dispute_requests: status='approved', resolved_at=now, resolved_by=staff_id, refund_amount_paise.
         - Log billing_event 'dispute_refund' on the session.
       - If deny:
         - Update dispute_requests: status='denied', resolved_at=now, resolved_by=staff_id, resolution_reason.
         - Log billing_event 'dispute_denied' on the session.
       - Return `{ ok: true, status, refund_amount_paise }`.

    4. Add BILL-08 comment markers on all dispute code.
  </action>
