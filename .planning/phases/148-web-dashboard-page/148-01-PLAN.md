- `error: string | null` — fetch error
- `collapsedZones: Record<string, boolean>` — zone collapse state
- `fullscreenCamera: CameraInfo | null` — currently fullscreen camera
- `webrtcStatus: string` — "connecting" | "connected" | "failed"
- `dragOverChannel: number | null` — channel being dragged over
- `statusText: string` — "X/13 online" counter

**Refs (useRef):**
- `pcRef: RTCPeerConnection | null` — active WebRTC peer connection
- `wsRef: WebSocket | null` — active WebRTC signaling socket
- `preWarmPcRef: RTCPeerConnection | null` — pre-warmed connection
- `preWarmWsRef: WebSocket | null` — pre-warmed signaling socket
- `preWarmChannelRef: number | null` — which channel is pre-warmed
- `preWarmTimerRef: NodeJS.Timeout | null` — hover timer
- `controlsHideTimerRef: NodeJS.Timeout | null` — fullscreen controls auto-hide
- `videoRef: HTMLVideoElement` — fullscreen video element
- `refreshTimerRef: NodeJS.Timeout | null` — snapshot polling interval
- `imgRefs: Record<number, HTMLImageElement>` — snapshot img elements by channel
- `dragSrcChannelRef: number | null` — source channel during drag
- `camerasRef: CameraInfo[]` — current cameras for use in intervals (avoids stale closure)

**Feature 1: Camera fetch + layout fetch (useEffect on mount):**
1. Fetch `${SENTRY_BASE}/api/v1/cameras`, sort by display_order, set cameras state
2. Then fetch `${SENTRY_BASE}/api/v1/cameras/layout`, apply grid_mode and reorder cameras by camera_order
3. Then start snapshot polling loop
4. Show loading spinner during fetch, error state with retry on failure

**Feature 2: Snapshot polling loop:**
- Use setInterval at `refreshRate` ms
- For each camera with nvr_channel, preload image via `new Image()` then set imgRef src
- Count successes/failures, update statusText to "X/13 online"
- On rate change, restart the interval
- Cleanup interval on unmount
