Update the three SQL queries inside public_leaderboard to conditionally filter by sim_type:

a) **track_records query** (line ~9352): When sim_type is Some, add `JOIN laps lap_ref ON tr.lap_id = lap_ref.id WHERE lap_ref.sim_type = ?`. When None, keep current behavior (all games). Since track_records now has sim_type column (from Plan 01), use `WHERE tr.sim_type = ?` directly instead of joining laps.

Build the query dynamically:
```rust
let (records_query, records_result) = if let Some(ref st) = params.sim_type {
    let q = "SELECT tr.track, tr.car, CASE WHEN d.show_nickname_on_leaderboard = 1 AND d.nickname IS NOT NULL THEN d.nickname ELSE d.name END, tr.best_lap_ms, tr.achieved_at, tr.lap_id
             FROM track_records tr
             JOIN drivers d ON tr.driver_id = d.id
             WHERE tr.sim_type = ?
             ORDER BY tr.achieved_at DESC";
    (q, sqlx::query_as::<_, (String, String, String, i64, String, Option<String>)>(q).bind(st).fetch_all(&state.db).await)
} else {
    let q = "SELECT tr.track, tr.car, ... ORDER BY tr.achieved_at DESC";
    (q, sqlx::query_as::<_, ...>(q).fetch_all(&state.db).await)
};
```

b) **tracks query** (line ~9362): When sim_type is Some, add `AND sim_type = ?`. When None, show all.

c) **top_drivers query** (line ~9373): When sim_type is Some, add `AND l.sim_type = ?`. When None, show all.

d) Include `"sim_type": params.sim_type` in the response JSON so the client knows what filter was applied.

e) Add an `"available_sim_types"` array to the response by querying: `SELECT DISTINCT sim_type FROM laps WHERE valid = 1 ORDER BY sim_type`. This helps the frontend build a game picker.
