<objective>
Add track name normalization and scope personal_bests/track_records by sim_type so multi-game lap data stays separated correctly.

Purpose: Without sim_type scoping, an F1 25 lap on "Monza" would overwrite an AC personal best on "monza" — they must be independent records. Track name normalization ensures that game-specific track identifiers (e.g. iRacing's "monza combined" or F1 25's "monza_circuit") resolve to the same canonical name used in the catalog.

Output: normalize_track_name() function in catalog.rs, updated persist_lap() with normalization and sim_type-scoped PB/TR logic, migrated DB schema.
</objective>
