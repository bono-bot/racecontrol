  <behavior>
    - Test 1: Empty snapshot returns empty string (no metrics recorded yet)
    - Test 2: Single metric without pod label renders as `racecontrol_cpu_usage 45.2` with HELP and TYPE lines
    - Test 3: Metric with pod label renders as `racecontrol_cpu_usage{pod="pod-3"} 67.8`
    - Test 4: Multiple metrics produce grouped output (all lines for same metric name together, separated by blank line from next metric)
    - Test 5: Metric names are prefixed with `racecontrol_` and use snake_case (pass-through since TSDB already uses snake_case)
  </behavior>
  <action>
    Create `crates/racecontrol/src/api/metrics_prometheus.rs` with:

    1. A `fn format_prometheus(entries: &[SnapshotEntry]) -> String` pure function that:
       - Groups entries by `name`
       - For each unique metric name, emits:
         ```
         # HELP racecontrol_{name} {description}
         # TYPE racecontrol_{name} gauge
         racecontrol_{name}{pod="pod-{N}"} {value}
         ```
       - If pod is None, omit the label block: `racecontrol_{name} {value}`
       - Use a match on metric name for HELP descriptions:
         - cpu_usage -> "CPU usage percentage"
         - gpu_temp -> "GPU temperature celsius"
         - fps -> "Frames per second"
         - billing_revenue -> "Billing revenue paise"
         - ws_connections -> "WebSocket connections"
         - pod_health_score -> "Pod health score 0-100"
         - game_session_count -> "Active game sessions"
         - _ -> "Metric value"
       - Separate metric groups with a blank line
       - Format floats without trailing zeros where possible (use `{:.1}` or similar — Prometheus accepts any float format)

    2. A `pub async fn prometheus_handler(State(state): State<Arc<AppState>>) -> impl IntoResponse` that:
       - Calls `metrics_query::query_snapshot(&state.db, None).await` to get ALL metrics (no pod filter)
       - Calls `format_prometheus(&entries)`
       - Returns the string with content-type `text/plain; version=0.0.4; charset=utf-8` (Prometheus exposition content type)
       - Returns 200 even if empty (valid — just no metrics yet)

    3. Unit tests for `format_prometheus` covering the 5 behaviors above. Use `#[cfg(test)] mod tests`.

    No `.unwrap()` in production code. Use `write!` to a String buffer for formatting.
  </action>
