    Create `crates/rc-mma/src/budget.rs` — a PERSISTENT budget guard for MMA sessions:

    1. `BudgetState` struct (Serialize, Deserialize): `daily_limit: f64`, `spent_today: f64`, `monthly_spent: f64`, `model_calls_today: u32`, `budget_date: String` (YYYY-MM-DD), `last_updated: String` (ISO 8601).

    2. `BudgetGuard` struct:
    - Fields: `state: BudgetState`, `file_path: String`.
    - `BudgetGuard::new(config: &MmaConfig) -> anyhow::Result<Self>`:
      - Reads `budget_state.json` from `config.budget_state_path`. If file exists, deserialize. If not, create with defaults.
      - Calls `maybe_reset()` immediately (handles midnight IST crossing).
    - `BudgetGuard::maybe_reset(&mut self)`:
      - Compute IST today: `Utc::now() + FixedOffset::east_opt(5*3600 + 30*60)`.
      - If `state.budget_date != ist_today_string`, roll `spent_today` into `monthly_spent`, reset daily counters, update `budget_date`.
      - Persist after reset.
    - `BudgetGuard::can_start_session(&mut self, estimated_cost: f64) -> bool`:
      - Calls `maybe_reset()`.
      - Returns false if `spent_today + estimated_cost > daily_limit` (MP-04: cost guard).
      - Returns false if remaining < 2.0 (minimum reserve, matching existing budget_tracker.rs pattern).
      - Logs at WARN level when refusing.
    - `BudgetGuard::record_spend(&mut self, actual_cost: f64) -> anyhow::Result<()>`:
      - Increments `spent_today` and `model_calls_today`.
      - **Immediately persists to file** (MP-04: "daily spend survives process reboot").
      - Logs at INFO level with cost, remaining, calls_today.
    - `BudgetGuard::persist(&self) -> anyhow::Result<()>`:
      - Writes `state` as JSON to `self.file_path`. Uses write-to-temp-then-rename pattern for atomicity (write to `{path}.tmp`, then `std::fs::rename`).
    - `BudgetGuard::status(&mut self) -> BudgetStatus`:
      - Returns a `BudgetStatus` struct (same shape as existing budget_tracker.rs).

    3. The persistence file path defaults to `budget_state.json` in CWD but is configurable via `MmaConfig.budget_state_path`. On pods this will be `C:\RacingPoint\budget_state.json`. On the server, `C:\RacingPoint\mma_budget_state.json`.

    4. CRITICAL per research: budget_state.json must be written on EVERY OpenRouter call, not just at session end. The `record_spend` method handles this.

    5. No SQLite dependency — rc-watchdog has no SQLite. Pure JSON file persistence.
