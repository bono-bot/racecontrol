# Plan 03: Sync Health Observability and End-to-End Verification

## Objective
Enhance the `/sync/health` endpoint with relay connection status and mode awareness, then verify the full relay integration works end-to-end.

## Context
- @crates/rc-core/src/cloud_sync.rs (dual-mode sync with relay support)
- @crates/rc-core/src/action_queue.rs (relay-aware action delivery)
- @crates/rc-core/src/api/routes.rs (sync_health endpoint, actions/process endpoint)
- @/root/comms-link/bono/comms-server.js (relay endpoints)

## Tasks

### Task 1: Enhance sync_health endpoint with relay status
type="auto"

**Done criteria:** `/sync/health` returns `relay_configured`, `relay_available`, `sync_mode` (relay|http|disabled), and `comms_link_url` fields alongside existing sync state info.

**Steps:**
1. Update `sync_health` handler in routes.rs to call `is_relay_available()` from cloud_sync
2. Add `relay_configured`, `relay_available`, `sync_mode`, and `comms_link_url` fields to the JSON response
3. The sync_mode should be: "relay" when relay is configured and available, "http" when relay is configured but unavailable or not configured, "disabled" when cloud sync is off

**Verification:** `cargo check` passes

### Task 2: Verify relay integration end-to-end
type="checkpoint:human-verify"

**Verification steps:**
1. Confirm rc-core is running on port 8080
2. Hit `GET http://localhost:8080/sync/health` and verify the response includes relay status fields
3. Confirm comms-link is running on port 8765 by checking `GET http://localhost:8765/relay/health`
4. Verify sync_mode shows "relay" when comms-link is up
