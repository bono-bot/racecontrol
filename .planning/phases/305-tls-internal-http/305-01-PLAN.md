# Phase 305 Plan 01: TLS for Internal HTTP

## Objective

Establish a self-signed venue Certificate Authority and wire mTLS into both the Axum server (:8080) and rc-agent (:8090). Tailscale connections (100.x.x.x) bypass mTLS since the tunnel already provides end-to-end encryption.

## Context

- `crates/racecontrol/src/tls.rs` â€” basic one-way TLS already exists (self-signed cert for HTTPS port). Needs extension to support CA + mTLS.
- `crates/racecontrol/src/config.rs` â€” `ServerConfig` has `tls_port`, `cert_path`, `key_path`. Needs `[tls]` section for CA/mTLS config.
- `crates/rc-agent/src/remote_ops.rs` â€” plain HTTP on :8090. Needs TLS acceptor.
- `crates/rc-agent/src/config.rs` â€” `AgentConfig` needs `[tls]` section.
- `Cargo.toml` (racecontrol crate) â€” already has `axum-server` + `rcgen`. Good.
- `Cargo.toml` (rc-agent crate) â€” needs `axum-server` + `rcgen` added.
- Tailscale IPs: `100.x.x.x` range â€” all pod and server nodes. Bypass mTLS for these.

## Tasks

### Task 1: CA generation shell script (TLS-01)
**type:** auto

Create `scripts/generate-venue-ca.sh` that uses `openssl` to:
1. Generate a venue root CA (4096-bit RSA, 10yr expiry) at `C:\RacingPoint\tls\ca\`
2. Generate the server cert signed by the CA (for :8080) with SANs: `192.168.31.23`, `localhost`, `racing-point-server`
3. Generate per-pod client certs for pods 1-8 (for mTLS client auth) with CN `pod-N`
4. Output a `bundle.pem` (CA + server cert chain) and `ca.pem` for distribution
5. On Linux (Bono VPS / CI): works identically since `openssl` is available

**Done criteria:**
- Script is executable and produces all cert files in one run
- Script is idempotent (skips existing certs, `--force` to regenerate)
- All certs signed by the same venue CA

### Task 2: Extend server TLS config structure (TLS-02 prep)
**type:** auto

Add `[tls]` section to `ServerConfig` in `crates/racecontrol/src/config.rs`:
```toml
[tls]
enabled = false           # opt-in; false = plain HTTP (backward compat)
ca_cert_path = "C:\\RacingPoint\\tls\\ca.pem"
server_cert_path = "C:\\RacingPoint\\tls\\server.pem"
server_key_path = "C:\\RacingPoint\\tls\\server-key.pem"
require_client_cert = false  # mTLS; false = TLS-only (phase-in safely)
tailscale_bypass = true      # 100.x.x.x connections bypass mTLS check
```

**Done criteria:**
- `TlsConfig` struct added with all fields + `Default` impl
- `ServerConfig` has `#[serde(default)] pub tls: TlsConfig`
- Config loads without error when `[tls]` section absent (backward compat)

### Task 3: Upgrade server tls.rs for CA-signed mTLS (TLS-02)
**type:** auto

Extend `crates/racecontrol/src/tls.rs`:
1. Add `load_mtls_config(tls: &TlsConfig) -> anyhow::Result<RustlsConfig>` that:
   - Loads CA cert + server cert + server key from config paths
   - When `require_client_cert = true`, adds a `ClientCertVerifier` that verifies against the venue CA
   - When `require_client_cert = false`, behaves like existing one-way TLS
2. In `main.rs`: when `config.server.tls.enabled`, use `load_mtls_config()` on the main :8080 listener instead of the existing simple TLS path
3. Tailscale bypass: the `classify_source_middleware` already tags `ConnectionSource` â€” extend it so Tailscale-sourced connections skip the client-cert check entirely (or accept without a client cert)

**Done criteria:**
- Unit test: `load_mtls_config` with a generated CA + cert stack succeeds
- Server starts with `tls.enabled = false` (plain HTTP, no regression)
- With `tls.enabled = true, require_client_cert = false`: accepts connections without client cert
- With `require_client_cert = true`: rejects connections without CA-signed client cert

**type:** auto

1. Add `axum-server` (with `tls-rustls`) and `rcgen` to `crates/rc-agent/Cargo.toml`
2. Add `[tls]` section to `AgentConfig` in `crates/rc-agent/src/config.rs`:

3. Create `crates/rc-agent/src/tls.rs` â€” mirrors server tls.rs but simpler (no mTLS client verification â€” agents don't need to verify caller certs, the service key handles auth)
4. In `remote_ops.rs` `start_with_result()` function: when `tls_config` is `Some(RustlsConfig)`, use `axum_server::bind_rustls()` instead of plain `axum::serve()`

**Done criteria:**
- rc-agent compiles with new deps
- Agent starts with `tls.enabled = false` â€” no behavior change
- Agent starts with `tls.enabled = true` â€” HTTPS on :8090 using venue CA cert

### Task 5: Tailscale bypass logic (TLS-04)
**type:** auto

Both server and agent should skip mTLS requirements for Tailscale connections:

**Server side** (`crates/racecontrol/src/main.rs`):
- The Bono relay listener already binds to `config.bono.tailscale_bind_ip` â€” it gets its own listener without mTLS (plain HTTP)
- For the main listener: extend `classify_source_middleware` to detect `100.x.x.x` source IP and set a `X-Connection-Source: tailscale` header, then skip client cert verification in the TLS acceptor

**Agent side** (`crates/rc-agent/src/remote_ops.rs`):
- Add `is_tailscale_ip(addr: &SocketAddr) -> bool` helper: checks if first octet of IP is 100
- Add `tailscale_bypass_middleware` that sets an extension marking Tailscale connections
- When `tls.tailscale_bypass = true`, Tailscale IPs are served via a second plain-HTTP listener on the same port logic path, or accepted without client cert in the TLS handshake

**Done criteria:**
- `is_tailscale_ip("100.92.122.89")` returns true
- `is_tailscale_ip("192.168.31.89")` returns false
- Unit test covering both cases
