Create scripts/intelligence/ directory and scripts/intelligence/pattern-tracker.sh.

The script is designed to be sourced into auto-detect.sh. It exports one function: update_pattern_db().

SUGGESTIONS_JSONL path: "${REPO_ROOT}/audit/results/suggestions.jsonl"

update_pattern_db() logic:
1. Guard: if RESULT_DIR is unset or findings.json missing, return 0 (degrade gracefully — never block pipeline).
2. Read findings.json array with jq. For each finding object, write one JSONL entry to suggestions.jsonl with these exact fields:
   - run_ts: current run TIMESTAMP (from env)
   - bug_type: finding.issue_type (or finding.category if issue_type empty)
   - pod_ip: finding.pod_ip
   - severity: finding.severity
   - fix_applied: boolean — true if pod_ip appears as host in fixes.jsonl for the same run
   - fix_success: boolean — true if fix_applied AND any entry in fixes.jsonl for same pod has after_state NOT containing "FAIL", "blocked", "skipped", "UNRESOLVED"
   - frequency: 1 (single-run increment; trend-analyzer accumulates this across runs)
   - source: "auto_detect"
3. To determine fix_applied and fix_success: read "${RESULT_DIR}/fixes.jsonl" (JSONL, one JSON object per line). Use jq -s to slurp, filter by .host == pod_ip.
4. Append each entry as one line to SUGGESTIONS_JSONL using >> (not overwrite).
5. Log: log INFO "[LEARN-01] pattern_db updated: N entries written" where N is count of findings processed.
6. If jq exits non-zero at any step, log WARN "[LEARN-01] pattern_db update failed: jq error" and return 0 (never propagate error).

Use set -uo pipefail at top. Source guard: [[ "${_PATTERN_TRACKER_SOURCED:-}" == "1" ]] && return 0; _PATTERN_TRACKER_SOURCED=1.

Export update_pattern_db. No set -e.

Create scripts/intelligence/trend-analyzer.sh. Designed to be sourced. Exports run_trend_analysis().

SUGGESTIONS_JSONL path same as pattern-tracker.sh: "${REPO_ROOT}/audit/results/suggestions.jsonl"
OUTLIER_THRESHOLD configurable: read from auto-detect-config.json field "trend_outlier_multiplier" (default 4.0 if missing).

run_trend_analysis() logic:
1. Guard: SUGGESTIONS_JSONL must exist with at least 10 lines (minimum statistical sample). If not, log INFO "[LEARN-04] trend analysis skipped: insufficient data (N entries)" and return 0.
2. Read all entries from SUGGESTIONS_JSONL using jq -s (slurp JSONL into array).
3. Per-bug-type analysis: for each unique bug_type, count occurrences per pod_ip. Compute fleet_avg = total_count / unique_pod_count.
4. Flag any pod where pod_count > fleet_avg * OUTLIER_THRESHOLD as outlier.
5. For each outlier found, append one JSONL entry to SUGGESTIONS_JSONL with fields:
   - run_ts: current TIMESTAMP
   - entry_type: "TREND_OUTLIER"
   - bug_type: the bug_type analyzed
   - pod_ip: the outlier pod
   - pod_count: count for this pod
   - fleet_avg: computed fleet average (round to 2 decimal places)
   - multiplier: pod_count / fleet_avg (round to 2 decimal places)
   - threshold_used: OUTLIER_THRESHOLD value
   - source: "trend_analyzer"
6. Log INFO "[LEARN-04] trend analysis: N outliers flagged across M bug types"
7. Degrade gracefully: jq failure → log WARN and return 0.

Use set -uo pipefail. Source guard. No set -e.
