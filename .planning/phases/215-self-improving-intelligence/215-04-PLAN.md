<objective>
Build the self-patch loop: the system modifies its own detector and healing scripts, verifies the patch works, and reverts automatically if verification fails.

Purpose: The self-patch loop closes the intelligence cycle. Approved 'new_audit_check' and 'self_patch' proposals mutate the live detection pipeline, test the mutation, and either commit the improvement or safely revert. The self_patch_enabled toggle (LEARN-09) is independent — operators can enable self-patching without enabling infrastructure fixes, or vice versa.

Output: scripts/intelligence/self-patch.sh, self_patch_enabled field in auto-detect-config.json (default false), auto-detect.sh wired to call self_patch_loop.

AUTO_DETECT_CONFIG: "${REPO_ROOT}/audit/results/auto-detect-config.json"
PROPOSALS_DIR: "${REPO_ROOT}/audit/results/proposals"

--- _self_patch_enabled() ---
Mirror the _auto_fix_enabled() pattern exactly, but:
- Field: "self_patch_enabled"
- Default: "false" (NOT "true" — LEARN-09 requires default=false)
- NO_SELFPATCH env var override (returns 1 if set to "true")
```bash
_self_patch_enabled() {
  if [[ "${NO_SELFPATCH:-false}" == "true" ]]; then return 1; fi
  if [[ ! -f "$AUTO_DETECT_CONFIG" ]]; then return 1; fi  # default=false: missing config = disabled
  local val
  val=$(jq -r '.self_patch_enabled // false' "$AUTO_DETECT_CONFIG" 2>/dev/null || echo "false")
  if [[ "$val" == "true" ]]; then return 0; fi
  return 1
}
export -f _self_patch_enabled
```

--- self_patch_loop() ---

1. Toggle gate: _self_patch_enabled || { log INFO "[LEARN-09] self-patch disabled — skipping"; return 0; }

2. Find queued proposals: scan $PROPOSALS_DIR/*.json for status=="queued_for_selfpatch".
   If none found: log INFO "[LEARN-07] self_patch_loop: no queued proposals" and return 0.

3. For each queued proposal (process one per run to limit blast radius):

   a. CE Step 1 — Document symptom: extract bug_type, evidence from proposal. Log to suggestions.jsonl:
      Append entry: {run_ts, entry_type:"SELFPATCH_ATTEMPT", proposal_id, bug_type, pod_ip, status:"started"}

   b. CE Step 2/3 — Identify patch target: find script file in ALLOWED_PATCH_DIRS that mentions bug_type
      (grep -rl "$bug_type" "${ALLOWED_PATCH_DIRS[@]}"). If no match: grep for closest related term.
      SAFETY CHECK: verify target file is under ALLOWED_PATCH_DIRS (use realpath comparison).
      If target is outside allowed dirs: log WARN "[LEARN-08] patch target outside allowed scope — skipping" and
      update proposal status to "patch_scope_rejected". Continue to next proposal.

   c. CE Step 4a — Apply the patch:
      For "threshold_tune" bug_type pattern: find THRESHOLD=N line and increment by 20%.
      For other patterns: apply minimal targeted change — add a comment-log line or adjust an existing variable.
      IMPORTANT: ONLY make changes that are pattern-matched (threshold variables). Do NOT generate new bash code.
      If no patchable pattern found: update proposal status to "no_patch_pattern" and return 0.

      Before modifying: save git stash or record git hash for revert reference:
      ```bash
      local pre_patch_hash
      pre_patch_hash=$(git -C "$REPO_ROOT" rev-parse HEAD)
      ```
      Apply change using sed in-place.

   d. CE Step 4b — Verify the patch:
      Run the modified detector script in dry-run/test mode. The verification method:
      - Check if the detector script accepts a --test or --dry-run flag. If yes, run it.
      - Otherwise: bash -n (syntax check only) as minimum verification.
      - If the proposal is for a threshold change: verify the new threshold value is present in the modified file.
      ```bash
      bash -n "$target_file" 2>&1 && grep -q "THRESHOLD" "$target_file"
      ```

      REVERT if verification fails:
      ```bash
      git -C "$REPO_ROOT" checkout -- "$target_file"
      # Log revert to suggestions.jsonl:
      printf '{"run_ts":"%s","entry_type":"SELFPATCH_REVERTED","proposal_id":"%s","bug_type":"%s","reason":"verification_failed","target_file":"%s"}\n' \
        "$TIMESTAMP" "$proposal_id" "$bug_type" "$target_file" >> "$SUGGESTIONS_JSONL"
      # Update proposal status to "patch_reverted"
      # Notify Bono: "Self-patch REVERTED for ${bug_type}: verification failed after modifying ${target_file}"
      log WARN "[LEARN-08] self-patch reverted: ${proposal_id} — verification failed"
      ```

   e. CE Step 5 — Commit and notify if verification passed:
      ```bash
      git -C "$REPO_ROOT" add "$target_file"
      git -C "$REPO_ROOT" commit -m "selfpatch(215): apply threshold tune for ${bug_type} — proposal ${proposal_id}"
      git -C "$REPO_ROOT" push
      # Update proposal status to "patch_applied"
      # Log to suggestions.jsonl: entry_type:"SELFPATCH_APPLIED"
      # Notify Bono dual-channel: "Self-patch applied: ${bug_type} threshold adjusted in ${target_file}. Proposal: ${proposal_id}"
      log INFO "[LEARN-07] self-patch applied: ${proposal_id}"
      ```

   f. Process only ONE proposal per loop invocation (break after first). This limits risk per run.

set -uo pipefail, source guard, no set -e, all errors return 0.
