Create scripts/intelligence/suggestion-engine.sh. Exports run_suggestion_engine() and get_suggestions_json().

SUGGESTIONS_JSONL: "${REPO_ROOT}/audit/results/suggestions.jsonl"
PROPOSALS_DIR: "${REPO_ROOT}/audit/results/proposals"
MIN_FREQUENCY: read from auto-detect-config.json field "suggestion_min_frequency" (default 3 — only propose fixes for issues seen at least 3 times).

--- run_suggestion_engine() ---

1. Guard: SUGGESTIONS_JSONL must exist. mkdir -p "$PROPOSALS_DIR".
2. Load all non-TREND_OUTLIER entries from SUGGESTIONS_JSONL (jq -R -s to parse JSONL). Filter out entries where entry_type == "TREND_OUTLIER".
3. Aggregate: group by bug_type + pod_ip, count total occurrences, count fix_success=true occurrences.
4. For each (bug_type, pod_ip) pair with total_count >= MIN_FREQUENCY:
   a. Determine category using this mapping:
      - fix_success_rate < 0.5 AND fix_applied_rate > 0.5 → "new_autofix_candidate" (fix is attempted but failing)
      - fix_applied_rate == 0 AND total_count >= MIN_FREQUENCY → "new_autofix_candidate" (never attempted)
      - bug_type matches pattern *drift* OR *mismatch* OR *desync* → "cascade_coverage_gap"
      - bug_type matches pattern *threshold* OR *anomaly* OR *rate* → "threshold_tune"
      - default (repeated unfixed issue) → "standing_rule_gap"
   b. Compute confidence_score: float 0.0-1.0 = min(1.0, total_count / 10.0). Round to 2 decimals.
   c. Build evidence string: "Seen ${total_count}x on pod ${pod_ip}. Fix applied: ${fix_applied_count}x. Fix succeeded: ${fix_success_count}x."
   d. Check if an existing PENDING proposal already covers same bug_type + pod_ip (to avoid duplicates):
      scan $PROPOSALS_DIR/*.json for status=="pending" AND bug_type match. If found, skip.
   e. Write proposal file: $PROPOSALS_DIR/$(date +%s)_${category}_${bug_type//\//_}.json
      Fields: {id (same as filename without .json), category, bug_type, pod_ip, confidence, evidence, status:"pending", created_ts, total_count, fix_success_rate}
5. Also process TREND_OUTLIER entries from SUGGESTIONS_JSONL:
   - For each TREND_OUTLIER entry, if no pending proposal covers it: create a "threshold_tune" proposal with evidence = "Pod ${pod_ip} has ${pod_count} occurrences vs fleet avg ${fleet_avg} (${multiplier}x threshold)."
6. Log INFO "[LEARN-02] suggestion engine: N proposals written, M skipped (existing pending)"

--- get_suggestions_json() ---

Reads all files in $PROPOSALS_DIR/*.json, outputs JSON array sorted by confidence DESC.
Returns empty array [] if no proposals or directory missing.
Usage: relay exec calls this and returns stdout.

Both functions: set -uo pipefail, source guard, export -f, no set -e, all errors return 0 (non-fatal).

```bash
# LEARN-06: relay exec alias — 'get_suggestions' command returns pending proposals as JSON
get_suggestions() {
  if [[ $(type -t get_suggestions_json) == "function" ]]; then
    get_suggestions_json
  else
    # Load suggestion-engine if not already sourced
    local _dir
    _dir="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    source "$_dir/suggestion-engine.sh" 2>/dev/null || { echo '[]'; return 0; }
    get_suggestions_json
  fi
}
export -f get_suggestions
```
