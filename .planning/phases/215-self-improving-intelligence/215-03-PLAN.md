Create scripts/intelligence/approval-sync.sh. Exports approve_suggestion() and apply_approved_suggestion().

PROPOSALS_DIR: "${REPO_ROOT}/audit/results/proposals"
SUPPRESS_JSON: "${REPO_ROOT}/audit/results/suppress.json"
AUTO_DETECT_CONFIG: "${REPO_ROOT}/audit/results/auto-detect-config.json"
STANDING_RULES_REGISTRY: "${REPO_ROOT}/standing-rules-registry.json"
COMMS_LINK_DIR: "${REPO_ROOT}/../comms-link" (use COMMS_LINK_DIR from env if set)

--- approve_suggestion(proposal_id) ---
1. Find proposal file: $PROPOSALS_DIR/${proposal_id}.json (try exact match, also try glob *${proposal_id}*.json).
2. Validate: file must exist, status must be "pending". If status != "pending", log WARN "already ${status}" and return 1.
3. Update status to "approved" in the file: use jq to update .status = "approved", overwrite in place.
4. Call apply_approved_suggestion "$proposal_id".

--- apply_approved_suggestion(proposal_id) ---
1. Load proposal file. Extract: category, bug_type, pod_ip, evidence, confidence.
2. Dispatch on category:

  [threshold_tune]
  - Find matching detector in scripts/detectors/ that handles this bug_type.
  - Look for a line matching pattern: THRESHOLD=N or _THRESHOLD=N or threshold=N (case-insensitive).
  - If found: increment threshold by 20% (round to nearest integer). Use sed to update in place.
  - Also write to auto-detect-config.json: add/update "threshold_overrides.${bug_type}" = new_value.
  - Commit: git add the detector script + auto-detect-config.json.

  [new_autofix_candidate]
  - Append bug_type to auto-detect-config.json "approved_auto_fixes" array (create array if absent).
  - Pattern: jq '. + {"approved_auto_fixes": (.approved_auto_fixes // [] | . + ["'"$bug_type"'"])}' config > tmp && mv tmp config
  - Commit: git add auto-detect-config.json.

  [standing_rule_gap]
  - If standing-rules-registry.json does not exist, create it as an empty JSON array [].
  - Generate next ID: SR-$(printf '%03d' $(($(jq 'length' registry.json) + 1))).
  - Append entry: {id, category:"process", rule:"Address recurring ${bug_type} on ${pod_ip}", why:evidence, added_ts, source:"suggestion_engine"}.
  - Commit: git add standing-rules-registry.json.

  [cascade_coverage_gap]
  - Append a note to suppress.json with reason "pending_cascade_review" and expires_at = 7 days from now.
  - Write structured entry: {issue_type:bug_type, pod_ip, reason:"cascade_coverage_gap_pending_review", expires_at, created_by:"suggestion_engine"}.
  - Commit: git add suppress.json (or audit/results/suppress.json).

  [new_audit_check | self_patch]
  - These categories require self-patch loop (LEARN-07, covered in 215-04). Log INFO "[LEARN-05] category=${category} — queued for self-patch loop" and update proposal status to "queued_for_selfpatch". Return 0.

3. After applying (for categories that applied successfully):
   - Run: git commit -m "learn(215): apply approved suggestion ${proposal_id} — ${category} for ${bug_type}"
   - Run: git push
   - Update proposal status to "applied": jq update + overwrite.
   - Notify Bono via dual-channel (WS + INBOX.md):
     Message: "Suggestion applied — ${category}: ${bug_type} on ${pod_ip}. Evidence: ${evidence}. Confidence: ${confidence}."
   - Log INFO "[LEARN-05] applied: ${proposal_id} (${category})"

Error handling: any step failure → log WARN, update proposal status to "apply_failed", return 1. Never hard-exit.

set -uo pipefail, source guard, no set -e.
