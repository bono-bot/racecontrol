    1. Create `crates/rc-core/src/fleet_health.rs` with:

       a. **Constant POD_FLEET**: Array of (pod_id, pod_number, pod_ip) tuples for pods 1-8, matching the IPs from seed_pods in routes.rs. This is the authoritative source for which pods exist — even if none have connected yet, the fleet API returns all 8.

       b. **Struct `PodFleetHealth`** (Debug, Clone, Serialize):
          - pod_id: String
          - pod_number: u32
          - pod_ip: String
          - ws_connected: bool (computed at query time from agent_senders, NOT stored)
          - http_reachable: bool (updated by probe loop)
          - agent_version: Option<String> (from StartupReport)
          - uptime_secs: Option<u64> (computed at query time from agent_started_at)
          - last_seen: Option<DateTime<Utc>> (from PodInfo.last_seen)
          - last_http_check: Option<DateTime<Utc>> (when last probe ran)

       c. **Struct `FleetHealthStore`** (stored in AppState): Per-pod mutable data that the probe loop and WS handler write to. NOT the full PodFleetHealth — that is assembled at query time. Fields:
          - http_reachable: bool
          - last_http_check: Option<DateTime<Utc>>
          - agent_version: Option<String>
          - agent_started_at: Option<DateTime<Utc>> (computed as Utc::now() - uptime_secs when StartupReport arrives)

       d. **`pub fn store_startup_report(store: &mut FleetHealthStore, version: &str, uptime_secs: u64)`** — stores version and computes agent_started_at = Utc::now() - Duration::from_secs(uptime_secs).

       e. **`pub fn clear_fleet_health_on_disconnect(store: &mut FleetHealthStore)`** — clears agent_version and agent_started_at (but NOT http_reachable — that is probe-driven).

       f. **`pub async fn probe_loop(state: Arc<AppState>)`** — every 15 seconds, probes all 8 pods in parallel using `futures_util::future::join_all`. Each probe: `state.http_client.get(format!("http://{}:8090/health", pod_ip)).timeout(Duration::from_secs(3)).send()`. Updates FleetHealthStore.http_reachable and last_http_check. Use a dedicated reqwest::Client with 3s timeout (NOT state.http_client which has 30s timeout — create it once inside the loop function, before the loop).

       g. **`pub async fn fleet_health_handler(State(state): State<Arc<AppState>>) -> Json<Value>`** — assembles the response by iterating POD_FLEET, reading from: (1) pod_fleet_health map for http_reachable/version/started_at, (2) agent_senders for ws_connected (using the `!sender.is_closed()` pattern from deploy.rs), (3) pods map for last_seen, (4) computing uptime_secs from agent_started_at. Returns `{ "pods": [...], "timestamp": "..." }` with pods sorted by pod_number. Missing entries in any map = defaults (false, None).
