Priority endpoints (all confirmed from routes.rs):

| Method | Path | Purpose |
|--------|------|---------|
| GET | /fleet/health | Array of PodFleetStatus (ws_connected, http_reachable, version, build_id, uptime_secs, ip_address, last_seen) |

| GET | /billing/report/daily | Daily billing summary |
| GET | /billing/sessions | Historical billing sessions |

CRITICAL: There is NO /fleet/exec endpoint. Use /pods/{id}/exec per-pod.
CRITICAL: No auth required for LAN requests (fleet/health confirmed public, others accessible from .27).
CRITICAL: Use native fetch() (Node 22) — do NOT install axios or node-fetch.

PodFleetStatus fields: pod_number, pod_id, ws_connected, http_reachable, version, build_id, uptime_secs, crash_recovery, ip_address, last_seen, last_http_check

    3. Create server.js with exactly these 10 tools:

       **Helper function:**
       ```javascript
       const BASE_URL = process.env.RACECONTROL_BASE_URL || 'http://192.168.31.23:8080/api/v1';

       async function apiCall(path, options = {}) {
         const url = `${BASE_URL}${path}`;
         const res = await fetch(url, options);
         if (!res.ok) {
           const text = await res.text();
           throw new Error(`${res.status} ${res.statusText}: ${text}`);
         }
         return res.json();
       }
       ```

       **Tool 1: get_fleet_health** — no params. `GET /fleet/health`. Description: "Get real-time status of all 8 sim racing pods. Returns ws_connected, http_reachable, version, uptime_secs, ip_address per pod."

       **Tool 2: list_sessions** — no params. `GET /sessions`. Description: "List active and recent racing sessions."

       **Tool 3: get_active_billing** — no params. `GET /billing/active`. Description: "Get currently active billing sessions across all pods."

       **Tool 4: list_laps** — optional params: `sessionId` (z.string().optional()), `limit` (z.number().default(50).optional()). `GET /laps?session_id={sessionId}&limit={limit}` (build query string from provided params only). Description: "List recent lap times. Optionally filter by session ID."

       **Tool 5: list_drivers** — no params. `GET /drivers`. Description: "List all registered drivers (name, phone, email)."

       **Tool 6: stop_billing** — params: `billingId` (z.string(), "Billing session ID to stop"). `POST /billing/{billingId}/stop`. Description: "End an active billing session by ID."

       **Tool 7: exec_on_pod** — params: `podId` (z.string(), "Pod ID, e.g. pod-1 through pod-8"), `cmd` (z.string(), "Command to execute on the pod"), `timeoutMs` (z.number().default(30000), "Timeout in milliseconds"). `POST /pods/{podId}/exec` with body `{ "cmd": cmd, "timeout_ms": timeoutMs }`. Content-Type: application/json. Description: "Execute a shell command on a specific pod via rc-agent. Pod IDs: pod-1 through pod-8."

       **Tool 8: pod_self_test** — params: `podId` (z.string(), "Pod ID, e.g. pod-1"). `GET /pods/{podId}/self-test`. Description: "Trigger a 30-second self-test on a pod. Returns port probe results and LLM health verdict."

       **Tool 9: get_daily_report** — no params. `GET /billing/report/daily`. Description: "Get today's billing summary: total sessions, revenue, per-pod breakdown."

       **Tool 10: list_billing_sessions** — no params. `GET /billing/sessions`. Description: "List historical billing sessions with start/end times and amounts."

       **Error handling:** Wrap each tool handler in try/catch. On error, return `{ content: [{ type: 'text', text: 'Error: ' + error.message }], isError: true }`.

       **Server setup:**
       ```javascript
       const server = new McpServer({ name: 'rc-ops-mcp', version: '1.0.0' });
       // ... register all 10 tools ...
       const transport = new StdioServerTransport();
       await server.connect(transport);
       ```

       NOTE: No Google OAuth needed — this is a pure HTTP proxy to racecontrol REST API.
       NOTE: No env var validation at startup (BASE_URL has a sensible default).
