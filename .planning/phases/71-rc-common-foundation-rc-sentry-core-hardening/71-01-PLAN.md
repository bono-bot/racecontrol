**Step 3: Create crates/rc-common/src/exec.rs**

Create the new module with these components:

1. **ExecResult struct** (public, with Debug, Clone):
   - `stdout: String`
   - `stderr: String`
   - `exit_code: i32`
   - `timed_out: bool`
   - `truncated: bool`

2. **`pub fn run_cmd_sync(cmd: &str, timeout: Duration, max_output: usize) -> ExecResult`**
   - Import `std::process::Command`
   - On Windows: `use std::os::windows::process::CommandExt;` with `const CREATE_NO_WINDOW: u32 = 0x08000000;`
   - Spawn `cmd.exe /C {cmd}` with `.stdout(Stdio::piped()).stderr(Stdio::piped())`
   - Apply `#[cfg(windows)] c.creation_flags(CREATE_NO_WINDOW);`
   - On spawn failure: return ExecResult with stderr = error message, exit_code = -1, timed_out = false, truncated = false
   - Import `wait_timeout::ChildExt;` and call `child.wait_timeout(timeout)`
   - On `Ok(None)` (timeout): call `child.kill()`, then `child.wait()` to reap, then read stdout/stderr pipes manually into Vec<u8>, truncate, convert
   - On `Ok(Some(status))` (exited): read stdout/stderr pipes into Vec<u8>, truncate, convert
   - On `Err(e)`: return ExecResult with stderr = error, exit_code = -1
   - Use `truncate_output()` helper for all output paths

   IMPORTANT: Cannot use `child.wait_with_output()` after `wait_timeout()` because wait_timeout already waits. After timeout branch: `child.kill()` then `child.wait()` to reap, then read remaining bytes from the piped stdout/stderr handles via `child.stdout.take()` and `child.stderr.take()`, read to Vec<u8>.
   After normal exit branch: similarly take stdout/stderr handles and read them.

3. **`fn truncate_output(out: Vec<u8>, err: Vec<u8>, max: usize) -> (String, String, bool)`**
   - Check if `out.len() + err.len() > max`
   - Truncate out to `max` bytes, truncate err to `max.saturating_sub(out.len())` bytes
   - CRITICAL: Truncate Vec<u8> BEFORE String::from_utf8_lossy (never truncate a String -- panics on char boundary)
   - Return (stdout_string, stderr_string, truncated_bool)

4. **`#[cfg(feature = "tokio")] pub async fn run_cmd_async(cmd: &str, timeout: Duration, max_output: usize) -> ExecResult`**
   - Import `tokio::process::Command` and `tokio::time::timeout as tokio_timeout`
   - Spawn `cmd.exe /C {cmd}` with `.kill_on_drop(true)` and `.stdout(Stdio::piped()).stderr(Stdio::piped())`
   - Apply `#[cfg(windows)] c.creation_flags(CREATE_NO_WINDOW);`
   - Wrap `child.wait_with_output()` in `tokio_timeout(timeout, ...)`
   - On timeout (Err): kill and return ExecResult with timed_out=true
   - On success: truncate output, return ExecResult
   - On spawn error: return ExecResult with stderr = error

5. **Unit tests** (inline `#[cfg(test)] mod tests`):
   - `test_run_cmd_sync_basic`: run `echo hello`, assert stdout contains "hello", exit_code == 0, timed_out == false
   - `test_run_cmd_sync_timeout`: run `ping -n 10 127.0.0.1` (takes ~9 seconds), timeout 1 second, assert timed_out == true
   - `test_run_cmd_sync_bad_command`: run `nonexistent_command_xyz`, assert exit_code != 0 or stderr non-empty
   - `test_output_truncation`: run `echo` with a small max_output (e.g., 5 bytes), assert truncated == true and stdout.len() <= 5
   - `test_truncate_output_fn`: directly test truncate_output with known Vec<u8> inputs exceeding max
