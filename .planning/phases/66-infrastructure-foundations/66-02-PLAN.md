  <action>
Step 1 — Discover server Tailscale IP from James's Tailscale:
Run `tailscale status` on James's machine to see all peers. Look for the server (.23 / Racing-Point-Server or similar hostname). Record the 100.x.x.x IP.

If server is not visible in James's tailscale status, discover from server side:
POST http://192.168.31.23:8090/exec with:
{"cmd": "tailscale status", "timeout_ms": 10000}
The first line of output shows the server's own Tailscale IP.

If Tailscale is not active on the server (command fails or returns error), document this and proceed with LAN-only path.

Step 2 — Test exec via Tailscale IP (if discovered):
Write a JSON payload file, then:
curl -s -X POST http://<TAILSCALE_IP>:8090/exec -H "Content-Type: application/json" -d @payload.json
where payload.json contains: {"cmd": "hostname", "timeout_ms": 5000}

Expect: {"success": true, "exit_code": 0, "stdout": "<server-hostname>", "stderr": ""}

Step 3 — Test exec via LAN IP:
curl -s -X POST http://192.168.31.23:8090/exec -H "Content-Type: application/json" -d @payload.json

Expect: same response as Step 2.

Step 4 — Run a more substantive command to confirm full exec pipeline:
{"cmd": "powershell -Command \"Get-Process racecontrol -ErrorAction SilentlyContinue | Select-Object ProcessName, Id, CPU | Format-Table -AutoSize\"", "timeout_ms": 10000}

This confirms rc-agent can run PowerShell commands and return meaningful output.

IMPORTANT: Write JSON payloads to temp files and use `curl -d @file` per CLAUDE.md rule.
DANGER: Never include "RCAGENT_SELF_RESTART" in any cmd string.

Note: rc-agent returns HTTP 500 for non-zero exit codes (Pitfall 3). Always check the response body, not just HTTP status.
