This task requires physical access to the venue router. It cannot be automated.

**Root cause:** TP-Link EX220 returned Error 5024 when attempting to add DHCP reservation because the server's active ARP entry conflicted with the reservation add.

**Fix procedure (at venue):**

1. **Reboot the TP-Link router** — power cycle it (unplug, wait 10 seconds, replug). This flushes the ARP cache that caused Error 5024.
2. **Wait for router to fully boot** — approximately 2 minutes until WiFi SSIDs reappear.
3. **Immediately add reservation before server reconnects:**
   - Open http://192.168.31.1 in a browser
   - Navigate to: Network > LAN Settings > Address Reservation (or DHCP > Address Reservation depending on firmware)
   - Click "Add"
   - MAC Address: `10:FF:E0:80:B1:A7` (or `10-FF-E0-80-B1-A7` depending on format the UI expects)
   - Reserved IP: `192.168.31.23`
   - Status: Enabled
   - Click Save
4. **Verify** the reservation appears in the table with Status: Enabled.
5. **If Error 5024 recurs:** The server reconnected too fast. Power cycle the router again and add the reservation faster, or temporarily disconnect the server's Ethernet cable before rebooting the router.

**WARNING:** Router reboot will cause ~2 minutes of network downtime for all venue devices. Do this during a maintenance window, not during active customer sessions.
