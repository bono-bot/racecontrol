# Phase 13.1 Plan 01: Pod Fleet Reliability — Config Self-Heal Summary

**rc-agent load_config() now searches the executable's directory first, making it self-healing when launched by DeskIn or SYSTEM services with a different CWD**

## Performance

- **Completed:** 2026-03-15T~06:18:00Z
- **Tasks:** 2 (TDD RED + GREEN)

## Accomplishments

- Extracted `config_search_paths()` as a pure testable function returning `[exe_dir/rc-agent.toml, rc-agent.toml, /etc/racecontrol/rc-agent.toml]`
- Updated `load_config()` to use `config_search_paths()` so exe directory is always the first candidate
- Error message on failure now lists all searched paths (e.g. `No config file found. Searched: C:\Program Files\DeskIn\rc-agent.toml, rc-agent.toml, /etc/racecontrol/rc-agent.toml`)
- Added 3 new unit tests verifying path priority, CWD fallback position, and error message format
- All 3 new tests RED (compile + fail) in Task 1, GREEN in Task 2; all 8 existing tests still pass

## Task Commits

2. **Task 2: Implement exe-path config resolution (GREEN)** - `613efe6` (feat)

_Note: TDD tasks have two commits — test (RED) then feat (GREEN)_

## Files Created/Modified

- `crates/rc-agent/src/main.rs` — Added `config_search_paths()` function and updated `load_config()` to use it; added 3 new test functions

## Decisions Made

- Exe directory is primary search candidate: rc-agent is self-contained regardless of how it is launched (DeskIn, SYSTEM watchdog, cargo run)
- CWD fallback kept at index 1 (not removed) so `cargo run` in dev continues to work without environment changes
- `config_search_paths()` stubbed as empty Vec during RED phase — this allows the 3 new tests to compile and fail at runtime without needing a serial_test crate
- `std::env::current_exe()` uses `if let Ok` pattern — no `.unwrap()` in production code as per project convention
