}

**compute_rollup() function** — pure computation, takes records and returns rollups:
```rust
pub fn compute_rollup(records: &[crate::model_eval_store::EvalRecord]) -> Vec<EvalRollup>
```
Algorithm:
1. Group records by model_id using a HashMap
2. For each group, compute: total_runs = records.len(), correct_runs = count where correct=true
3. accuracy = if total_runs > 0 { correct_runs as f64 / total_runs as f64 } else { 0.0 }
4. total_cost = sum of cost_usd
5. avg_cost_usd = if total_runs > 0 { total_cost / total_runs as f64 } else { 0.0 }
6. cost_per_correct_usd = if correct_runs > 0 { total_cost / correct_runs as f64 } else { 0.0 }
7. period_start = earliest created_at in group; period_end = latest created_at in group
8. Skip groups with 0 records entirely

**spawn() function** — weekly cron using the IST midnight pattern from weekly_report.rs:
```rust
pub fn spawn(eval_store: std::sync::Arc<std::sync::Mutex<crate::model_eval_store::ModelEvalStore>>) {
    tokio::spawn(async move {
        tracing::info!(target: "state", task = "eval_rollup", event = "lifecycle", "lifecycle: started");
        loop {
            let secs = crate::weekly_report::seconds_until_next_sunday_midnight_ist();
            let jitter_secs: u64 = rand::random::<u64>() % 300; // 0-5 min jitter
            tracing::info!(
                target: LOG_TARGET,
                secs_until_rollup = secs,
                "Sleeping until next Sunday midnight IST for eval rollup"
            );
            tokio::time::sleep(std::time::Duration::from_secs(secs + jitter_secs)).await;
            run_weekly_rollup(&eval_store).await;
        }
    });
}
```

**run_weekly_rollup() async function**:
1. Compute period: from = `(Utc::now() - chrono::Duration::days(7)).to_rfc3339()`, to = `Utc::now().to_rfc3339()`
2. Lock eval_store and call `query_all(Some(&from), Some(&to))` — release the lock immediately after (never hold across .await)
3. Call `compute_rollup(&records)`
4. Open `ModelEvalRollupStore::open(EVAL_DB_PATH)` in a `spawn_blocking` closure (rusqlite conn creation is sync)
5. Insert each rollup row
6. Log summary: `tracing::info!(target: LOG_TARGET, model_count, "EVAL-02: weekly rollup complete")`

**Important — never hold Mutex across .await**: Follow the pattern: `let records = { let guard = eval_store.lock()?; guard.query_all(...)?  }; // guard dropped here`. Then do async work outside the lock.

**seconds_until_next_sunday_midnight_ist()**: This function is currently in `weekly_report.rs`. Change its visibility to `pub` in weekly_report.rs so eval_rollup.rs can call it as `crate::weekly_report::seconds_until_next_sunday_midnight_ist()`. Do not duplicate the logic.
