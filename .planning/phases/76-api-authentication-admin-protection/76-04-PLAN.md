    Create `crates/racecontrol/src/auth/rate_limit.rs`:

    ```rust
    use tower_governor::{GovernorConfig, GovernorConfigBuilder, GovernorLayer};
    use std::sync::Arc;

    /// Rate limit config for auth endpoints: 5 requests per 60 seconds per IP.
    /// Applied to: /auth/admin-login, /customer/login, /customer/verify-otp,
    ///             /auth/validate-pin, /auth/kiosk/validate-pin
    pub fn auth_rate_limit_layer() -> GovernorLayer<tower_governor::key_extractor::PeerIpKeyExtractor> {
        let config = GovernorConfigBuilder::default()
            .per_second(12)  // Replenish rate: 1 token per 12 seconds
            .burst_size(5)   // Max burst: 5 requests
            .finish()
            .expect("GovernorConfig builder");
        GovernorLayer {
            config: Arc::new(config),
        }
    }
    ```

    Note: `per_second(12)` with `burst_size(5)` means 5 burst requests allowed, then 1 every 12 seconds (5 per minute sustained). Verify tower_governor 0.8 API at implementation time -- the builder API may differ from 0.4. If `GovernorConfigBuilder` does not exist in 0.8, use whatever the current API provides. The key extractor may be `SmartIpKeyExtractor` or `PeerIpKeyExtractor` -- check the crate docs.
