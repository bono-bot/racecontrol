    In `crates/racecontrol/src/config.rs`, add to `AuthConfig`:

    Add env var override in `apply_env_overrides()` (after the jwt_secret section):
    ```rust
    if let Ok(val) = std::env::var("RACECONTROL_ADMIN_PIN_HASH") {

            tracing::info!("Overriding admin_pin_hash from RACECONTROL_ADMIN_PIN_HASH env var");

        }
    }
    ```

    **Step 3: Create admin.rs module**

    Create `crates/racecontrol/src/auth/admin.rs` with:

    1. `AdminLoginRequest` (Deserialize) with `pin: String`
    2. `AdminLoginResponse` (Serialize) with `token: String`, `expires_in: u64`
    3. `hash_admin_pin(pin: &str) -> Result<String, String>` using Argon2id + random salt
    4. `verify_admin_pin(pin: &str, hash_str: &str) -> bool` using Argon2 PasswordVerifier
    5. `admin_login` handler:
       - Extract `admin_pin_hash` from `state.config.auth` (return 503 if None)
       - Run `verify_admin_pin` via `tokio::task::spawn_blocking` (argon2 is CPU-heavy)
       - If invalid: return 401
       - If valid: use `super::middleware::StaffClaims` (from Plan 01) + `jsonwebtoken::encode` to create a 12h JWT
       - Return `AdminLoginResponse { token, expires_in: 43200 }`

    Plan 01 is a dependency, so `StaffClaims` from middleware.rs will already exist. Use `super::middleware::StaffClaims` directly -- no need for a local copy.

For the handler test, build a minimal AppState with a known admin_pin_hash (generate with hash_admin_pin in the test setup), create a test Router with `.route("/admin-login", post(admin_login))`, and use `tower::ServiceExt::oneshot` to test.
