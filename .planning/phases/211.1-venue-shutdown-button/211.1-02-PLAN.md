**Create `kiosk/src/app/shutdown/page.tsx`:**

A dedicated shutdown page with these states:

```
States: idle -> confirming -> auditing -> audit_passed -> shutting_down -> complete
                                       -> audit_blocked (shows P1 findings)
                                       -> error
```

**Structure:**

```tsx
"use client";

import { useState, useEffect } from "react";
import { StaffLoginScreen } from "@/components/StaffLoginScreen";
import { api } from "@/lib/api";
import type { VenueShutdownResponse } from "@/lib/types";
```

**Auth gate:** Same pattern as staff/page.tsx -- read sessionStorage in useEffect, show StaffLoginScreen if not authed. Include hydrated flag.

**Page layout:**
- Full-screen dark background (bg-rp-black)
- Centered card (max-w-lg mx-auto)
- Racing Point branding at top
- Big red "Shutdown Racing Point" button (idle state)
- Back link to /staff page

**State machine:**

1. **idle**: Shows the shutdown button. Red bg, white text, large. Below it: "This will safely shut down all pods, POS, and server. James (.27) will stay online."

2. **confirming**: Modal/overlay confirmation dialog:
   - "Shut down all Racing Point systems?"
   - "James (.27) will stay online to implement fixes on next boot."
   - Two buttons: "Yes, Shutdown" (red) and "Cancel" (grey border)

3. **auditing**: After confirmation, call `api.venueShutdown()`. Show:
   - Spinner animation
   - "Running pre-shutdown audit..."
   - "This checks for critical issues before shutdown. Please wait."
   - Note: The API call blocks while audit runs (up to ~120s). Use a 150s fetch timeout.

4. **audit_passed -> shutting_down**: If response.status === "shutting_down":
   - Green checkmark icon
   - "Audit passed. Shutdown sequence initiated."
   - Show ordered list with animated progress:
     - "Pods 1-8: Shutting down..." (with spinner)
     - "POS PC: Waiting..."
     - "Server: Waiting..."
   - After 45s, update pods to "Done", POS to "Shutting down..."
   - After 60s, update POS to "Done", Server to "Shutting down..."
   - After 120s, show "Complete" state
   - (These are visual timers -- actual shutdown is handled by the script)

5. **audit_blocked**: If response.status === "blocked":
   - Red X icon
   - If reason === "billing_active": "Cannot shutdown: {active_sessions} billing sessions still active. End all sessions first."
   - If reason === "audit_failed": "Pre-shutdown audit found critical issues:" + truncated output
   - If reason === "james_offline": "James is offline. Contact Bono for remote shutdown."
   - "Return to Staff Terminal" button

6. **error**: Network or unexpected errors. Show message + retry button.

**Styling:** Use existing Tailwind classes: bg-rp-black, bg-rp-red, hover:bg-rp-red-hover, text-rp-grey, border-rp-border, bg-rp-surface, rounded-xl. Font: Montserrat body. Match the existing kiosk dark theme.
