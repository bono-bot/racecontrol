## Gaps Summary

**1 gap blocking full goal achievement:**

The Bono fallback path (James offline scenario — SHUTDOWN-BONO-FALLBACK, Success Criterion 4) is **functionally implemented in Rust** but the **kiosk UI breaks for this case**. The Rust handler returns `{"status": "fallback_bono", ...}` when James is unreachable and Bono orchestrates shutdown. However:

- `VenueShutdownResponse` in `kiosk/src/lib/types.ts` does not include `"fallback_bono"` in the status union
- `shutdown/page.tsx` has no state machine branch for this status
- The response falls through to the `catch-all` error handler and displays "Shutdown Failed"

This is a broken user experience for the offline-James scenario: shutdown actually succeeds (pods via wol::shutdown_pod, server via `shutdown /s /t 60`), but staff sees an error. Plan 03 acceptance criteria item 5 explicitly required: "Also update VenueShutdownResponse type in kiosk to handle `fallback_bono` status" — this step was not completed.

**Fix required:** Add `"fallback_bono"` to the type union in types.ts and add a `shutting_down`-equivalent state branch in shutdown/page.tsx for `status === "fallback_bono"`.

The fix is small (2 files, ~15 lines) and does not require backend changes.
