## What Was Built

### Task 1: boot-time-fix.sh

A startup script that James runs on boot to process pre-shutdown findings saved by venue-shutdown.sh:

1. Reads `audit/results/pre-shutdown-findings.json` -- created by venue-shutdown.sh at shutdown
2. Sources `audit/lib/fixes.sh` to get APPROVED_FIXES whitelist and apply_fix functions
3. For each failed step in findings, checks if it matches an APPROVED_FIXES entry and applies the fix
4. Logs all actions to LOGBOOK.md in standard pipe-delimited format
5. Notifies Bono via comms-link send-message.js when fixes were applied
6. Archives the findings file with timestamp (mv, not delete -- audit trail)
7. Checks INBOX.md for Bono fallback shutdown findings and logs if found

Features:
- Idempotent: no findings file = clean no-op exit
- DRY_RUN=true mode for testing without applying fixes
- jq required for JSON parsing -- exits cleanly if missing
- Handles CLEAN verdict and bugs_found=0 as no-op with cleanup

### Task 2: Bono fallback in venue_shutdown_handler

Extended `venue_shutdown_handler` with a `bono_fallback_shutdown()` helper:

**New flow:**
1. Billing drain check (unchanged)
2. Try James SSH for audit (unchanged)
3. **NEW:** If James SSH fails -> `bono_fallback_shutdown()`
   - Tries Bono relay HTTP POST to `srv1422716.hstgr.cloud:8766/relay/exec/run`
   - If Bono also unreachable -> returns `{"status": "blocked", "reason": "both_offline", ...}`
   - If Bono reachable: shuts down all 8 pods via `wol::shutdown_pod()`
   - Notifies Bono best-effort (non-blocking)
   - Schedules server self-shutdown via `shutdown /s /t 60`
   - Returns `{"status": "fallback_bono", ...}` with per-pod results

**Error handling:**
- No `.unwrap()` anywhere in fallback path
- Pod shutdown failures are logged and included in response, not fatal
- Bono notification failure is logged but non-blocking
- Self-shutdown spawn failure is logged and included in response
