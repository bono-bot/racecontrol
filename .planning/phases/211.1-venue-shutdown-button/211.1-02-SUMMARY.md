## Decisions Made

- Shutdown API call blocks for the full audit duration (up to 150s) rather than polling — simpler client state, fewer edge cases
- Visual countdown timers (45s/60s/120s) are purely cosmetic — actual shutdown sequencing happens in venue-shutdown.sh
- Blocked state message is reason-specific: billing_active shows active session count, audit_failed shows truncated output snippet, james_offline redirects to Bono for remote shutdown
