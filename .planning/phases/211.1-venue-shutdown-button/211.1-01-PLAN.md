**Step 1: Billing drain check.**
Read `state.billing_timers` (or equivalent -- check state.rs). Count active billing sessions. If any active sessions exist, return immediately:
```json
{"status": "blocked", "reason": "billing_active", "active_sessions": N, "message": "N active billing sessions. End or wait for them to complete before shutdown."}
```

**Step 2: Trigger pre-shutdown audit via comms-link relay.**
Use `state.http_client` (reqwest::Client from AppState) to POST to `http://localhost:8766/relay/exec/run` with body:

Wait up to 120 seconds (set timeout on the request). Parse relay response JSON. Check the nested `exitCode` field:
- exitCode 0: audit passed
- exitCode 1: P1 issues found -- BLOCK
- exitCode 2 or timeout: audit error -- BLOCK

If blocked, return:
```json
{"status": "blocked", "reason": "audit_failed", "exit_code": N, "output": "truncated stdout last 500 chars", "message": "Pre-shutdown audit found critical issues. Fix before shutting down."}
```

**Step 3: Trigger ordered shutdown via relay.**
If audit passes, POST to relay again:

```json
{"status": "shutting_down", "message": "Shutdown sequence initiated. Pods -> POS -> Server. James stays alive."}
```

**Important:** Use `reqwest::Client` with explicit timeouts, NOT `.unwrap()`. All errors return proper JSON error responses. If relay is unreachable (James offline), return `{"status": "error", "reason": "james_offline", "message": "James is offline. Try Bono fallback or manual shutdown."}`.

```bash
#!/usr/bin/env bash
# venue-shutdown.sh -- Ordered venue shutdown
# Called by venue_shutdown_handler via comms-link relay
# Order: Pods (1-8) -> POS PC (.20) -> Server (.23)
# James (.27) stays alive.

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
LOG_FILE="$REPO_ROOT/audit/results/venue-shutdown-$(TZ=Asia/Kolkata date '+%Y-%m-%d_%H-%M').log"
SERVER_URL="${SERVER_URL:-http://192.168.31.23:8080}"
TIMESTAMP=$(TZ=Asia/Kolkata date '+%Y-%m-%d %H:%M IST')

mkdir -p "$(dirname "$LOG_FILE")"

log() { echo "[$(TZ=Asia/Kolkata date '+%H:%M:%S')] $1" | tee -a "$LOG_FILE"; }
```

**Step 1: Save pre-shutdown findings for boot-time fix.**
Copy the latest auto-detect findings to a known location:
```bash
FINDINGS_FILE="$REPO_ROOT/audit/results/pre-shutdown-findings.json"
LATEST_AUTODETECT=$(ls -td "$REPO_ROOT/audit/results/auto-detect-"* 2>/dev/null | head -1)
if [[ -n "$LATEST_AUTODETECT" ]] && [[ -f "$LATEST_AUTODETECT/summary.json" ]]; then
  cp "$LATEST_AUTODETECT/summary.json" "$FINDINGS_FILE"
  log "Saved pre-shutdown findings to $FINDINGS_FILE"
fi
```

**Step 2: Shutdown pods 1-8 in parallel via server API.**
For each pod, call `POST $SERVER_URL/api/v1/pods/{pod_id}/shutdown` using curl. Run all 8 in parallel with `&` and `wait`. Use pod IDs pod-1 through pod-8 (check the actual ID format from routes.rs -- may be numeric or string). Log each result. Timeout: 30 seconds per pod.

```bash
log "Step 2: Shutting down pods 1-8..."
for i in $(seq 1 8); do
  (
    result=$(curl -s --max-time 30 -X POST "$SERVER_URL/api/v1/pods/pod-$i/shutdown" \
      -H "Content-Type: application/json" 2>&1)
    log "Pod $i: $result"
  ) &
done
wait
log "All pod shutdown commands sent"
```

**Step 3: Wait 45 seconds for pods to complete shutdown.**
```bash
log "Step 3: Waiting 45s for pods to power off..."
sleep 45
```
