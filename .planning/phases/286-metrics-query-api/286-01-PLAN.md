**Query params:**

pub struct MetricsQueryParams {
    pub metric: String,

    pub pod: Option<u32>,
    pub resolution: Option<String>,
}

**Auto-resolution function (per D-07):**
```rust
fn select_resolution(from: i64, to: i64, override_res: Option<&str>) -> &'static str {
    if let Some(r) = override_res {
        match r { "raw" | "hourly" | "daily" => return r, _ => {} }
    }
    let range = to - from;
    if range < 86_400 { "raw" } else if range < 7 * 86_400 { "hourly" } else { "daily" }
}
```

**query_handler (QAPI-01, QAPI-04, QAPI-05):**
- Validate `from < to`, else return 400 with `{"error": "from must be less than to"}` (per D-05)
- Call `select_resolution(from, to, resolution.as_deref())`
- Convert pod number to `format!("pod-{}", n)` for DB binding
- For "raw": query `metrics_samples` with `WHERE metric_name = ? AND recorded_at >= datetime(?, 'unixepoch') AND recorded_at < datetime(?, 'unixepoch')` plus optional `AND pod_id = ?` (or `AND pod_id IS NULL` when no pod filter and querying server-level metrics — actually, when no pod filter, omit the pod_id clause entirely to return ALL pods)
- For "hourly"/"daily": query `metrics_rollups` with `WHERE resolution = ? AND metric_name = ? AND period_start >= datetime(?, 'unixepoch') AND period_start < datetime(?, 'unixepoch')` plus optional pod filter. Use `avg_value` as the response `value` field.
- Use `CAST(strftime('%s', recorded_at) AS INTEGER)` for ts (CRITICAL: strftime returns TEXT, must CAST)
- Return 200 with QueryResponse (per D-04, empty results are fine — just empty points vec)

**names_handler (QAPI-02):**
- `SELECT DISTINCT metric_name FROM metrics_samples UNION SELECT DISTINCT metric_name FROM metrics_rollups ORDER BY metric_name`
- Return NamesResponse

**snapshot_handler (QAPI-03, QAPI-04):**
- Accept optional `?pod=N` filter via PodFilterParams
- Use self-join pattern for latest-per-group:
```sql
SELECT s.metric_name, s.pod_id, s.value,
       CAST(strftime('%s', s.recorded_at) AS INTEGER) AS updated_at
FROM metrics_samples s
INNER JOIN (
    SELECT metric_name, pod_id, MAX(recorded_at) AS max_ts
    FROM metrics_samples GROUP BY metric_name, pod_id
) latest ON s.metric_name = latest.metric_name
         AND COALESCE(s.pod_id, '') = COALESCE(latest.pod_id, '')
         AND s.recorded_at = latest.max_ts
ORDER BY s.metric_name, s.pod_id
```
- When pod filter present, add `WHERE s.pod_id = ?` to outer query
- Parse pod_id "pod-N" back to `Option<u32>` for response: `pod_id.and_then(|s| s.strip_prefix("pod-").and_then(|n| n.parse().ok()))`

**CRITICAL anti-patterns to avoid:**
- No `.unwrap()` — use `unwrap_or_default()` for fetch_all, `?` for critical paths
- COALESCE on pod_id in join (NULL = NULL is false in SQLite)
- CAST strftime result to INTEGER
