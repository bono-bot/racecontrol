**3. event_archive.rs — Create new module with 5 public functions:**

`const LOG_TARGET: &str = "event_archive";`

**a) `append_event(db: &SqlitePool, event_type: &str, source: &str, pod: Option<&str>, payload: serde_json::Value)`**
- Fire-and-forget: compute UUID + IST timestamp before `tokio::spawn`
- Clone all values into the async block (same as activity_log.rs pattern)
- INSERT INTO system_events (id, event_type, source, pod, timestamp, payload) VALUES (?, ?, ?, ?, ?, ?)
- Log errors at WARN level, do NOT propagate

**b) `spawn(state: Arc<AppState>)`**
- Check `state.config.event_archive.enabled`, log + return if disabled
- tokio::spawn with hourly interval (3600s) — same as backup_pipeline
- Track `last_remote_transfer: Option<chrono::NaiveDate>` (module-private, NOT shared with backup_pipeline)
- Each tick calls `archive_tick(state, &mut last_remote_transfer).await`

**c) `async fn archive_tick(state, last_remote_transfer)` — private**
- Step 1: `export_daily_jsonl(&state.db, &archive_dir)` — export yesterday's events
- Step 2: `purge_old_events(&state.db, retention_days)` — ALWAYS after export
- Step 3: If remote_enabled, `transfer_jsonl_to_remote(...)` — IST 02:00-03:59 window only

**d) `async fn export_daily_jsonl(db: &SqlitePool, archive_dir: &str) -> anyhow::Result<String>`**
- Compute yesterday IST: `chrono::Utc::now().with_timezone(&chrono_tz::Asia::Kolkata) - Duration::days(1)`
- Filename: `events-YYYY-MM-DD.jsonl`
- Idempotent: if file exists, return filename immediately
- SELECT all rows WHERE date(timestamp) = yesterday, ORDER BY timestamp ASC
- Write one JSON line per row (parse payload back to Value to avoid double-encoding)
- `std::fs::create_dir_all(archive_dir)` before writing

**e) `async fn purge_old_events(db: &SqlitePool, retention_days: u32) -> anyhow::Result<u64>`**
- `DELETE FROM system_events WHERE timestamp < datetime('now', '-{retention_days} days')`
- Use format! for the days parameter in the SQL string (SQLite datetime modifier)
- Log count of deleted rows at INFO level

**f) `async fn transfer_jsonl_to_remote(state, filepath, filename, last_remote_transfer) -> anyhow::Result<()>`**
- IST window check: `ist_hour == 2 || ist_hour == 3` (same as backup_pipeline)
- NaiveDate dedup: if `last_remote_transfer == Some(today)`, skip
- Reuse backup_pipeline Steps A-E VERBATIM:
  - A: `ssh -o StrictHostKeyChecking=no -o BatchMode=yes -o ConnectTimeout=10 {remote_host} mkdir -p {remote_path}`
  - B: Local SHA256 of the JSONL file
  - C: `scp -o StrictHostKeyChecking=no -o BatchMode=yes -o ConnectTimeout=10` with 120s timeout
  - D: Remote `sha256sum {remote_path}/{filename}` via SSH
  - E: Compare checksums, update `last_remote_transfer = Some(today)` on match
- Use `tokio::process::Command` for ssh/scp (same as backup_pipeline)
