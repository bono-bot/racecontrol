**In crates/rc-agent/src/ffb_controller.rs**, add three new functions AFTER the existing `impl FfbController` block (as free functions, since they don't need &self):

1. **`close_conspit_link(timeout: Duration) -> bool`** (pub, sync):
   - Use FindWindowW with title array: `["Conspit Link 2.0", "ConspitLink2.0", "Conspit Link", "ConspitLink"]` (same as minimize_conspit_window in ac_launcher.rs)
   - Send `PostMessageW(hwnd, WM_CLOSE, 0, 0)` on first match
   - If no window found, log debug and return true (not running = already closed)
   - Poll `is_process_running("ConspitLink2.0.exe")` every 250ms until timeout
   - Return true if process exits, false if still running after timeout
   - Use `#[cfg(windows)]` / `#[cfg(not(windows))]` pattern (non-windows returns true)
   - Import: `use crate::ac_launcher::is_process_running;` (it's `pub`)

2. **`restart_conspit_link()`** (pub, sync):
   - Same logic as `ensure_conspit_link_running()` but without the "is already running" check
   - Path: `r"C:\Program Files (x86)\Conspit Link 2.0\ConspitLink2.0.exe"`
   - Check path exists, log and return if not installed
   - Launch via `hidden_cmd("cmd").args(["/c", "start", "", conspit_path]).spawn()`
   - Spawn thread: sleep 4s, then call `crate::ac_launcher::minimize_conspit_window()` (make it `pub(crate)`)
   - Import: `use crate::ac_launcher::{hidden_cmd, minimize_conspit_window};`
   - Use `#[cfg(windows)]` guard

3. **`pub async fn safe_session_end(ffb: &FfbController)`**:
   - Step 1: `let closed = tokio::task::spawn_blocking(|| close_conspit_link(Duration::from_secs(5))).await.unwrap_or(false);`
   - If !closed, log warn: "ConspitLink did not close within 5s -- proceeding with HID commands (P-20 risk accepted)"
   - Step 2: Clone ffb, spawn_blocking:
     - Call `ffb_clone.fxm_reset()` — log warn on Err but continue
     - Sleep 50ms
     - 5-step idlespring ramp: target = 2000, for step 1..=5: value = (target * step) / 5, call `ffb_clone.set_idle_spring(value)`, sleep 100ms
   - Step 3: `tokio::task::spawn_blocking(|| restart_conspit_link());` (fire and forget — do NOT .await, ConspitLink restart is background)
   - Log info: "Session-end safety sequence complete — wheel centering with idlespring"
   - Total blocking time: ~5s max (WM_CLOSE timeout) + ~550ms (fxm.reset + ramp) = fits 2-3s when CL closes fast
