Create `scripts/deploy/verify-frontend-deploy.sh` — a verification script that runs all 5 ROADMAP Phase 262 success criteria and reports PASS/FAIL per check.

Script header:
```bash
#!/bin/bash
# verify-frontend-deploy.sh — Phase 262 post-deploy verification gate
# Usage: bash verify-frontend-deploy.sh [repo_root] [server_ip]
#   repo_root: path to racecontrol repo (default: /c/Users/bono/racingpoint/racecontrol)
#   server_ip: server IP (default: 192.168.31.23)
# Exits 0 if all checks pass. Exits 1 if any check fails.
# Run after every frontend deploy to confirm pipeline is correct.
set -uo pipefail

REPO_ROOT="${1:-/c/Users/bono/racingpoint/racecontrol}"
SERVER_IP="${2:-192.168.31.23}"
FAIL=0
PASS_COUNT=0
TOTAL=5
```

Implement the 5 checks in sequence, each printing `[CHECK N/5]` header, result line, and updating FAIL/PASS_COUNT:

**Check 1 — Web static file HTTP 200:**
```
[CHECK 1/5] Web static files served (HTTP 200)
```
- Find the first CSS file in `$REPO_ROOT/web/.next/static/css/` (local build output)
- If no local CSS found, construct URL with `app.css` as the name (common Next.js output name)
- `curl -s -o /dev/null -w "%{http_code}" --max-time 10 "http://$SERVER_IP:3200/_next/static/css/$CSS_FILE"`
- PASS if HTTP 200, FAIL otherwise
- On fail: print `  FIX: cp -r web/.next/static web/.next/standalone/.next/static && restart web`

**Check 2 — Kiosk static file HTTP 200:**
```
[CHECK 2/5] Kiosk static files served (HTTP 200)
```
- Same logic, but URL is `http://$SERVER_IP:3300/kiosk/_next/static/css/$CSS_FILE`
- Find CSS from `$REPO_ROOT/kiosk/.next/static/css/`
- On fail: print `  FIX: cp -r kiosk/.next/static kiosk/.next/standalone/.next/static && restart kiosk`

**Check 3 — NEXT_PUBLIC_ env var audit (both apps):**
```
[CHECK 3/5] NEXT_PUBLIC_ env vars have LAN IPs (not localhost)
```
- Run `bash "$REPO_ROOT/scripts/deploy/check-frontend-env.sh" "$REPO_ROOT/web"` — capture output and exit code
- Run `bash "$REPO_ROOT/scripts/deploy/check-frontend-env.sh" "$REPO_ROOT/kiosk"` — same
- PASS if both exit 0, FAIL if either exits non-zero
- Print the captured output on failure so the operator knows which var is wrong

**Check 4 — outputFileTracingRoot in both next.config.ts:**
```
[CHECK 4/5] outputFileTracingRoot set in both next.config.ts
```
- `grep -l "outputFileTracingRoot" "$REPO_ROOT/web/next.config.ts" "$REPO_ROOT/kiosk/next.config.ts" 2>/dev/null | wc -l`
- PASS if count is 2 (both files contain it), FAIL if 0 or 1
- On fail: print which file is missing the setting and the required line:
  `outputFileTracingRoot: path.join(__dirname)`

**Check 5 — leaderboard-display unauthenticated (HTTP 200, not 302):**
```
[CHECK 5/5] /leaderboard-display is unauthenticated (200 not 302)
```
- `curl -s -o /dev/null -w "%{http_code}" --max-time 10 -L "http://$SERVER_IP:3200/leaderboard-display"`
  Note: use `-L` to follow redirects so we get the final code
- Actually we want to detect the redirect: use `-s -o /dev/null -w "%{http_code}" --max-redirect 0`
  (no -L flag) to get the immediate response code
- PASS if 200, FAIL if 301/302 (redirecting to login)
- On fail: print `  FIX: Remove AuthGate wrapper from leaderboard-display page. Do NOT wrap this route.`

**Final summary block:**
```bash
echo ""
echo "=============================="
echo "Phase 262 Deploy Gate Results"
echo "=============================="
echo "  Passed: $PASS_COUNT/$TOTAL"
if [ "$FAIL" -eq 0 ]; then
  echo "  Status: PASS — deploy pipeline fully verified"
  echo "=============================="
  exit 0
else
  FAIL_COUNT=$(( TOTAL - PASS_COUNT ))
  echo "  Status: FAIL — $FAIL_COUNT check(s) failed"
  echo "  Fix the issues above and re-run: bash scripts/deploy/verify-frontend-deploy.sh"
  echo "=============================="
  exit 1
fi
```

Make executable: `chmod +x scripts/deploy/verify-frontend-deploy.sh`

Important implementation notes:
- Use `|| true` on curl calls to prevent `set -e` from aborting the script when a check fails (we want to run all 5 checks even if earlier ones fail) — use `FAIL=1` to track failures, do NOT use `set -e` for the individual checks
- Each check must be wrapped in logic that sets FAIL=1 but continues to the next check
- The script should work even if the apps are not currently deployed (checks will fail gracefully with clear messages, not bash errors)
- For CSS filename: use `$(ls "$REPO_ROOT/web/.next/static/css/"*.css 2>/dev/null | head -1 | xargs basename 2>/dev/null || echo "app.css")` pattern
