<task type="auto">
  <name>Task 1: Create check-frontend-env.sh pre-build audit script</name>
  <files>scripts/deploy/check-frontend-env.sh</files>
  <action>
Create `scripts/deploy/check-frontend-env.sh` — a bash script that audits NEXT_PUBLIC_ env vars before any Next.js build.

The script takes one argument: the app source directory (e.g. `/c/Users/bono/racingpoint/racecontrol/web`).

Logic:
1. Scan the app's `src/` directory for all NEXT_PUBLIC_ variable references:
   `grep -rn "NEXT_PUBLIC_" "$APP_SRC/src/" --include="*.ts" --include="*.tsx" | grep -o "NEXT_PUBLIC_[A-Z_]*" | sort -u`

2. For each unique var found, check that it exists in `$APP_SRC/.env.production.local`:
   - If the var is missing from .env.production.local → print error and set FAIL=1
   - If the var value contains "localhost" or "127.0.0.1" → print error and set FAIL=1

3. Also check that the file itself exists at all — if `.env.production.local` is missing → error.

4. Exit 0 if all vars are present and none contain localhost. Exit 1 if any check fails.

Output format on failure:
```
ERROR: NEXT_PUBLIC_WS_URL not found in web/.env.production.local
ERROR: NEXT_PUBLIC_API_URL uses localhost — must be LAN IP (192.168.31.23)
FAIL: env var audit failed for web — fix .env.production.local before building
```

Output on success:
```
OK: web env var audit passed — 3 NEXT_PUBLIC_ vars, all have LAN IPs
```

Script header must include:
```bash
#!/bin/bash
# check-frontend-env.sh — pre-build NEXT_PUBLIC_ env var audit
# Usage: bash check-frontend-env.sh <app_src_dir>
# Exits 0 if all NEXT_PUBLIC_ vars are present with LAN IPs.
# Exits 1 if any var is missing or uses localhost.
# Standing rule: run before every Next.js build (DQ-02)
set -euo pipefail
```

Make executable: `chmod +x scripts/deploy/check-frontend-env.sh`

Modify `scripts/deploy/deploy-nextjs.sh` to add two hardening steps. The existing 8-step pipeline becomes 9 steps — shift the numbers so:
- Steps [1]-[8] remain the same content but [1] gets the env audit injected before the build command
- Step [9] is a new static file smoke test added after health check passes

**Change 1 — Inject env audit into step [1/9] (was [1/8]):**

Update the step header to `[1/9]` and add BEFORE the `npm run build` call:

```bash
echo "[1/9] Pre-build env var audit..."
SCRIPT_DIR_DEPLOY="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
bash "$SCRIPT_DIR_DEPLOY/check-frontend-env.sh" "$SRC"
echo "  Env var audit passed."
echo ""
echo "[1/9] Building $APP..."
```

Note: the env audit exits non-zero on failure, which with `set -euo pipefail` will abort the deploy immediately before any build runs.

**Change 2 — Renumber all subsequent steps from [2/8]–[8/8] to [2/9]–[8/9]** (just header strings, not logic).

**Change 3 — Add step [9/9] static file smoke test after health check passes (inside the `if [ "$HEALTH_OK" = true ]` block):**

After `DEPLOY_RESULT="success"` is set and before the closing `fi` of the health check block, add:

```bash
    # Static file smoke test — verifies .next/static was copied into standalone
    echo ""
    echo "[9/9] Static file smoke test..."
    # Determine the correct static URL for this app
    case "$APP" in
      kiosk) STATIC_BASE_PATH="/kiosk" ;;
      admin) STATIC_BASE_PATH="/admin" ;;
      web)   STATIC_BASE_PATH="" ;;
    esac

    # Find the first CSS file that was built
    FIRST_CSS=$(ls "$SRC/.next/static/css/"*.css 2>/dev/null | head -1 | xargs -I{} basename {})

    if [ -z "$FIRST_CSS" ]; then
      echo "  WARNING: No CSS file found in .next/static/css/ — skipping static smoke test"
    else
      STATIC_URL="http://192.168.31.23:$PORT${STATIC_BASE_PATH}/_next/static/css/$FIRST_CSS"
      echo "  Testing: $STATIC_URL"
      STATIC_HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 "$STATIC_URL" 2>/dev/null || echo "000")
      if [ "$STATIC_HTTP_CODE" = "200" ]; then
        echo "  PASS: Static file returned HTTP $STATIC_HTTP_CODE"
      else
        echo "  FAIL: Static file returned HTTP $STATIC_HTTP_CODE (expected 200)"
        echo "  Static files not served correctly — deploy may have succeeded but CSS is broken"
        echo "  Manual fix: cp -r $SRC/.next/static $SRC/.next/standalone/.next/static"
        DEPLOY_RESULT="degraded"
        ERROR="static file smoke test failed: $STATIC_URL returned $STATIC_HTTP_CODE"
      fi
    fi
```

Also update the final success echo to include static check result:
```bash
echo "=== DEPLOY SUCCESS === $APP ($SERVICE_NAME) on :$PORT — health + static serving verified"
```

The script must remain `set -euo pipefail` — the static smoke test does NOT abort the deploy (it logs degraded), because the health check already passed and the app is serving HTML. But it logs the failure so the operator knows static is broken.
