Create `audit/lib/results.sh` with 3 exported functions following the same style as core.sh (comment header per function, `export -f` after each):

**Function 1: `finalize_results`**
- Called after all phases complete, before exit code counting
- Counts PASS/WARN/FAIL/QUIET results by iterating `$RESULT_DIR/phase-*.json` files via jq
- Writes counts into `$RESULT_DIR/run-meta.json` (merge with existing run-meta using jq: add `completed_at`, `venue_state`, `counts: {pass, warn, fail, quiet, total}`)
- Calls `update_index`

**Function 2: `update_index`**
- Reads `$SCRIPT_DIR/results/index.json` (create `[]` if missing)
- Appends one entry: `{timestamp: (ist_now), mode: $AUDIT_MODE, result_dir: $RESULT_DIR, venue_state: $VENUE_STATE, counts: {pass, warn, fail, quiet, total}}`
- Writes back atomically: jq to temp file, then mv (prevent corruption on ctrl-C)
- Index path: `"${SCRIPT_DIR}/results/index.json"` where SCRIPT_DIR comes from audit.sh

**Function 3: `find_previous_run`**
- Reads `$SCRIPT_DIR/results/index.json`
- Returns the `result_dir` of the second-to-last entry (the run before the current one)
- If fewer than 2 entries exist, returns empty string (no previous run available)
- Use: `jq -r '.[-2].result_dir // empty'`
- Print the path to stdout so callers can capture with `$(find_previous_run)`

File header: `#!/usr/bin/env bash` + comment block matching core.sh style.
All functions use `local` for internal variables. No `set -e`.
