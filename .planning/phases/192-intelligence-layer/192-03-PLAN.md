  <action>
Create `audit/lib/suppress.sh` with 3 exported functions following core.sh style:

**Function 1: `check_suppression` (phase, host, message)**
- Reads `$SCRIPT_DIR/suppress.json` (where SCRIPT_DIR is set by audit.sh, the audit/ directory)
- If file doesn't exist, returns 1 (not suppressed)
- Gets current IST date: `TZ=Asia/Kolkata date '+%Y-%m-%d'`
- Iterates suppress.json entries via jq:
  1. Filter entries where `.phase` matches the `$phase` arg
  2. Filter entries where `.host_pattern` matches `$host` (use jq `test()` after converting glob `*` to regex `.*`)
  3. Filter entries where `.message_pattern` is empty OR `$message` matches the pattern (jq `test()`)
  4. Filter entries where `.expires_date >= $current_date` (string comparison works for ISO dates)
  5. If any entry matches all filters, print the `reason` to stdout and return 0 (suppressed)
  6. If no match, return 1 (not suppressed)

**Function 2: `apply_suppressions`**
- Called once after all phases complete, before report generation
- Iterates all `$RESULT_DIR/phase-*.json` files
- For each file with status FAIL or WARN:
  - Calls `check_suppression` with the phase, host, and message from the JSON
  - If suppressed (return 0): rewrites the JSON file with `status: "SUPPRESSED"` and adds `suppression_reason` field (the reason from check_suppression)
  - If not suppressed: leaves the file unchanged
- Prints summary: "Suppressed: N phase(s)"

**Function 3: `get_severity_score` (status, severity)**
- Returns a numeric score for sorting/priority:
  - FAIL + P1 = 100
  - FAIL + P2 = 80
  - FAIL + P3 = 60
  - WARN + P1 = 50
  - WARN + P2 = 40
  - WARN + P3 = 30
  - SUPPRESSED = 10
  - QUIET = 5
  - PASS = 0
- Print score to stdout, return 0

All functions use `local` variables. No `set -e`. Each function has a comment header block.
File ends with 3 `export -f` lines.
