**Suite 5 structure:**

```
# ===========================================================================
# Suite 5: Domain-matched verification (GATE-01..04)
# ===========================================================================
```

**GATE-02 — Display domain:**
If `DOMAIN_DISPLAY=1`:
- Check if env var `VISUAL_VERIFIED=true` is set
- If NOT set, print: `"BLOCKED: Display-domain changes detected but VISUAL_VERIFIED=true not set"` followed by the triggering files
- Print: `"To pass: verify screens on pods, then re-run with VISUAL_VERIFIED=true bash test/gate-check.sh"`
- Set `DOMAIN_FAIL=1`
- If IS set, print: `"Display verification: VISUAL_VERIFIED=true confirmed"` with PASS label

**GATE-03 — Network domain:**
If `DOMAIN_NETWORK=1`:
- Run `curl -sf -m 5 http://192.168.31.23:8080/api/v1/health` and check exit code
- If curl fails: `"BLOCKED: Network-domain changes detected but server health check failed"`, set `DOMAIN_FAIL=1`
- If curl succeeds: print PASS with response summary
- If changed files contain `ws_handler` or `WebSocket` (case-insensitive): also print a reminder `"WebSocket changes detected — verify live WS connections after deploy"`

**GATE-04 — Parse domain:**
If `DOMAIN_PARSE=1`:
- Check if env var `PARSE_TEST_INPUT` is set (path to test input file)
- Check if env var `PARSE_TEST_EXPECTED` is set (expected output string or file path)
- If either is missing: `"BLOCKED: Parse-domain changes detected but PARSE_TEST_INPUT and PARSE_TEST_EXPECTED not provided"`
- Print: `"To pass: re-run with PARSE_TEST_INPUT=/path/to/input PARSE_TEST_EXPECTED='expected_value' bash test/gate-check.sh"`
- Set `DOMAIN_FAIL=1`
- If both are set: verify the input file exists, print PASS with both values logged as evidence

**GATE-01 — Billing and Config domains:**
If `DOMAIN_BILLING=1`:
- Print reminder: `"Billing-domain changes detected — verify session start/stop and rate calculation after deploy"`
- Does NOT block (informational), but records in output
If `DOMAIN_CONFIG=1`:
- Print reminder: `"Config-domain changes detected — verify config loads correctly on target machines after deploy"`
- Does NOT block (informational)

**Suite result:**
- If no domains detected: `record_suite 5 "domain verification" "PASS (no domain-specific changes)"`
- If domains detected and all passed: `record_suite 5 "domain verification" "PASS (N domains verified)"`
- If any domain blocked: `record_suite 5 "domain verification" "FAIL (blocked by: display|network|parse)"`, set `OVERALL_FAIL=1`

**Evidence output:**
After all domain checks, if any domain was detected, print a summary block:
```
  Domain verification evidence:
    VISUAL_VERIFIED=true|false
    Server health: OK|FAIL
    Parse test: PROVIDED|MISSING
```
