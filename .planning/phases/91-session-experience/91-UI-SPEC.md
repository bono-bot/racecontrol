# Phase 91 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no shadcn, no component library |
| Preset | not applicable |
| Component library | none — Tailwind CSS 4 utility classes only |
| Icon library | inline SVG — no external icon library (matches existing PWA pattern) |
| Font | Montserrat (body + headings), system-ui fallback stack |
| New libraries | canvas-confetti 1.9.4 (confetti animation), sonner 2.0.7 (toast notifications) |

No `components.json` found in `/root/racecontrol/pwa/`. The PWA uses Tailwind CSS 4 with custom design tokens declared in `globals.css` under `@theme`. This is the established pattern across all 21 existing PWA routes — do not introduce shadcn or any component library for this phase.

**canvas-confetti** and **sonner** are animation/notification utilities, not component libraries. They do not affect the design system classification.

## Typography

Two weights only: 400 (regular) and 700 (bold). Inherited from Phase 90 UI-SPEC.

| Role | Size | Weight | Line Height | Tailwind | Notes |
|------|------|--------|-------------|----------|-------|
| Label | 12px | 400 regular | 1.4 | `text-xs` | Secondary labels, percentile sublabel, "Best Lap · Lap N" |
| Body | 14px | 400 regular | 1.5 | `text-sm` | Main content text, stats grid values |
| Heading | 20px | 700 bold | 1.2 | `text-xl` | Section headings, PB banner heading, percentile ranking text, PB "NEW PERSONAL BEST!" banner |
| Hero/Display | 36px | 700 bold | 1.0 | `text-4xl` | Peak moment lap time — dominant visual centrepiece of the session report page, also used for the page-level "Session Report" heading if one is added |

Notes:

- The 36px Hero/Display size is reserved for the single lap time value in PeakMomentHeroCard and any top-level page heading. Do not apply it to section headings or secondary data.
- PB toast title "NEW PERSONAL BEST!" uses sonner's built-in success variant rendering — do not override font size in the toast body.
- All type is white (`text-white`) or muted grey (`text-rp-grey: #5A5A5A`).
- Lap time values always use `font-mono` in addition to the declared weight.

Source: Phase 90 UI-SPEC (inherited), RESEARCH.md Pattern 5

## Color

Inherited from Phase 90 UI-SPEC. Source: `globals.css`, `sessions/[id]/page.tsx`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1A1A1A` (`rp-dark`) | Page background (`min-h-screen bg`) |
| Secondary (30%) | `#222222` (`rp-card`) | All card surfaces, hero card background, stats grid cells |
| Accent (10%) | `#E10600` (`rp-red`) | Reserved for specific elements — see below |
| Border | `#333333` (`rp-border`) | Card borders, section dividers |
| Muted text | `#5A5A5A` (`rp-grey`) | Labels, secondary info, "Faster than X% of drivers" context text |
| PB gold | `rgb(234 179 8)` (`yellow-500`) | PB banner background tint (`bg-yellow-500/10`), PB banner text (`text-yellow-400`) — matches existing share card PB highlight pattern |
| PB improvement | `rgb(52 211 153)` (`emerald-400`) | Improvement delta text (`-0:00.000`) — matches existing share card pattern |
| Confetti | `#E10600` (rp-red), `#FFD700` (gold), `#FFFFFF` (white) | canvas-confetti particle colors — brand palette |

Accent (`#E10600`) reserved for:
1. Active BottomNav tab icon and label (existing)
2. Loading spinner border (`border-rp-red`) (existing)
3. Percentile ranking banner background tint (`bg-rp-red/10`) and text (`text-rp-red`) — inline on session detail page
4. Share button background (`bg-rp-red`) (existing — unchanged)
5. Session usage progress bar fill (existing — unchanged)
6. PB confetti primary particle color

Do NOT apply rp-red to the PB banner — the PB banner uses yellow-500 tones (matching existing share card PB highlight). Percentile banner uses rp-red tones. These two states are visually distinct.

Source: Phase 90 UI-SPEC (inherited), `sessions/[id]/page.tsx` lines 356–406 (share card patterns)

**ConfettiOnMount** fires after peak moment hero card renders, with 300ms delay on first view only. Controlled by `sessionStorage.getItem("confetti_shown_" + sessionId)` — skip if key exists, set key after firing.

### New: PeakMomentHeroCard

Conditions: render only when `session.status === "completed"` AND `laps.length > 0`.

Layout:
```
div.bg-rp-card.border.border-rp-border.rounded-xl.p-5.mb-4
  div.text-center.mb-2
    span.text-xs.text-rp-grey.uppercase.tracking-wider  "Peak Moment"
  div.text-center.mb-4
    p.text-4xl.font-bold.text-white.font-mono  {best lap time, e.g. "1:43.271"}
    p.text-xs.text-rp-grey.mt-1               "Best Lap · Lap {peak_lap_number}"
  [if is_new_pb === true]
    div.bg-yellow-500/10.border.border-yellow-500/30.rounded-lg.p-3.text-center
      p.text-yellow-400.font-bold.text-xl  "NEW PERSONAL BEST!"
  [if improvement_ms present and > 0]
    p.text-xs.text-rp-grey.text-center.mt-2
      span.text-emerald-400.font-mono  "-{improvement_delta}"
      span  " faster than your previous best"
```

When `is_new_pb === true`: border changes to `border-yellow-500/30` on the outer card.
When `is_new_pb === false` (normal session, has laps): outer card border stays `border-rp-border`.

### New: PercentileRankingBanner

Conditions: render only when `session.percentile_rank` is present (backend returns non-null value, which requires >= 5 drivers on this track+car combo).

Layout:
```
div.bg-rp-red/10.border.border-rp-red/20.rounded-xl.p-4.mb-4.text-center
  p.text-rp-red.font-bold.text-xl  "Faster than {percentile_rank}% of drivers"
  p.text-xs.text-rp-grey.mt-1     "{track} · {car}"
```

Do NOT show this banner when `percentile_rank` is null or undefined. Do NOT show "Not enough data for ranking" — simply omit the banner entirely below the 5-driver threshold. The backend enforces the threshold.

### New: ConfettiOnMount (in `components/Confetti.tsx`)

A client-side utility component. Returns null from JSX — no visual output.

```tsx
"use client";
// Fires canvas-confetti on mount, once per session (sessionStorage gate)
// Colors: #E10600, #FFD700, #FFFFFF (RacingPoint brand)
// Three-burst pattern: center spread at y:0.6, then left and right side bursts at 250ms delay
// particleCount: 100 (center), 50 (left), 50 (right)
// spread: 70 (center), 55 (left/right)
```

Export both `ConfettiOnMount` (component for session detail page) and `fireConfetti` (callable function for active session toasts).

### New: Toaster Provider (in `components/Toaster.tsx` + `app/layout.tsx`)

Mount once in `app/layout.tsx`. Configuration:

```tsx
<Toaster
  theme="dark"
  position="top-center"
  richColors
  toastOptions={{
    style: {
      background: "#222222",
      border: "1px solid #333333",
      color: "#FFFFFF",
    },
  }}
/>
```

`theme="dark"` is mandatory — RacingPoint PWA is always dark. Without this, sonner defaults to system preference and may show a white toast on the dark background.

### Modified: ActiveSessionPage (`/book/active/page.tsx`)

**Add PB polling hook.** The page already polls `loadState` every 3000ms for session state. Add a separate PB polling effect that runs every 5000ms while the session is active.

PB polling visual output: on each new PB event from the polling endpoint:
1. Call `toast.success("NEW PERSONAL BEST!", { description: "{lap_time} on {track}", duration: 5000 })` via sonner
2. Call `fireConfetti()` from the Confetti module

Toast styling uses the Toaster's `toastOptions` (dark background). The success variant from sonner's `richColors` shows a green left border — this is acceptable as a positive indicator. Do not override the success variant color; the border and checkmark are part of the established sonner dark+richColors pattern.

Polling lifecycle rules:
- Initialize `since` to `new Date().toISOString()` at the moment the polling `useEffect` mounts — only PBs after page open trigger toasts
- Stop polling when `billing?.status !== 'active'` — cleanup in `useEffect` return via `clearInterval`
- Track last-seen PB `lap_id` in a `useRef` to deduplicate if the same event appears in consecutive polls

## Interaction Patterns

### Loading State

Identical to all existing pages (inherited from Phase 90):
```tsx
<div className="min-h-screen pb-20 flex items-center justify-center">
  <div className="w-8 h-8 border-2 border-rp-red border-t-transparent rounded-full animate-spin" />
</div>
```

### Confetti Trigger Rules

| Scenario | Trigger | Frequency |
|----------|---------|-----------|
| Session detail page load, `is_new_pb === true` | `ConfettiOnMount` fires after 300ms | Once per session (sessionStorage gate) |
| Active session: PB event arrives from polling | `fireConfetti()` called immediately | Once per unique PB lap_id |
| Session detail re-visit (already seen) | No confetti | sessionStorage key exists → skip |
| Session with no PB (`is_new_pb === false`) | No confetti | Not triggered |

### Toast Trigger Rules (active session only)

| Scenario | Toast | Duration |
|----------|-------|----------|
| New PB detected by polling | `toast.success("NEW PERSONAL BEST!", { description: "{lap_time} on {track}" })` | 5000ms |
| First PB ever (first lap on track) | Same toast as above — no special copy difference | 5000ms |
| Session not active (billing ended) | Polling stops; no toast | — |

### Percentile Rendering Rules

| Condition | Render |
|-----------|--------|
| `percentile_rank` present (>= 5 drivers sample) | Show PercentileRankingBanner |
| `percentile_rank` null/undefined | Omit banner entirely |
| `percentile_rank === 100` (top driver) | Show "Faster than 100% of drivers" — this is legitimate when the data confirms it |
| Active session (not yet completed) | Never show percentile — applies to completed sessions only |

### Stats Grid — Best Lap Highlight

The existing stats grid moves to position 5 in the peak-end layout. The "Best Lap" stat cell gets a background highlight to visually reinforce the peak moment:

```
div.bg-rp-red/10.border.border-rp-red/20.rounded-lg.p-3   ← "Best Lap" cell only
  p.text-rp-grey.text-xs  "Best Lap"
  p.text-white.font-mono.font-bold  {best_lap_display}
```

All other stat cells retain `bg-[#1A1A1A]` (the existing inner cell color from the share card pattern).

### Session Summary Header — Condensed

In peak-end layout, the Session Summary Header moves to position 4 and is condensed. Remove the usage progress bar (time-usage visualization) from the summary card — it is now secondary context. Keep: pod number, status badge, experience/game/track/car, driving time, date. The usage bar was the dominant visual element; removing it prevents it from competing with the PeakMomentHeroCard.

### Auth Redirect

Inside `useEffect` only — never at render time (hydration safety):
```tsx
if (!isLoggedIn()) { router.replace("/login"); return; }
```

## Copywriting Contract

| Element | Copy |
|---------|------|
| Peak moment card label | "Peak Moment" |
| Peak moment sublabel | "Best Lap · Lap {N}" |
| PB banner in hero card | "NEW PERSONAL BEST!" |
| PB improvement line | "-{time} faster than your previous best" |
| Percentile banner | "Faster than {N}% of drivers" |
| Percentile sublabel | "{track} · {car}" |
| PB toast title (active session) | "NEW PERSONAL BEST!" |
| PB toast description | "{lap_time} on {track}" |
| Share button | "Share Session Report" (existing — unchanged) |
| Empty state heading (no laps) | "No lap data yet" |
| Empty state body | "Lap times will appear once you complete your session." |
| Error state | "Could not load session. Go back and try again." |

No destructive actions in this phase.

Note on first-ever PB: The research document recommends "First Personal Best!" vs "NEW Personal Best!" to distinguish a first-ever PB. This distinction is NOT implemented in this phase — both cases use "NEW PERSONAL BEST!" The backend does not currently expose whether it is a first-ever PB vs an improvement PB. If the backend adds an `is_first_pb` flag in a future phase, the copy can be updated then.
