```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests/e2e/playwright',
  fullyParallel: false,
  retries: process.env.CI ? 2 : 1,
  workers: 1,
  reporter: [
    ['html', { open: 'never' }],
    ['junit', { outputFile: 'test-results/junit.xml' }],

  use: {

    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
    video: 'off',
  },
  projects: [
    {
      name: 'chromium-kiosk',
      use: { ...devices['Desktop Chrome'] },
    },
  ],
  webServer: {
    command: 'npm run start',
    url: 'http://192.168.31.23:3300/kiosk',
    reuseExistingServer: true,
    timeout: 30_000,
    cwd: './kiosk',
  },
});
```

Key config decisions (all locked in STATE.md — non-negotiable):
- `fullyParallel: false` — game launch tests mutate live pod state
- `workers: 1` — single worker, no parallel game launches
- `reuseExistingServer: true` — venue kiosk is already running on :3300
- `baseURL` defaults to `http://192.168.31.23:3300` (venue server kiosk)
- `KIOSK_BASE_URL` env var overrides for dev/CI environments
- `screenshot: 'only-on-failure'` — captures screenshots for debugging failed tests
- `trace: 'on-first-retry'` — traces captured only on retry (zero overhead on passing tests)
- `video: 'off'` — too large for venue machine storage

**Step 1: Install cargo-nextest**

```bash
export PATH="$PATH:/c/Users/bono/.cargo/bin"
cargo install cargo-nextest --locked
```

This installs the `cargo-nextest` binary to `~/.cargo/bin/`. The `--locked` flag pins the dependency resolver. This is a one-time install — the binary persists across sessions.

If cargo-nextest is already installed, `cargo install` will skip with "already installed" — this is idempotent.

**Step 2: Verify installation**

```bash
cargo nextest --version
```

Expected output: `cargo-nextest nextest-runner 0.9.x` (exact patch version varies).

**Step 3: Create .config/nextest.toml**

Create directory `C:/Users/bono/racingpoint/racecontrol/.config/` if it does not exist.

Create `C:/Users/bono/racingpoint/racecontrol/.config/nextest.toml` with exactly this content:

```toml
# .config/nextest.toml
# cargo-nextest configuration for RaceControl workspace.
# Per-process test isolation is nextest's default — no explicit setting needed.
# Each test binary runs in its own subprocess, preventing billing state leakage.

[profile.default]
retries = { backoff = "fixed", count = 2, delay = "1s" }
slow-timeout = { period = "60s" }

[profile.default.junit]
path = "test-results/nextest.xml"
```

Key decisions:
- `retries` with fixed backoff: retry failed tests twice with 1s delay (catches transient failures)
- `slow-timeout`: warn if a test takes >60s (catches hangs in billing lifecycle tests)
- `junit.path`: JUnit XML output for test result aggregation (same `test-results/` dir as Playwright)
- Per-process isolation is nextest's default — NOT explicitly set because it is always on

**Step 4: Run cargo nextest to verify configuration**

```bash
export PATH="$PATH:/c/Users/bono/.cargo/bin"
cargo nextest run --workspace 2>&1 | tail -20
```

This should discover tests across all workspace crates (rc-common, racecontrol-crate, rc-agent-crate) and run them with per-process isolation. Exit 0 means all tests pass.

NOTE: If some tests fail due to venue-specific conditions (e.g., database not present, server not running), that is expected. The goal is that nextest itself runs without configuration errors — test failures due to missing runtime environment are acceptable at this stage.
