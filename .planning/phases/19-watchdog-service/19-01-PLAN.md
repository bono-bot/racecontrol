<objective>
Create the rc-watchdog crate: a Windows SYSTEM service binary that monitors rc-agent process health, restarts it in Session 1 on crash, and reports crash events to racecontrol.

Purpose: An unhandled panic or OOM in rc-agent currently leaves the pod permanently dead until a human physically intervenes. The watchdog eliminates this by auto-restarting rc-agent within 10 seconds of any crash, with crash telemetry sent to racecontrol for fleet monitoring.

Output: Compilable rc-watchdog binary with service lifecycle, process polling, Session 1 spawn, crash reporting, and WatchdogCrashReport type in rc-common.
</objective>

    **Step 5: Create crates/rc-watchdog/src/service.rs**
    Service main loop:
    - `pub fn run(arguments: Vec<OsString>) -> anyhow::Result<()>`
    - Register service control handler with mpsc channel for stop/shutdown signals
    - Set service status to Running with STOP + SHUTDOWN accepted
    - Load config: read `C:\RacingPoint\rc-agent.toml` for pod_id and server URL. If not found, derive pod_id from COMPUTERNAME env var, default server URL to `http://192.168.31.23:8080`
    - Main poll loop (5-second sleep):
      1. Check shutdown_rx.try_recv() — break if stop signal
      2. Call `is_rc_agent_running()` — if running, continue
      3. Check restart grace window (15s since last restart) — if active, continue
      4. Log "rc-agent not running, restarting in Session 1"
      5. Call `session::spawn_in_session1(exe_dir)` — log result
      6. If spawn succeeded: increment restart_count, record last_restart_at, call `reporter::send_crash_report()` (fire-and-forget, log on error)
      7. If spawn failed (no session): log and continue — will retry next cycle
    - On loop exit: set service status to Stopped

    `is_rc_agent_running()` function:
    - Run `tasklist /NH /FI "IMAGENAME eq rc-agent.exe"` with CREATE_NO_WINDOW flag (0x08000000)
    - Check stdout contains "rc-agent" — return bool
    - On error: return true (conservative: assume running if can't check)

    Include unit tests:
    - Test is_rc_agent_running logic with mock output strings (extract string-checking to a testable helper `fn output_contains_agent(stdout: &str) -> bool`)
    - Test restart grace window logic (extract to `fn restart_grace_active(last_restart: Option<Instant>, grace_secs: u64) -> bool`)

    **Step 6: Create crates/rc-watchdog/src/session.rs**
    Session 1 process spawn:
    - `pub fn spawn_in_session1(exe_dir: &std::path::Path) -> anyhow::Result<()>`
    - `#[cfg(windows)]` implementation:
      1. `WTSGetActiveConsoleSessionId()` — bail if 0xFFFF_FFFF ("No active console session")
      2. `WTSQueryUserToken(session_id, &mut user_token)` — bail on failure with GetLastError
      3. `DuplicateTokenEx(user_token, TOKEN_ALL_ACCESS, null, SecurityImpersonation, TokenPrimary, &mut dup_token)` — close user_token after
      4. `CreateEnvironmentBlock(&mut env_block, dup_token, 0)`
      5. Build command: `cmd.exe /c "C:\RacingPoint\start-rcagent.bat"`
      6. `CreateProcessAsUserW(dup_token, null, cmd_wide, ...)` with CREATE_UNICODE_ENVIRONMENT + CREATE_NO_WINDOW flags, lpDesktop = "winsta0\\default" for interactive desktop
      7. Close all handles (dup_token, process, thread, env_block via DestroyEnvironmentBlock)
      8. Log success with session_id
    - `#[cfg(not(windows))]` stub: returns `Err(anyhow!("Session 1 spawn not supported on this platform"))`
    - IMPORTANT: Only call start-rcagent.bat when rc-agent is confirmed NOT running (service.rs checks this before calling)

    Include unit test:
    - `#[cfg(not(windows))]` test that spawn_in_session1 returns Err on non-Windows
