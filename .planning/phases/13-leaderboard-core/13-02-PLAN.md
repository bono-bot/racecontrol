<feature>
  <name>Leaderboard sim_type filtering + circuit/vehicle records</name>
  <files>crates/rc-core/src/api/routes.rs, crates/rc-core/tests/integration.rs</files>
  <behavior>
    - test_leaderboard_sim_type_filter: Insert laps for two sim_types (assetto_corsa, f1_25) on the same track. Query with sim_type=assetto_corsa. Verify only AC laps returned. (Covers LB-01, LB-04)
    - test_leaderboard_no_cross_sim: Same setup, query sim_type=f1_25. Verify no AC laps leak.
    - test_leaderboard_suspect_hidden: Insert a valid lap + a valid-but-suspect lap for same driver. Default query returns only non-suspect. (Covers LB-06)
    - test_leaderboard_invalid_toggle: Same setup, query with show_invalid=true. Verify valid=0 laps appear but suspect=1 still hidden. (Covers LB-06)
    - test_circuit_records: Insert best laps for 3 (track, car, sim_type) combos. Query /public/circuit-records. Verify returns one record per combo with correct best time. (Covers LB-02)
    - test_vehicle_records: Insert laps for one car on 3 tracks. Query /public/vehicle-records/{car}. Verify returns best per track. (Covers LB-03)
  </behavior>
  <implementation>
    1. Define LeaderboardQuery struct with sim_type (Option, defaults to "assetto_corsa"), car (Option), show_invalid (Option bool).
    2. Modify public_track_leaderboard:
       - Accept Query(params): Query of LeaderboardQuery
       - Default sim_type to "assetto_corsa" if absent
       - Build WHERE clause: `l.track = ? AND l.sim_type = ?`
       - Default: `AND l.valid = 1 AND (l.suspect IS NULL OR l.suspect = 0)`
       - If show_invalid=true: `AND (l.suspect IS NULL OR l.suspect = 0)` (drops valid=1 but keeps suspect filter)
       - If car filter provided: add `AND l.car = ?`
       - Also update the subquery for lap_id to include sim_type
       - Also update stats query to include sim_type
    3. Create public_circuit_records handler:
       - Query: SELECT track, car, sim_type, MIN(lap_time_ms), driver_id FROM laps WHERE valid=1 AND (suspect IS NULL OR suspect=0) GROUP BY track, car, sim_type ORDER BY track, car
       - JOIN drivers for display name (nickname-aware)
       - Optional sim_type query param filter
       - Register route: .route("/public/circuit-records", get(public_circuit_records))
    4. Create public_vehicle_records handler:
       - Path param: car
       - Query: SELECT track, sim_type, MIN(lap_time_ms), driver_id FROM laps WHERE car=? AND valid=1 AND (suspect IS NULL OR suspect=0) GROUP BY track, sim_type ORDER BY track
       - JOIN drivers for display name
       - Register route: .route("/public/vehicle-records/{car}", get(public_vehicle_records))
    5. Tests use direct SQL queries against in-memory SQLite (same pattern as existing test_leaderboard_ordering) — they test the SQL logic, not HTTP routing.
  </implementation>
</feature>
