Purpose: This is the foundation for all Phase 13 leaderboard queries — every query will filter on `suspect = 0` by default. Without this, gaming the leaderboard with fabricated laps is trivial.

<feature>
  <name>Lap validity hardening with suspect column</name>
  <files>crates/rc-core/src/db/mod.rs, crates/rc-core/src/lap_tracker.rs, crates/rc-core/tests/integration.rs</files>
  <behavior>
    - test_lap_suspect_sector_sum: Insert a lap where sector1_ms + sector2_ms + sector3_ms differs from lap_time_ms by more than 500ms. Verify suspect=1 in DB.
    - test_lap_suspect_sanity: Insert a lap with lap_time_ms = 15000 (15 seconds). Verify suspect=1 in DB.
    - test_lap_not_suspect_valid: Insert a lap with lap_time_ms = 90000, sectors summing to 89800. Verify suspect=0.
    - test_lap_not_suspect_no_sectors: Insert a lap with NULL sectors and lap_time_ms = 90000. Verify suspect=0 (missing sectors do not flag).
    - test_lap_suspect_zero_sectors_ignored: Insert a lap with all sectors = 0 and lap_time_ms = 90000. Verify suspect=0 (zero sectors treated as absent).
  </behavior>
  <implementation>
    1. In db/mod.rs migrate(): Add `ALTER TABLE laps ADD COLUMN suspect INTEGER NOT NULL DEFAULT 0` (idempotent with let _ =).
    2. In tests/integration.rs run_test_migrations(): Add `suspect INTEGER DEFAULT 0` to the laps CREATE TABLE.
    3. In lap_tracker.rs persist_lap(): BEFORE the INSERT, compute suspect:
       - sector_sum_ok: if all three sectors present AND all > 0, check abs(s1+s2+s3 - lap_time_ms) <= 500. If sectors absent or any is 0, treat as ok (true).
       - sanity_ok: lap.lap_time_ms >= 20_000
       - suspect = !sector_sum_ok || !sanity_ok
       - Convert to i32: if suspect { 1 } else { 0 }
    4. Add suspect bind to the INSERT statement (add column name + .bind(suspect_flag)).
    5. Update the current early-return for !lap.valid: suspect laps should still be persisted (they are valid per the game), but flagged. Only lap_time_ms == 0 should be skipped.
       IMPORTANT: The existing `if !lap.valid { return false; }` check should remain — invalid laps per the game are NOT persisted. Suspect is orthogonal: a lap can be valid=1 AND suspect=1 (game says valid, but sector mismatch or impossible time).
  </implementation>
</feature>
