## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Button inner padding (compact), icon button gutters |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, between pod cards |
| xl | 32px | Panel header padding, major layout gaps |
| 2xl | 48px | Side panel section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Game logo images: 40x40px (not a spacing unit — fixed touch target for tap accuracy on kiosk touchscreen)
- Confirmation banner height: 48px minimum (touch-safe for staff finger confirmation)

## Component Inventory

New or modified components for this phase:

### New: GamePickerPanel

Triggered when staff clicks "Launch Game" on a pod card that is in the `idle` state and the game is non-AC. Replaces the SetupWizard for non-AC games.

- Layout: vertical list of available games filtered to `pod.installed_games`
- Each row: 40x40px game logo (PNG from `/game-logos/{slug}.png`) + game name (16px, weight 400) + "Launch" button (accent red, 12px label, weight 600)
- If no games installed beyond AC: show empty state
- Closes on launch or on "Cancel" tap

### Modified: KioskPodCard

- Add game logo display in pod card body when `gameInfo.game_state === "running"` or `"launching"` (40x40px image, left-aligned beside game state text)
- "Launch Game" button already exists at line 389 — path change only (non-AC bypasses wizard, AC still opens wizard)

### New: GameLaunchRequestBanner

Shown in staff/page.tsx when `DashboardEvent::GameLaunchRequested` is received via WebSocket.

- Position: fixed top banner, full width, z-index above pod grid
- Background: `#2A2A2A` (`--color-rp-surface`), 2px left border in `#ca8a04` (warning yellow)
- Content: "[Driver Name] on Pod [N] requests [Game Name]" (14px body) + "Confirm Launch" button (accent red) + "Dismiss" button (text, white, 14px)
- Auto-expires: dismissed automatically after 60s with no action (matches server-side TTL)
- Multiple pending requests: stack banners, newest on top

### Modified: spectator/page.tsx

- `gameLabel()` function already handles all `SimType` variants — no code change needed
- Spectator shows text-only game label (as per locked decision) — existing `gameLabel()` function output suffices

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — staff game launch | "Launch [Game Name]" (e.g. "Launch F1 25") |
| Game picker panel title | "Select Game for Pod [N]" |
| Launch confirmation button | "Launch [Game Name]" |
| Game request banner heading | "[Driver Name] wants to play [Game Name]" |
| Game request confirm button | "Confirm Launch" |
| Game request dismiss button | "Dismiss" |
| Game request auto-expire notice | "Request expired" (replaces banner after 60s) |
| Empty state — no non-AC games installed | "No additional games installed on this pod" |
| Empty state sub-copy | "Only Assetto Corsa is available. Update the pod TOML to add more games." |
| Error state — game launch failed | "Launch failed. Check pod connection and try again." |
| Crash recovery banner | "Game crashed. Relaunching automatically..." |
| Crash recovery failed (max attempts) | "Game failed to relaunch. Restart manually." |
| Running game badge | Abbreviation from `gameLabel()`: "F1", "AC", "ACR", "ACE", "LMU", "iR" |
| Spectator game name display | Full display name: "F1 25", "iRacing", "AC EVO", "EA WRC", "Le Mans Ultimate" |

No destructive actions in this phase. No confirmation dialogs required beyond the PWA request confirm/dismiss.

## Interaction States

### Game Launch Flow (Staff, Non-AC)

| State | Visual |
|-------|--------|
| Idle pod, no game | "Launch Game" button visible (existing pattern) |
| Game picker open | GamePickerPanel slides in or overlays pod card (implementation discretion: inline expand) |
| Launching | Game badge shows "Launching" in yellow (`#ca8a04`), pod card shows `KioskPodState === "selecting"` |
| Running | Game badge shows abbreviation in green (`#16a34a`), game logo at 40x40px visible |
| Crashed | Existing "crashed" state styling — red badge, "Relaunch" button |

### PWA Request Flow (Staff Confirmation)

| State | Visual |
|-------|--------|
| Request received | `GameLaunchRequestBanner` appears at top, yellow left border |
| Staff confirms | Banner dismissed, pod card transitions to "Launching" state |
| Staff dismisses | Banner dismissed with no action |
| Auto-expire (60s) | Banner shows "Request expired" for 2s, then removes |

### Game Logo Display

- Image path: `/game-logos/{slug}.png` (Next.js static from `kiosk/public/game-logos/`)
- Slugs (all lowercase kebab-case, mandatory — see Pitfall 5 in RESEARCH.md):
  - `assetto-corsa.png`
  - `assetto-corsa-evo.png`
  - `assetto-corsa-rally.png`
  - `iracing.png`
  - `f1-25.png`
  - `le-mans-ultimate.png`
  - `forza.png` (disabled in SetupWizard but logo should exist for completeness)
- Fallback: if logo fails to load, show game name abbreviation in a `#333333` rounded chip
