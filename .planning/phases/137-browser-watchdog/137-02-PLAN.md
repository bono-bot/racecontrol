**1. Add `is_browser_alive()` method** on LockScreenManager (place near close_browser, around line 726):

```rust
/// Check if the browser process we spawned is still running.
/// Returns false if no browser was spawned or if the child process has exited.
#[cfg(windows)]
pub fn is_browser_alive(&mut self) -> bool {
    match self.browser_process.as_mut() {
        Some(child) => {
            // try_wait: Ok(Some(_)) = exited, Ok(None) = still running, Err = unknown
            match child.try_wait() {
                Ok(Some(_status)) => {
                    // Process has exited — clear the handle
                    tracing::warn!(target: LOG_TARGET, "Browser process has exited (watchdog detected)");
                    self.browser_process = None;
                    false
                }
                Ok(None) => true, // still running
                Err(e) => {
                    tracing::warn!(target: LOG_TARGET, "Failed to check browser process status: {}", e);
                    false
                }
            }
        }
        None => false,
    }
}

#[cfg(not(windows))]
pub fn is_browser_alive(&mut self) -> bool {
    false
}
```

**3. Add watchdog tick handler in the `tokio::select!` block** (place after the maintenance_retry_interval handler, around line 920). Follow the exact pattern of existing tick handlers:

```rust
_ = conn.browser_watchdog_interval.tick() => {
    // BWDOG-04: skip entirely during safe mode
    if state.safe_mode_active.load(std::sync::atomic::Ordering::Relaxed) {
        continue;
    }

    // Only check when lock screen is active (not Hidden)
    if !state.lock_screen.is_active() {
        continue;
    }

    // BWDOG-02: check for Edge process stacking (>5 msedge.exe)
    let edge_count = crate::lock_screen::LockScreenManager::count_edge_processes();
    if edge_count > 5 {
        tracing::warn!(
            target: LOG_TARGET,
            "Browser watchdog: Edge stacking detected ({} msedge.exe processes) — killing all and relaunching",
            edge_count
        );
        state.lock_screen.close_browser();
        state.lock_screen.launch_browser();
        continue;
    }

    // BWDOG-01: check browser child process liveness
    if !state.lock_screen.is_browser_alive() {
        tracing::warn!(
            target: LOG_TARGET,
            "Browser watchdog: Edge not alive — relaunching"
        );
        state.lock_screen.close_browser();
        state.lock_screen.launch_browser();
    }
}
```

**Key design decisions:**
- Stacking check runs BEFORE liveness check (stacking is a worse condition — kill all first)
- Safe mode gate at the top skips the entire tick (BWDOG-04)
- `is_active()` check prevents watchdog from firing when lock screen is Hidden (no browser expected)
- Uses `continue` after stacking relaunch to avoid redundant liveness check
- Standing rule #14: lifecycle logged via warn-level tracing on relaunch events
