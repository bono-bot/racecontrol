> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — no shadcn, no component library |
| Preset | not applicable |
| Component library | none — Tailwind CSS utility classes only |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge grid item gaps (`gap-1`) |
| sm | 8px | Compact element spacing, label-to-value gaps |
| md | 16px | Default horizontal page padding (`px-4`), card internal padding (`p-4`) |
| lg | 24px | Section spacing between passport tiers, mb between heading and grid |
| xl | 32px | Vertical breathing room between major page sections |
| 2xl | 48px | — (reserved, not used in this phase) |
| 3xl | 64px | — (reserved, not used in this phase) |

Exceptions:

| Value | Usage | Justification |
|-------|-------|---------------|
| 44px | Minimum touch target height for CollectionTile and passport grid items | Mobile tap target standard — not a layout spacing value |
| 64px (`h-16`) | BottomNav fixed height | Existing convention across all PWA pages — do not modify |
| 48px (`pt-12`) | Page top padding | Clears device status bar — existing convention across all PWA pages |
| 80px (`pb-20`) | Page bottom padding | BottomNav clearance — 16px above BottomNav height (64px) + nav itself; existing convention |

Source: detected from `profile/page.tsx`, `stats/page.tsx`, `BottomNav.tsx`

## Typography

Two weights only: 400 (regular) and 700 (bold).

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 | Main content text |
| Label | 12px (`text-xs`) | 400 regular | 1.4 | Secondary labels, tier progress counters, badge counts, badge names, CollectionTile secondary text — visual distinction from body comes from size, not weight |
| Heading | 24px (`text-2xl`) | 700 bold | 1.2 | Page headings, stat values in summary card |
| Display | 20px (`text-xl`) | 700 bold | 1.2 | Section headings |

Notes:

- All type is white (`text-white`) or muted grey (`text-rp-grey: #5A5A5A`) — no mid-grey exceptions
- BottomNav tab names use `text-[10px]` weight 400 — this is a pre-existing convention inherited from the existing BottomNav component and is outside this phase's scope. It is not counted in the declared 4-size scale above.

Source: detected from `profile/page.tsx`, `stats/page.tsx`, `globals.css`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1A1A1A` (`rp-dark`) | Page background (`min-h-screen bg`) |
| Secondary (30%) | `#222222` (`rp-card`) | All card surfaces, BottomNav background, tier section backgrounds |
| Accent (10%) | `#E10600` (`rp-red`) | Reserved for specific elements — see below |
| Border | `#333333` (`rp-border`) | Card borders, tier dividers, BottomNav top border |
| Muted text | `#5A5A5A` (`rp-grey`) | Labels, locked/undriven item text, secondary info |
| Destructive | `rgb(239 68 68 / 0.1)` bg + `rgb(248 113 113)` text | Sign Out button — no destructive actions in this phase |

Accent (`#E10600`) reserved for:
1. Active BottomNav tab icon and label
2. Loading spinner border (`border-rp-red`)
3. Earned badge icon background tint (`bg-rp-red/20`)
4. Tier completion progress bar fill
5. Passport card "View Passport" CTA link text on profile page
6. Currently-active tier label text

Undriven/locked items use `opacity-30` on the full tile — same technique as existing badge grid pattern from RESEARCH.md code example. Do NOT apply rp-red to locked items.

Source: detected from `globals.css`, `profile/page.tsx`, `stats/page.tsx`, `BottomNav.tsx`

## Component Inventory

### New: PassportPage (`/passport/page.tsx`)

Layout structure:
```
min-h-screen pb-20
  div.px-4.pt-12.pb-4.max-w-lg.mx-auto
    h1 "My Passport"           — text-2xl font-bold text-white mb-6
    PassportSummaryCard         — 4-stat grid (tracks, cars, laps, streak)
    PassportSection (Tracks)    — tier heading + CollectionGrid
    PassportSection (Cars)      — tier heading + CollectionGrid
  BottomNav
```

### New: PassportSummaryCard

4-stat grid (2 columns, `grid grid-cols-2 gap-3`):
- Tracks Driven (count)
- Cars Driven (count)
- Total Laps (count)
- Week Streak (N weeks)

Each stat: `bg-rp-card border border-rp-border rounded-xl p-4`
Label: `text-xs text-rp-grey`
Value: `text-2xl font-bold text-white`

### New: PassportSection (Tracks and Cars)

One section per collection type. Each section:
- Section heading: `text-sm font-bold text-white mb-3` with tier name + progress count
- Three tier subsections (Starter, Explorer, Legend) rendered in sequence
- Each tier: tier label row (`text-xs text-rp-grey mb-2`) + progress bar + item grid

Tier progress bar:
- Container: `bg-rp-border rounded-full h-1 mb-3`
- Fill: `bg-rp-red rounded-full h-1` with inline `width: N%` style
- Do NOT animate progress bar — static is sufficient for this phase

### New: CollectionTile (track or car item)

Driven state (full opacity):
```
bg-rp-card border border-rp-border rounded-xl p-3 flex flex-col gap-1
  span.text-xs.text-white         — item name
  span.text-xs.text-rp-grey       — category / best lap or lap count
```

Undriven state (same structure, add `opacity-30`):
```
bg-rp-card border border-rp-border rounded-xl p-3 opacity-30 flex flex-col gap-1
  span.text-xs.text-white         — item name
  span.text-xs.text-rp-grey       — category
```

Grid layout: `grid grid-cols-3 gap-2` for tracks (wider names), `grid grid-cols-3 gap-2` for cars.

Minimum tile height: 44px (touch target compliance).

## Interaction Patterns

### Loading State

Identical to all existing pages:
```tsx
<div className="min-h-screen pb-20 flex items-center justify-center">
  <div className="w-8 h-8 border-2 border-rp-red border-t-transparent rounded-full animate-spin" />
</div>
```

### Auth Redirect

Inside `useEffect` only — never at render time (hydration safety):
```tsx
if (!isLoggedIn()) { router.replace("/login"); return; }
```

### Backfill Transparency

No loading indicator for backfill — it runs server-side before API response returns. The user sees the spinner while the full request (including lazy backfill) completes. Do NOT show a separate "backfill in progress" state.

### Passport Navigation

Passport page is NOT added to BottomNav (7 tabs is already the maximum for the nav bar width). Access via:
1. Link row on `/profile` page — same style as "Friends" and "View Leaderboard" rows
2. PassportCard on `/dashboard` page (if dashboard has a relevant section)

The `/passport` route uses the same `BottomNav` component. The Stats tab stays active when on `/passport` (since `/passport` doesn't match any tab's `href`, all tabs appear inactive — this is acceptable).

## Tier Visual Hierarchy

All three tiers (Starter, Explorer, Legend) are always visible in the passport. Visual distinction:

| Tier | Heading Color | Progress Bar | Driven tiles | Undriven tiles |
|------|--------------|--------------|--------------|----------------|
| Starter | `text-white` | `bg-rp-red` fill | full opacity | `opacity-30` |
| Explorer | `text-white` | `bg-rp-red` fill | full opacity | `opacity-30` |
| Legend | `text-white` | `bg-rp-red` fill | full opacity | `opacity-30` |

Do NOT add tier-gating (showing only the "current" tier). The research document explicitly identifies this as Pitfall 3 (Zeigarnik backfire). All tiers visible simultaneously — the customer sees the full journey.
