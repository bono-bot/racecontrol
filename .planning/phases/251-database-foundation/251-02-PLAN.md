<objective>
Detect orphaned billing sessions both at server startup and via a recurring background job, flag them, and alert staff.

Purpose: After Plan 01 persists timer state with `last_timer_sync_at`, this plan uses that timestamp to detect sessions where the server (or agent) has gone silent. Without this, a server crash can leave billing sessions permanently "active" in the DB with no one monitoring them — the customer gets charged forever or the session blocks the pod.

Output: Startup orphan scan + 5-minute background orphan detector, both logging and alerting on detected orphans.

**Function 1: detect_orphaned_sessions_on_startup (FSM-10)**

```rust
/// On server startup, detect billing sessions that were "active" in DB but have
/// a stale last_timer_sync_at (>5 minutes ago). These are sessions that were
/// running when the server crashed/restarted. Flag them and alert staff.
///
/// Called AFTER recover_active_sessions() — sessions already recovered into memory
/// are NOT orphans (they were properly persisted). This catches sessions where
/// last_timer_sync_at is NULL (never synced — server crashed before first 60s sync)
/// or older than 5 minutes.
pub async fn detect_orphaned_sessions_on_startup(state: &Arc<AppState>) {
    let threshold_minutes = 5;

    // Find active sessions with stale or NULL last_timer_sync_at
    let orphans = sqlx::query_as::<_, (String, String, String, Option<String>, i64)>(
        "SELECT id, pod_id, driver_id, last_timer_sync_at, driving_seconds
         FROM billing_sessions
         WHERE status IN ('active', 'paused_manual', 'paused_disconnect')
         AND (last_timer_sync_at IS NULL
              OR last_timer_sync_at < datetime('now', ?))
        "
    )
    .bind(format!("-{} minutes", threshold_minutes))
    .fetch_all(&state.db)
    .await;

    match orphans {
        Ok(rows) if rows.is_empty() => {
            tracing::info!("Startup orphan check: no orphaned sessions found");
        }
        Ok(rows) => {
            let count = rows.len();
            tracing::error!(
                "STARTUP ORPHAN DETECTION: Found {} billing session(s) with no heartbeat for {}+ minutes",
                count, threshold_minutes
            );

            let mut details = Vec::new();
            for (session_id, pod_id, driver_id, last_sync, driving_secs) in &rows {
                let sync_info = last_sync.as_deref().unwrap_or("NEVER");
                tracing::error!(
                    "  Orphaned session: {} on {} (driver={}, last_sync={}, driving={}s)",
                    session_id, pod_id, driver_id, sync_info, driving_secs
                );
                details.push(format!("{} on {} ({}s)", session_id, pod_id, driving_secs));

                // Mark session with end_reason for tracking
                let _ = sqlx::query(
                    "UPDATE billing_sessions SET end_reason = 'orphan_flagged_startup' WHERE id = ? AND end_reason IS NULL"
                )
                .bind(session_id)
                .execute(&state.db)
                .await;
            }

            // Send WhatsApp alert to staff
            let alert_msg = format!(
                "ORPHAN ALERT (startup): {} stale billing session(s) detected with no heartbeat for {}+ min. Sessions: {}. Check admin dashboard.",
                count, threshold_minutes, details.join(", ")
            );
            if let Some(ref wa_tx) = state.wa_alert_tx {
                let _ = wa_tx.send(alert_msg.clone());
            }
            // Also log to activity for dashboard visibility
            log_pod_activity(&state.db, "server", "orphan_detection", &alert_msg).await;
        }
        Err(e) => {
            tracing::error!("Failed to check for orphaned sessions on startup: {}", e);
        }
    }
}
```

**Function 2: detect_orphaned_sessions_background (RESIL-03)**

```rust
/// Background task: every 5 minutes, check for active billing sessions whose
/// last_timer_sync_at is older than 5 minutes. This catches sessions that became
/// orphaned while the server is running (e.g., agent disconnected, timer loop crashed).
pub async fn detect_orphaned_sessions_background(state: &Arc<AppState>) {
    let threshold_minutes = 5;

    // Only check sessions NOT in the active_timers map — sessions with an active
    // in-memory timer are being managed. Orphans are DB-only sessions.
    let active_session_ids: HashSet<String> = {
        let timers = state.billing.active_timers.read().await;
        timers.values().map(|t| t.session_id.clone()).collect()
    };

    let db_active = sqlx::query_as::<_, (String, String, String, Option<String>, i64)>(
        "SELECT id, pod_id, driver_id, last_timer_sync_at, driving_seconds
         FROM billing_sessions
         WHERE status IN ('active', 'paused_manual', 'paused_disconnect')
         AND (last_timer_sync_at IS NULL
              OR last_timer_sync_at < datetime('now', ?))
        "
    )
    .bind(format!("-{} minutes", threshold_minutes))
    .fetch_all(&state.db)
    .await;

    match db_active {
        Ok(rows) => {
            // Filter to only sessions NOT in active_timers (true orphans)
            let orphans: Vec<_> = rows.into_iter()
                .filter(|(id, _, _, _, _)| !active_session_ids.contains(id))
                .collect();

            if orphans.is_empty() {
                tracing::debug!("Background orphan check: no orphaned sessions");
                return;
            }

            let count = orphans.len();
            tracing::error!(
                "BACKGROUND ORPHAN DETECTION: Found {} billing session(s) with stale heartbeat ({}+ min)",
                count, threshold_minutes
            );

            let mut details = Vec::new();
            for (session_id, pod_id, driver_id, last_sync, driving_secs) in &orphans {
                let sync_info = last_sync.as_deref().unwrap_or("NEVER");
                tracing::error!(
                    "  Orphaned session: {} on {} (driver={}, last_sync={}, driving={}s)",
                    session_id, pod_id, driver_id, sync_info, driving_secs
                );
                details.push(format!("{} on {} ({}s)", session_id, pod_id, driving_secs));

                // Flag in DB
                let _ = sqlx::query(
                    "UPDATE billing_sessions SET end_reason = 'orphan_flagged_background' WHERE id = ? AND end_reason IS NULL"
                )
                .bind(session_id)
                .execute(&state.db)
                .await;
            }

            // Alert staff
            let alert_msg = format!(
                "ORPHAN ALERT (background): {} stale billing session(s) — no heartbeat for {}+ min. Sessions: {}. Investigate immediately.",
                count, threshold_minutes, details.join(", ")
            );
            if let Some(ref wa_tx) = state.wa_alert_tx {
                let _ = wa_tx.send(alert_msg.clone());
            }
            log_pod_activity(&state.db, "server", "orphan_detection", &alert_msg).await;
        }
        Err(e) => {
            tracing::error!("Background orphan detection query failed: {}", e);
        }
    }
}
```

- Check if `state.wa_alert_tx` is an `Option<Sender>` or a bare `Sender` — read whatsapp_alerter.rs to determine the correct send pattern. Adjust the `if let Some(ref wa_tx)` accordingly. If it's a broadcast channel, use `state.wa_alert_tx.send(...)` directly.
- The `log_pod_activity` function is imported at the top of billing.rs already (`use crate::activity_log::log_pod_activity`). Verify its signature — it may be `pub async fn log_pod_activity(db: &SqlitePool, pod_id: &str, event: &str, details: &str)`.
- Use `HashSet` which is already imported at the top of billing.rs.
- Snapshot the active_timers lock and drop before any DB query (standing rule: no lock across .await).
- No .unwrap() — use unwrap_or for Option values.

After the timer-persist spawn added in Plan 01 (or after the billing-db-sync spawn if easier to locate), add:

```rust
// Spawn orphan detection background task (RESIL-03: every 5 minutes)
let orphan_state = state.clone();
tokio::spawn(async move {
    tracing::info!("orphan-detector task started (300s interval)");
    // Initial delay: wait 5 minutes before first background check
    // (startup check already ran, avoid duplicate alerts)
    tokio::time::sleep(Duration::from_secs(300)).await;
    let mut interval = tokio::time::interval(Duration::from_secs(300));
    loop {
        interval.tick().await;
        billing::detect_orphaned_sessions_background(&orphan_state).await;
    }
});
```

The initial 5-minute sleep prevents the background detector from immediately re-flagging sessions that the startup detector just found. After the first sleep, it runs every 5 minutes.

**IMPORTANT:** The task logs its lifecycle (info on start). It uses the same interval pattern as all other background tasks in main.rs. No restart loop wrapper needed — orphan detection is non-critical (unlike billing tick which has panic recovery).
