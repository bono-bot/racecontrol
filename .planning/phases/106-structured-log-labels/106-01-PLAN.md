Two changes in main.rs:

**Change A: build_id in root span (line 291 area)**

Add this const near the top of main() or as a module-level const:
```rust
const BUILD_ID: &str = env!("GIT_HASH");
```

Change line 291 from:
```rust
let _pod_span = tracing::info_span!("rc-agent", pod_id = %pod_id_str).entered();
```
To:
```rust
let _pod_span = tracing::info_span!(
    "rc-agent",
    pod_id = %pod_id_str,
    build_id = BUILD_ID,
).entered();
```

**Change B: Add LOG_TARGET const and target: to all tracing calls**

Add at the top of main.rs (after use statements):
```rust
const LOG_TARGET: &str = "rc-agent";
```

For every `tracing::info!`, `tracing::warn!`, `tracing::error!`, `tracing::debug!`, `tracing::trace!` call in main.rs (66 total), add `target: LOG_TARGET,` as the first argument after the macro name.

Example transforms:
- `tracing::info!("Structured logging initialized for {}", pod_id_str);`
  becomes `tracing::info!(target: LOG_TARGET, "Structured logging initialized for {}", pod_id_str);`

- `tracing::warn!("Some message with [bracket] prefix");`
  becomes `tracing::warn!(target: LOG_TARGET, "Some message without bracket prefix");`

**CRITICAL: Strip legacy bracket prefixes.** If any tracing call in main.rs has a message starting with `[something]`, remove the bracket prefix from the message string when adding `target:`. Example:
- `tracing::info!("[rc-agent] starting up")` becomes `tracing::info!(target: LOG_TARGET, "starting up")`

Do NOT modify `tracing::info_span!` calls — only `tracing::info!`, `tracing::warn!`, `tracing::error!`, `tracing::debug!`, `tracing::trace!`.

After all edits, run `cargo check -p rc-agent` to verify compilation.
