1. Create `crates/rc-sentry-ai/src/stream.rs`:
   - `pub async fn camera_loop(camera: CameraConfig, rtsp_base: String, frame_buf: FrameBuffer)`:
     - Infinite loop: call `connect_and_stream`, on error log warning and sleep 5 seconds before retry
     - On Ok (stream ended normally), log info and reconnect immediately
     - Each iteration logs the camera name and reconnect attempt number

   - `async fn connect_and_stream(camera: &CameraConfig, rtsp_base: &str, frame_buf: &FrameBuffer) -> anyhow::Result<()>`:
     - Build relay URL: `camera.relay_url(rtsp_base)` (e.g., `rtsp://127.0.0.1:8554/entrance`)
     - Parse URL with `url::Url::parse`
     - Create retina session:
       ```rust
       let session_group = retina::client::SessionGroup::default();
       let mut session = retina::client::Session::describe(
           url,
           retina::client::SessionOptions::default()
               .session_group(session_group),
       ).await?;
       ```
     - Setup first video stream (index 0) with TCP transport:
       ```rust
       session.setup(0, retina::client::SetupOptions::default()
           .transport(retina::client::Transport::Tcp(retina::client::TcpTransportOptions::default())))
           .await?;
       ```
     - Play and demux:
       ```rust
       let mut session = session.play(retina::client::PlayOptions::default())
           .await?
           .demuxed()?;
       ```
     - Frame extraction loop using `futures::StreamExt::next()`:
       ```rust
       use futures::StreamExt;
       while let Some(item) = session.next().await {
           match item? {
               retina::codec::CodecItem::VideoFrame(frame) => {
                   frame_buf.update(&camera.name, frame.data().to_vec()).await;
                   // Rate limit: sleep for 1/fps seconds
                   tokio::time::sleep(Duration::from_millis(1000 / camera.fps as u64)).await;
               }
               _ => {} // Ignore audio, metadata
           }
       }
       ```
     - Note: `frame.data()` contains H.264 NAL units, NOT decoded pixels. This is correct for this phase — downstream phases will decode.
