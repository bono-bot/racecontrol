1. Create `crates/rc-sentry-ai/src/relay.rs`:
   - `pub struct RelayStatus` with fields: `status: String`, `api_url: String`, derive `serde::Serialize`, `Clone`
   - Status values: `"healthy"`, `"unreachable"`, `"error"`
   - `pub async fn check_relay_health(api_url: &str) -> RelayStatus`:
     - Use `reqwest::Client::new().get(format!("{api_url}/api/streams")).send().await`
     - On success (2xx): return `RelayStatus { status: "healthy".into(), api_url: api_url.into() }`
     - On HTTP error: return `RelayStatus { status: "error".into(), ... }`
     - On connection error (reqwest::Error): return `RelayStatus { status: "unreachable".into(), ... }`
     - Use a 3-second timeout: `reqwest::Client::builder().timeout(Duration::from_secs(3)).build()?`

2. Create `crates/rc-sentry-ai/src/health.rs`:
   - Axum shared state struct `pub struct AppState` with fields:
     - `frame_buf: FrameBuffer`
     - `relay_api_url: String`
     - `start_time: std::time::Instant`
   - Wrap in `Arc` for Axum: `type SharedState = Arc<AppState>;`

   - `pub fn health_router(state: SharedState) -> axum::Router`:
     - Route `GET /health` to `health_handler`
     - Route `GET /cameras` to `cameras_handler`
     - Attach state via `.with_state(state)`

   - `async fn health_handler(State(state): State<SharedState>) -> axum::Json<serde_json::Value>`:
     - Call `state.frame_buf.status().await` for per-camera metrics
     - Call `relay::check_relay_health(&state.relay_api_url).await` for relay status
     - Derive per-camera status string:
       - `last_frame_secs_ago < 10.0` => `"connected"`
       - `last_frame_secs_ago < 30.0` => `"reconnecting"`
       - `last_frame_secs_ago >= 30.0` or absent => `"disconnected"`
     - Return JSON matching the research specification:
       ```json
       {
         "service": "rc-sentry-ai",
         "status": "ok",
         "uptime_secs": 3600,
         "cameras": {
           "entrance": { "status": "connected", "last_frame_secs_ago": 0.5, "frames_total": 7200 },
           "reception": { "status": "connected", "last_frame_secs_ago": 0.3, "frames_total": 7200 },
           "reception_wide": { "status": "reconnecting", "last_frame_secs_ago": 12.0, "frames_total": 6500 }
         },
         "relay": { "status": "healthy", "api_url": "http://127.0.0.1:1984" }
       }
       ```
     - Overall service `status`: `"ok"` if relay healthy AND at least 1 camera connected, `"degraded"` if relay healthy but some cameras disconnected, `"error"` if relay unreachable

   - `async fn cameras_handler(State(state): State<SharedState>) -> axum::Json<serde_json::Value>`:
     - Return just the cameras section of the health response (lighter endpoint)

3. Update `crates/rc-sentry-ai/src/main.rs`:
