**Phase 07 (phase07.sh) -- CH-01: Allowlist content spot-verification**

The current check only verifies `.processes | length >= 100`. Add a content spot-check:

1. Keep the existing per-pod allowlist count check (it catches empty lists).
2. After the count check for EACH pod, add a spot-verification sub-check:
   - From the already-fetched response, check if `svchost.exe` appears in the allowlist:
     `printf '%s' "$response" | jq -r '.processes[]?' 2>/dev/null | grep -qi 'svchost\.exe'`
   - If svchost.exe is found, PASS with "Allowlist content verified (svchost.exe present)".
   - If svchost.exe is NOT found but count >= 100, WARN with "Allowlist populated but svchost.exe missing -- allowlist may be from wrong source".
   - If count < 10 (already caught by existing check), skip the spot-check.
3. Emit as `server-23-allowlist-pod${n}-content` (new sub-check alongside existing count check).
4. This catches the scenario where the allowlist is populated from a misconfigured source (e.g., wrong pod's process list) -- svchost.exe is a universal Windows process that MUST be in any valid allowlist.

**Phase 25 (phase25.sh) -- CH-02: Menu item availability check**

The current check only counts menu items. Add availability verification:

1. Keep the existing item_count check.
2. After the count check, add a sub-check:
   - From the same response, check if at least one item has `available=true` (or `in_stock=true` -- check the actual field name):
     `printf '%s' "$response" | jq '[.[] | select(.available == true or .in_stock == true)] | length' 2>/dev/null`
   - If available_count >= 1, PASS with "Menu: ${available_count}/${item_count} items available".
   - If available_count == 0 and item_count >= 1, WARN with "Menu has ${item_count} items but NONE available -- all items marked unavailable".
   - If item_count == 0, skip (already handled by existing check).
3. Emit as `server-23-cafe-menu-availability` (new sub-check).
4. Note: the cafe menu response may use `available`, `in_stock`, or `is_available` -- use jq `select(.available == true or .in_stock == true or .is_available == true)` to cover all variants.
