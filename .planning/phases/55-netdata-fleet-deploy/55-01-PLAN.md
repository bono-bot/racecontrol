**Step 2: Create `deploy-staging/deploy-netdata.py`.**

Python script modelled on deploy_pod.py. Must support:
- `python deploy-netdata.py <pod_number>` — install Netdata on one pod
- `python deploy-netdata.py all` — install on all 8 pods sequentially (Pod 8 first)
- `python deploy-netdata.py --check <pod_number|all>` — verify-only mode (no install)
- `python deploy-netdata.py --help` — print usage (must not error)

For each pod, execute these commands via POST to `http://POD_IP:8090/exec`:

1. Defender exclusion: `powershell -Command "Add-MpPreference -ExclusionPath 'C:\\RacingPoint'"` (timeout_ms: 30000)
2. Download MSI: `curl.exe -s -f -o C:\\RacingPoint\\netdata.msi http://192.168.31.27:9998/netdata.msi` (timeout_ms: 120000)
3. Silent install: `msiexec /qn /i C:\\RacingPoint\\netdata.msi /norestart` (timeout_ms: 180000) — CRITICAL: /qn not /quiet, timeout 180s not default 10s
4. Firewall rule: `netsh advfirewall firewall add rule name="Netdata" dir=in action=allow protocol=TCP localport=19999` (timeout_ms: 30000)
5. Verify service: `sc query netdata` (timeout_ms: 10000) — check stdout contains "RUNNING"
6. Clean up MSI: `del /Q C:\\RacingPoint\\netdata.msi` (timeout_ms: 10000)

After all exec steps, verify from James's machine: `curl -sf http://POD_IP:19999/api/v1/info` — confirm HTTP 200 with JSON containing "version" field.

Use POD_IPS dict from deploy_pod.py (copy, don't import). Print PASS/FAIL per step per pod. If `all` mode, deploy Pod 8 (.91) first as canary, then Pods 1-7 sequentially. Print summary at end.

Handle errors gracefully: if a step fails, print the error and continue to next step (some steps are idempotent — firewall rule may already exist). If msiexec fails, still try sc query to check if already installed.
