    Add data retention configuration table:
    ```sql
    CREATE TABLE IF NOT EXISTS data_retention_config (
        id TEXT PRIMARY KEY,
        financial_records_years INTEGER NOT NULL DEFAULT 8,
        pii_inactive_months INTEGER NOT NULL DEFAULT 24,
        updated_at TEXT DEFAULT (datetime('now'))
    );
    INSERT OR IGNORE INTO data_retention_config (id) VALUES ('default');
    ```

    Add driver columns for retention tracking:
    ```rust
    let _ = sqlx::query("ALTER TABLE drivers ADD COLUMN last_activity_at TEXT").execute(pool).await;
    let _ = sqlx::query("ALTER TABLE drivers ADD COLUMN pii_anonymized BOOLEAN DEFAULT 0").execute(pool).await;
    let _ = sqlx::query("ALTER TABLE drivers ADD COLUMN pii_anonymized_at TEXT").execute(pool).await;
    let _ = sqlx::query("ALTER TABLE drivers ADD COLUMN consent_revoked BOOLEAN DEFAULT 0").execute(pool).await;
    let _ = sqlx::query("ALTER TABLE drivers ADD COLUMN consent_revoked_at TEXT").execute(pool).await;
    ```

    Add index: `CREATE INDEX IF NOT EXISTS idx_drivers_last_activity ON drivers(last_activity_at)`

    **LEGAL-08: PII anonymization background job (routes.rs)**

    Add a function `spawn_data_retention_job(state: Arc<AppState>)` that runs daily (86400s interval, 3600s initial delay to avoid boot congestion):

    1. Query data_retention_config for pii_inactive_months
    2. Find drivers with `last_activity_at < datetime('now', '-' || pii_inactive_months || ' months')` AND `pii_anonymized = 0` AND `consent_revoked = 0`
    3. For each such driver, anonymize PII:
       ```sql
       UPDATE drivers SET
           name = 'ANONYMIZED-' || substr(id, 1, 8),
           email = NULL,
           phone = NULL,
           phone_hash = NULL,
           email_hash = NULL,
           guardian_name = NULL,
           guardian_phone = NULL,
           dob = NULL,
           pii_anonymized = 1,
           pii_anonymized_at = datetime('now')
       WHERE id = ? AND pii_anonymized = 0
       ```
    4. Log count of anonymized records: `tracing::info!(target: "data_retention", count = anonymized_count, "PII anonymization cycle complete")`
    5. Do NOT touch financial records (journal_entries, billing_sessions, invoices, wallet_transactions) — these must be retained for 8 years per Income Tax Act

    Wire the job: in the startup section where other background jobs are spawned (search for `tokio::spawn` near orphan detection), add:
    ```rust
    {
        let state_clone = state.clone();
        tokio::spawn(async move {
            spawn_data_retention_job(state_clone).await;
        });
    }
    ```

    Also update `last_activity_at` on key customer actions. Add one-line updates in:
    - start_billing: after successful session creation, `UPDATE drivers SET last_activity_at = datetime('now') WHERE id = ?`
    - topup handler: same update
    - This ensures active customers never get anonymized

    **LEGAL-09: Consent revocation endpoint (routes.rs)**

    Add to customer authenticated routes:
    ```rust
    .route("/customer/revoke-consent", post(revoke_consent_handler))
    ```

    Handler:
    ```rust
    async fn revoke_consent_handler(
        State(state): State<Arc<AppState>>,
        Extension(claims): Extension<Option<StaffClaims>>,  // or customer JWT claims
        Json(body): Json<Value>,
    ) -> Json<Value>
    ```

    The endpoint accepts `{ "driver_id": "...", "reason": "..." }`. For guardian-initiated revocation (minor data deletion), the driver_id comes from the body. For self-revocation, driver_id comes from JWT claims.

    Processing:
    1. Verify the driver exists and is not already revoked
    2. Anonymize PII immediately (same UPDATE as the background job)
    3. Set consent_revoked = 1, consent_revoked_at = datetime('now')
    4. Log to audit_log: action = "consent_revocation", details = reason
    5. Return `{ "ok": true, "message": "Personal data has been anonymized. Financial records retained per legal requirements." }`

    Note: Financial records (journal entries, invoices, billing sessions) are NOT deleted — only PII on the driver record is anonymized. The billing_session.driver_id foreign key still works (driver row exists, just anonymized).

    Also add a staff endpoint for guardian-initiated revocation (guardian calls venue, staff processes it):
    ```rust
    .route("/drivers/{id}/revoke-consent", post(staff_revoke_consent_handler))
    ```
    in staff_routes (cashier+ access). Same logic but takes driver_id from path and requires staff auth.
