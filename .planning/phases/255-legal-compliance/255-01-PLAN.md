    **LEGAL-01: GST separation in post_session_debit (accounting.rs)**

    Modify `post_session_debit()` to split the session amount into 3 journal lines instead of 2:
    - GST is 18% inclusive, so: net_revenue = amount_paise * 100 / 118, gst_amount = amount_paise - net_revenue
    - Use integer arithmetic to avoid floating point: `let net_paise = amount_paise * 100 / 118;` and `let gst_paise = amount_paise - net_paise;`
    - Line 1: Debit acc_wallet for full amount_paise (customer pays full price)
    - Line 2: Credit acc_racing_rev for net_paise (revenue net of tax)
    - Line 3: Credit acc_gst_payable for gst_paise (tax liability)
    - The 3 lines still balance: debit = credit (amount_paise = net_paise + gst_paise)

    Also add a new convenience function `pub async fn post_session_debit_with_gst()` that returns (net_paise, gst_paise) for the invoice generator to use. Actually, simpler: just modify the existing post_session_debit to do the 3-line split and return the computed values as a tuple Result<(String, i64, i64), String> where the tuple is (entry_id, net_paise, gst_paise). Update the existing callers to use the new return type (they currently ignore the return).
