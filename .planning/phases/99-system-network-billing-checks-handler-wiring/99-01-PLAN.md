Add 4 new async check functions to pre_flight.rs, placed after the existing check_window_rect section:

**1. check_billing_stuck(billing_active: bool) -> CheckResult**
- Pure logic, no I/O. Name: "billing_stuck".
- If billing_active is true: Fail with detail "Billing session still active from previous customer (stuck session)".
- If false: Pass with detail "No stuck billing session".

**2. check_disk_space() -> CheckResult**
- Use spawn_blocking + sysinfo::Disks pattern from self_test.rs probe_disk().
- Name: "disk_space". Threshold: 1_000_000_000 bytes (1GB, NOT 2GB like self_test).
- On pass: detail "C: drive: {X}GB free". On fail: detail "C: drive low: {X}MB free (< 1GB)".
- On C: not found: Warn (not Fail) with detail "C: drive not found in sysinfo disk list".
- On spawn_blocking panic: Fail.

**3. check_memory() -> CheckResult**
- Use spawn_blocking + sysinfo::System pattern from self_test.rs probe_memory().
- Name: "memory". Threshold: 2_147_483_648 bytes (2GB, NOT 1GB like self_test).
- On pass: detail "{X}GB RAM available". On fail: detail "Low memory: {X}MB available (< 2GB)".
- On spawn_blocking panic: Fail.

**4. check_ws_stability(ws_connect_elapsed_secs: u64) -> CheckResult**
- Pure logic, no I/O. Name: "ws_stability".
- If ws_connect_elapsed_secs >= 10: Pass with detail "WebSocket stable ({X}s connected)".
- If ws_connect_elapsed_secs < 10: Warn (NOT Fail per NET-01 spec) with detail "WebSocket recently reconnected ({X}s ago, < 10s stability threshold)".
