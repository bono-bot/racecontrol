Create `crates/rc-agent/src/firewall.rs` following the research template from 16-RESEARCH.md.

Structure the module for testability by separating arg-building logic from subprocess execution:

1. **Constants:** `RULE_ICMP = "RacingPoint-ICMP"`, `RULE_TCP = "RacingPoint-RemoteOps"`
2. **Enum:** `FirewallResult { Configured, Failed(String) }` with `#[derive(Debug, PartialEq)]`
3. **Public function:** `configure() -> FirewallResult` that:
   - Logs "Applying firewall rules (profile=any)..."
   - Deletes stale rules by name (idempotent -- exit 0 even if absent)
   - Adds ICMP echo-request rule (icmpv4:8,any, dir=in, action=allow, profile=any, enable=yes)
   - Adds TCP 8090 rule (TCP, localport=8090, dir=in, action=allow, profile=any, enable=yes)
   - Returns `Configured` if both succeed, `Failed(msg)` if either fails
   - Logs the result (info on success, warn on failure)
4. **Helper:** `run_netsh(args: &[&str]) -> bool` using `std::process::Command::new("netsh")` with `#[cfg(windows)] cmd.creation_flags(CREATE_NO_WINDOW)`. Logs stderr on failure. Returns `out.status.success()`.
5. **Testable helpers (private):** `build_icmp_args()`, `build_tcp_args()`, `build_delete_args(name)` that return `Vec<String>` -- these are what unit tests verify.

Key constraints:
- Use `#[cfg(windows)]` for `CommandExt` import and `.creation_flags()` call (matches remote_ops.rs pattern)
- Do NOT use `.unwrap()` or `.expect()` on netsh results -- always return `Failed` on error
- Do NOT use async/tokio -- this is synchronous, called before tokio event loop
- Do NOT check rule existence before deleting -- delete-first is simpler and equally correct
- Do NOT clean up old batch file rules (AllowICMP, RCAgent) -- leave them, they're additive
- Add module doc comment: `//! Firewall auto-configuration -- ensures ICMP echo + TCP 8090 are open on every startup.`

Unit tests (in `#[cfg(test)] mod tests`):
- Verify rule name constants are namespaced with "RacingPoint-"
- Verify rule names are distinct
- Verify FirewallResult enum variant inequality
- Verify build_icmp_args contains required protocol/profile/dir/action args
- Verify build_tcp_args contains required protocol/port/profile/dir/action args
- Verify build_delete_args contains "delete", "rule", and the rule name
