**Step 1: Append cafe tables to db/mod.rs migrate() function.**

Find the end of the `migrate()` function (before the closing `Ok(())`). Append these CREATE TABLE + CREATE INDEX statements:

```rust
// ─── Cafe Menu ──────────────────────────────────────────────────────────
sqlx::query(
    "CREATE TABLE IF NOT EXISTS cafe_categories (
        id TEXT PRIMARY KEY,
        name TEXT NOT NULL UNIQUE,
        sort_order INTEGER DEFAULT 0,
        created_at TEXT DEFAULT (datetime('now'))
    )",
)
.execute(pool)
.await?;

sqlx::query(
    "CREATE TABLE IF NOT EXISTS cafe_items (
        id TEXT PRIMARY KEY,
        name TEXT NOT NULL,
        description TEXT,
        category_id TEXT NOT NULL REFERENCES cafe_categories(id),
        selling_price_paise INTEGER NOT NULL,
        cost_price_paise INTEGER NOT NULL,
        is_available BOOLEAN DEFAULT 1,
        created_at TEXT DEFAULT (datetime('now')),
        updated_at TEXT
    )",
)
.execute(pool)
.await?;

sqlx::query("CREATE INDEX IF NOT EXISTS idx_cafe_items_category ON cafe_items(category_id)")
    .execute(pool)
    .await?;
sqlx::query("CREATE INDEX IF NOT EXISTS idx_cafe_items_available ON cafe_items(is_available)")
    .execute(pool)
    .await?;

// Seed default categories (idempotent)
for (name, order) in [("Beverages", 1), ("Snacks", 2), ("Meals", 3)] {
    let id = uuid::Uuid::new_v4().to_string();
    sqlx::query("INSERT OR IGNORE INTO cafe_categories (id, name, sort_order) VALUES (?, ?, ?)")
        .bind(&id)
        .bind(name)
        .bind(order)
        .execute(pool)
        .await?;
}
```

**Step 2: Create crates/racecontrol/src/cafe.rs** with all types and handlers:

Types to define (all derive `Debug, Serialize, Deserialize, sqlx::FromRow` where applicable):
- `CafeCategory` — id: String, name: String, sort_order: i64, created_at: Option<String>
- `CafeItem` — id: String, name: String, description: Option<String>, category_id: String, selling_price_paise: i64, cost_price_paise: i64, is_available: bool, created_at: Option<String>, updated_at: Option<String>
- `CreateCafeItemRequest` (Deserialize only) — name: String, description: Option<String>, category_id: String, selling_price_paise: i64, cost_price_paise: i64
- `UpdateCafeItemRequest` (Deserialize only) — name: Option<String>, description: Option<String>, category_id: Option<String>, selling_price_paise: Option<i64>, cost_price_paise: Option<i64>
- `CreateCafeCategoryRequest` (Deserialize only) — name: String, sort_order: Option<i64>

Handlers to implement (all use `State(state): State<Arc<AppState>>` and return `Result<Json<serde_json::Value>, StatusCode>`):

1. `pub async fn list_cafe_items` — SELECT all items ORDER BY name ASC. Return `{ "items": [...], "total": N }`.
2. `pub async fn create_cafe_item` — Accept `Json<CreateCafeItemRequest>`. Validate: name non-empty, selling_price_paise > 0, cost_price_paise >= 0. Check category_id exists with `SELECT id FROM cafe_categories WHERE id = ?` — return 400 if not found. Generate UUID v4 id. INSERT into cafe_items. Return 201 with `{ "id": id }`.
3. `pub async fn update_cafe_item` — Accept `Path(id)` + `Json<UpdateCafeItemRequest>`. Build dynamic UPDATE SET clause for non-None fields. Always include `updated_at = datetime('now')`. If category_id provided, verify it exists. Return updated item as JSON.
4. `pub async fn delete_cafe_item` — Accept `Path(id)`. DELETE FROM cafe_items WHERE id = ?. Return `{ "ok": true }`. Return 404 if no row affected.
5. `pub async fn toggle_cafe_item_availability` — Accept `Path(id)`. UPDATE cafe_items SET is_available = NOT is_available, updated_at = datetime('now') WHERE id = ?. Return `{ "id": id, "is_available": new_value }`. Fetch the new value after update.
6. `pub async fn list_cafe_categories` — SELECT all categories ORDER BY sort_order ASC, name ASC. Return `{ "categories": [...] }`.
7. `pub async fn create_cafe_category` — Accept `Json<CreateCafeCategoryRequest>`. Use `INSERT OR IGNORE INTO cafe_categories`. Then SELECT by name to return the id (handles both new and existing). Return `{ "id": id, "name": name }`.
8. `pub async fn public_menu` — SELECT items JOIN cafe_categories WHERE is_available = 1 ORDER BY cafe_categories.sort_order, cafe_items.name. Include `category_name` in response by using a custom struct or serde_json::Value. Return `{ "items": [...], "total": N }`.

Error handling: All DB errors map to `StatusCode::INTERNAL_SERVER_ERROR` with `tracing::warn!`. Validation errors return `StatusCode::BAD_REQUEST` with `{ "error": "message" }`. No `.unwrap()` anywhere.
