Create a new module `whatsapp_escalation.rs` in the racecontrol server crate. Register it as `pub mod whatsapp_escalation;` in lib.rs (or main.rs, wherever modules are declared — grep for `pub mod` to find the pattern).

The module has 3 parts: dedup map, WhatsApp send via Bono relay, and INBOX.md fallback.

**1. WhatsAppEscalation struct:**
```rust
pub struct WhatsAppEscalation {
    /// incident_id -> last_sent timestamp for 30-min suppression
    dedup: DashMap<String, Instant>,
    /// HTTP client for Bono relay calls
    client: reqwest::Client,
    /// Uday's WhatsApp number (international, no +)
    uday_number: String,
    /// Bono relay URL
    relay_url: String,
    /// Comms-link repo path for INBOX.md fallback
    comms_link_path: PathBuf,
}
```

**2. Constructor** — `WhatsAppEscalation::new()`:
- `uday_number`: read from env var `ESCALATION_WHATSAPP_NUMBER` with fallback to `"919059833001"` (Uday's customer WhatsApp from MEMORY.md project_whatsapp_phone_mapping)
- `relay_url`: `"http://localhost:8766/relay/exec/run"` (James machine runs comms-link relay)
- `comms_link_path`: `PathBuf::from("C:/Users/bono/racingpoint/comms-link")` — only used as fallback
- `client`: `reqwest::Client::builder().timeout(Duration::from_secs(15)).build()?`

**3. `handle_escalation(&self, payload: EscalationPayload)` — main entry point (async):**

Step A — Dedup check (ESC-03):
- Check `self.dedup` for `payload.incident_id`
- If found AND `Instant::now() - stored_instant < Duration::from_secs(1800)` (30 min): log at info level "Suppressing duplicate escalation for incident {id}" and return early
- Otherwise insert/update the entry with `Instant::now()`
- Periodically clean stale entries: if `dedup.len() > 100`, remove entries older than 30 min (do this inline, not in a background task — escalations are rare)

Step B — Format WhatsApp message (ESC-02):
```
let message = format!(
    "*Tier 5 Escalation*\n\
     Severity: {}\n\
     Pod: {}\n\
     Issue: {}\n\
     AI tried: {}\n\
     Impact: {}\n\
     Dashboard: {}",
    payload.severity,
    payload.pod_id,
    payload.summary,
    payload.actions_tried.join(", "),
    payload.impact,
    payload.dashboard_url,
);
```

Step C — Send via Bono relay (ESC-01, ESC-04):
- Build JSON body: `{"command":"whatsapp_send","reason":"Tier 5 escalation","params":{"number":"<uday_number>","message":"<escaped_message>"}}`
- Write JSON to a temp file (standing rule: Git Bash JSON via file), but since this is Rust with reqwest, we can use `.json()` directly — no shell escaping issues
- POST to `self.relay_url` with Content-Type: application/json
- Timeout: 15 seconds
- On success (200 OK): log at warn level (Tier 5 is always noteworthy) "WhatsApp escalation sent for incident {id} to {number}"
- On failure: log error and fall through to Step D

Step D — Fallback to INBOX.md (ESC-04):
- Only runs if Step C failed (relay down, timeout, non-200 response)
- Compute IST timestamp: `Utc::now() + chrono::Duration::hours(5) + chrono::Duration::minutes(30)`
- Format entry:
  ```
  ## {YYYY-MM-DD HH:MM IST} -- from james (auto-escalation)

  **Tier 5 Escalation** — incident {incident_id}
  - Severity: {severity}
  - Pod: {pod_id}
  - Issue: {summary}
  - AI tried: {actions_tried}
  - Impact: {impact}
  - Dashboard: {dashboard_url}

  WhatsApp delivery FAILED — this is the fallback notification.
  ```
- Append to `{comms_link_path}/INBOX.md` using `tokio::fs::OpenOptions::new().append(true).open()`
- Then run `git -C {comms_link_path} add INBOX.md && git -C {comms_link_path} commit -m "auto-escalation: {incident_id}" && git -C {comms_link_path} push` via `tokio::process::Command`
- If git push also fails, log at error level "CRITICAL: Both WhatsApp and INBOX.md fallback failed for incident {id}" — this is the last resort

**4. Dedup cleanup:**
Add a method `cleanup_stale(&self)` that removes entries older than 30 min from the DashMap. Call it at the start of `handle_escalation` if `dedup.len() > 50`. Use `dedup.retain(|_, v| v.elapsed() < Duration::from_secs(1800))`.

**Dependencies to add to Cargo.toml** (if not already present in racecontrol crate):
- `dashmap` — already in workspace (used by rc-agent)
- `reqwest` — likely already present for server HTTP calls
- Check with `grep reqwest crates/racecontrol/Cargo.toml`

IMPORTANT: No `.unwrap()`. All errors handled with `match` or `if let Err(e)`. The escalation path must never panic — a panic here means the server crashes during the worst moment (when pods are already failing).
