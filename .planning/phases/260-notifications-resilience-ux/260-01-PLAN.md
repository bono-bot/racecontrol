    1. In db/mod.rs run_migrations(), add migration:
       ```sql
       CREATE TABLE IF NOT EXISTS notification_outbox (
         id TEXT PRIMARY KEY,
         recipient TEXT NOT NULL,
         channel TEXT NOT NULL CHECK(channel IN ('whatsapp','sms','screen')),
         payload TEXT NOT NULL,
         status TEXT NOT NULL DEFAULT 'pending' CHECK(status IN ('pending','sent','delivered','failed','exhausted')),
         retry_count INTEGER NOT NULL DEFAULT 0,
         max_retries INTEGER NOT NULL DEFAULT 3,
         next_retry_at TEXT DEFAULT (datetime('now')),
         fallback_channel TEXT,
         fallback_token TEXT,
         context_type TEXT,
         context_id TEXT,
         created_at TEXT DEFAULT (datetime('now')),
         updated_at TEXT DEFAULT (datetime('now'))
       )
       ```
       Add index: `CREATE INDEX IF NOT EXISTS idx_notif_outbox_pending ON notification_outbox(status, next_retry_at)`

    2. Create crates/racecontrol/src/notification_outbox.rs:
       - `pub async fn enqueue_notification(db: &SqlitePool, recipient: &str, channel: &str, payload: &str, context_type: Option<&str>, context_id: Option<&str>) -> Result<String>` — INSERT into outbox, returns notification id.
       - `pub async fn enqueue_otp_notification(db: &SqlitePool, phone: &str, otp_message: &str) -> Result<String>` — convenience wrapper: channel='whatsapp', fallback_channel='screen', generates a one-time UUID fallback_token.
       - `pub async fn get_otp_by_fallback_token(db: &SqlitePool, token: &str) -> Result<Option<String>>` — returns payload if token valid and notification status='failed' (for on-screen OTP display). Marks token consumed (UPDATE status='delivered').
       - `pub async fn notification_worker_task(state: Arc<AppState>)` — background loop every 10s:
         a. SELECT WHERE status='pending' AND next_retry_at <= datetime('now') ORDER BY created_at LIMIT 20
         b. For each: attempt delivery via channel (whatsapp → send_whatsapp, screen → skip, sms → log placeholder)
         c. On success: UPDATE status='sent', updated_at=now
         d. On failure: INCREMENT retry_count. If retry_count >= max_retries AND fallback_channel IS NOT NULL: UPDATE channel=fallback_channel, status='pending', retry_count=0 (UX-02 fallback chain). If retry_count >= max_retries AND no fallback: UPDATE status='exhausted'.
         e. Exponential backoff: next_retry_at = datetime('now', '+' || (10 * 2^retry_count) || ' seconds')
       - Log lifecycle: startup, first process, exit (standing rule: long-lived tasks).

    3. Add GET /customer/otp-fallback/{token} to public_routes in routes.rs — calls get_otp_by_fallback_token, returns JSON {otp: "..."} or 404. This is the on-screen OTP display endpoint (UX-02).

    4. Register module in lib.rs (`pub mod notification_outbox;`). Spawn notification_worker_task in main.rs alongside whatsapp_alerter_task.

    5. IMPORTANT: Do NOT use .unwrap() anywhere. Use `?` or match. Notification worker failures must log WARN, never panic.
