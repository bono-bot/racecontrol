  <action>
Create `audit/lib/parallel.sh` with these functions (all `export -f`):

1. **`semaphore_acquire()`** -- File-based lock using `$RESULT_DIR/.sem/` directory. Creates numbered lock files (slot-0 through slot-3). Loop: check if any of 4 slots is free (lock file doesn't exist or is stale > 30s). If free, create lock file with `$$` PID. If all full, `sleep 0.2` and retry. Use `mkdir` for atomic lock creation (mkdir is atomic on all filesystems): `mkdir "$RESULT_DIR/.sem/slot-${i}.lock" 2>/dev/null` -- if it succeeds, slot acquired. Write `$$` into `slot-${i}.lock/pid`. Max 4 slots (0-3). Sets `ACQUIRED_SLOT` variable for caller.

2. **`semaphore_release(slot_num)`** -- Remove the lock directory: `rm -rf "$RESULT_DIR/.sem/slot-${slot_num}.lock"`.

3. **`parallel_pod_loop(phase_fn)`** -- The main parallel dispatcher. Takes a function name as argument. That function signature must be: `phase_fn(ip, host)`. For each pod IP in `$PODS`:
   - `sleep 0.2` (200ms stagger to avoid ARP flood)
   - Launch background subshell: `( semaphore_acquire; local slot=$ACQUIRED_SLOT; "$phase_fn" "$ip" "$host"; semaphore_release "$slot" ) &`
   - Store the PID in an array
   After all launched, call `wait` for all PIDs. No output interleaving because each phase_fn writes to its own `$RESULT_DIR/phase-XX-host.json` via `emit_result`.

4. **`wait_all_jobs()`** -- Simple wrapper: loops over stored PIDs and calls `wait $pid`. Returns max exit code (though phases always return 0).

Implementation details:
- `set -u` and `set -o pipefail` at top (no `set -e`)
- Header comment: `# audit/lib/parallel.sh -- Parallel pod execution with 4-connection semaphore`
- `ACQUIRED_SLOT` variable set by `semaphore_acquire` for the caller to pass to `semaphore_release`
- Stale lock detection: if `$RESULT_DIR/.sem/slot-N.lock/pid` file exists and `kill -0 $(cat pid) 2>/dev/null` fails, remove the stale lock
- The `parallel_pod_loop` function derives `host` from IP using the established pattern: `host="pod-$(echo "$ip" | sed 's/192\.168\.31\.//')"`
- All 4 functions exported with `export -f`
- `MAX_CONCURRENT=4` as a variable at the top (not hardcoded in loop)
- `STAGGER_MS=0.2` as a variable at the top
- Initialize semaphore dir in parallel_pod_loop: `mkdir -p "$RESULT_DIR/.sem"`
