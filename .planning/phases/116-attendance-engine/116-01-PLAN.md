Output: New `attendance` module with SQLite table, broadcast subscriber, and 5-minute dedup window. Pipeline modified to broadcast RecognitionResult events.

1. Create `crates/rc-sentry-ai/src/attendance/engine.rs`:
   - `pub async fn run(mut rx: tokio::sync::broadcast::Receiver<RecognitionResult>, db_path: String, config: AttendanceConfig)`:
     - Maintain `HashMap<i64, Instant>` for cross-camera dedup (key = person_id, value = last logged time).
     - On each received `RecognitionResult`:
       a. Check dedup map: if `person_id` was logged within `config.dedup_window_secs` seconds, skip (log at debug level).
       b. Otherwise, compute `day` as `result.timestamp.with_timezone(&chrono_tz::Asia::Kolkata).format("%Y-%m-%d").to_string()` (IST day boundary per user timezone requirement). NOTE: Use `chrono::FixedOffset::east_opt(5 * 3600 + 1800).unwrap()` instead of chrono_tz to avoid adding a dependency -- construct IST offset manually.
       c. Call `spawn_blocking` with `attendance::db::insert_attendance(conn, person_id, person_name, camera_id, confidence, day)`.
       d. Update dedup map with current Instant.
       e. Log at info level: `"attendance logged"` with person_name, camera, day.
     - Every 5 minutes, clean dedup map entries older than `dedup_window_secs`.
     - Use `tokio::select!` with `rx.recv()` and a 5-minute interval timer for cleanup.
   - Open SQLite connection once at start via `spawn_blocking`, call `attendance::db::create_tables`, then keep the connection path for per-event inserts (open/close per insert via spawn_blocking, same pattern as enrollment).

2. Wire in `crates/rc-sentry-ai/src/main.rs`:
   - After the gallery/tracker init section, create the broadcast channel:
     ```rust
     let (recognition_tx, _) = tokio::sync::broadcast::channel::<recognition::types::RecognitionResult>(256);
     ```
   - Pass `Some(recognition_tx.clone())` as the new `recognition_tx` parameter when spawning `detection::pipeline::run()` for each camera.
   - If `config.attendance.enabled`, subscribe and spawn the attendance engine:
     ```rust
     if config.attendance.enabled {
         let attendance_rx = recognition_tx.subscribe();
         let att_db_path = config.recognition.gallery_db_path.clone();
         let att_config = config.attendance.clone();
         tokio::spawn(async move {
             attendance::engine::run(attendance_rx, att_db_path, att_config).await;
         });
         tracing::info!("attendance engine started");
     }
     ```
   - The attendance tables are in the SAME SQLite database as faces (gallery_db_path = `C:\RacingPoint\data\faces.db`). This avoids a second db file.
