    **2. Create security_headers_layer() function** (can be a standalone fn or inline):
    ```rust
    fn security_headers_layer() -> HelmetLayer {
        use tower_helmet::header::ContentSecurityPolicy;
        let mut directives = std::collections::HashMap::new();
        directives.insert("default-src", vec!["'self'"]);
        directives.insert("script-src", vec!["'self'"]);
        directives.insert("style-src", vec!["'self'", "'unsafe-inline'"]);  // Tailwind needs unsafe-inline
        directives.insert("img-src", vec!["'self'", "data:"]);
        directives.insert("connect-src", vec!["'self'", "ws:", "wss:"]);    // WebSocket connections
        directives.insert("frame-ancestors", vec!["'none'"]);
        directives.insert("base-uri", vec!["'self'"]);
        directives.insert("form-action", vec!["'self'"]);

        let csp = ContentSecurityPolicy {
            directives,
            ..Default::default()
        };

        HelmetLayer::with_defaults().enable(csp)
    }
    ```

    NOTE: HSTS max-age MUST be 300 (5 minutes) for initial deploy. NOT 31536000. Research explicitly warns about HSTS lockout during testing (Pitfall 4). Increase only after 1 week of stable operation.

    If tower-helmet 0.3 API does not match exactly (e.g., HelmetLayer does not have `.enable()` or ContentSecurityPolicy has different fields), fall back to manual `tower_http::set_header::SetResponseHeaderLayer` for the 4 required headers:
    - `Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'self' ws: wss:; frame-ancestors 'none'; base-uri 'self'; form-action 'self'`
    - `X-Frame-Options: DENY`
    - `X-Content-Type-Options: nosniff`
    - `Strict-Transport-Security: max-age=300; includeSubDomains` (SHORT max-age for testing!)

    **3. Add HelmetLayer to middleware stack** — Insert BEFORE the CorsLayer in the `.layer()` chain:
    ```rust
    .layer(security_headers_layer())  // NEW — security response headers
    .layer(
        CorsLayer::new()
        // ...
    )
    ```
    In Axum, `.layer()` calls are outermost-first. Placing helmet before cors in code means helmet wraps cors, so CORS preflight responses also get security headers.

    ```rust
    .allow_origin(AllowOrigin::predicate(|origin: &HeaderValue, _| {
    ```

    **5. Wire HTTPS listener** after the existing HTTP bind. Replace the current server startup block (lines 573-582) with:

    ```rust
    // Start HTTP server
    let listener = tokio::net::TcpListener::bind(&bind_addr).await?;
    tracing::info!("RaceControl HTTP on http://{}", bind_addr);
    tracing::info!("Dashboard:    http://{}/", bind_addr);
    tracing::info!("API:          http://{}/api/v1/health", bind_addr);
    tracing::info!("Agent WS:     ws://{}/ws/agent", bind_addr);
    tracing::info!("Dashboard WS: ws://{}/ws/dashboard", bind_addr);
    tracing::info!("AI WS:        ws://{}/ws/ai", bind_addr);

    // Start HTTPS server (if tls_port configured)
    if let Some(tls_port) = state.config.server.tls_port {
        let tls_config = tls::load_or_generate_rustls_config(
            &state.config.server.host,
            state.config.server.cert_path.as_deref(),
            state.config.server.key_path.as_deref(),
        ).await?;
        let https_addr: std::net::SocketAddr = format!("{}:{}", state.config.server.host, tls_port)
            .parse()
            .expect("invalid tls_port address");
        let https_app = app.clone();
        tracing::info!("RaceControl HTTPS on https://{}", https_addr);
        tokio::spawn(async move {
            if let Err(e) = axum_server::bind_rustls(https_addr, tls_config)
                .serve(https_app.into_make_service())
                .await
            {
                tracing::error!("HTTPS listener failed: {}", e);
            }
        });
    }

    // HTTP listener (blocking — keeps main alive)
    axum::serve(listener, app.into_make_service_with_connect_info::<SocketAddr>()).await?;
    ```

    NOTE on ConnectInfo: `axum_server::bind_rustls().serve()` uses `.into_make_service()` (not `into_make_service_with_connect_info`). This means `ConnectInfo<SocketAddr>` extraction will NOT work on the HTTPS port. The HTTP port retains `into_make_service_with_connect_info` for tower_governor rate limiting. If rate limiting is needed on HTTPS too, this can be addressed in a later plan by switching to header-based IP extraction for HTTPS. For now, the primary HTTPS consumers are customer browsers (not staff routes that are rate-limited).


    With protocol-aware version:
    ```typescript
    const API_BASE =
      process.env.NEXT_PUBLIC_API_URL ||
      (typeof window !== "undefined"
        ? `${window.location.protocol}//${window.location.host}`
        : "http://localhost:8080");
    ```

    This change means:
    - When served from `http://192.168.31.23:8080` (pod kiosk via HTTP) -> API_BASE = `http://192.168.31.23:8080` (same as before)
    - When served from `https://192.168.31.23:8443` (customer WiFi via HTTPS) -> API_BASE = `https://192.168.31.23:8443` (no mixed content)
    - SSR fallback stays `http://localhost:8080` (server-side rendering)

    NOTE: `window.location.host` includes the port (e.g., `192.168.31.23:8443`), unlike `window.location.hostname` which does not. This is correct because the API is served on the same port as the page (both HTTP 8080 and HTTPS 8443 serve the same Axum router including the kiosk proxy fallback).
