**Cart helper functions:**
- `addToCart(item: CafeMenuItem)` — if item.out_of_stock, do nothing (button should be disabled). If already in cart, increment quantity (but not beyond stock_quantity for countable items). If new, add with quantity 1.
- `removeFromCart(itemId: string)` — remove entirely
- `updateQuantity(itemId: string, qty: number)` — if qty <= 0, remove. For countable items, cap at stock_quantity.
- `cartTotal` — computed: sum of item.selling_price_paise * quantity
- `cartItemCount` — computed: sum of quantities

**Menu item card changes:**
- For each item card, add an "Add to Cart" button (Racing Red `#E10600` bg, white text)
- If `item.out_of_stock === true`, show "Out of Stock" badge (grey bg) and disable the add button
- If item is already in cart, show quantity +/- controls instead of "Add" button
- Use the existing card/image layout — just add the interactive elements

**Cart sidebar/bottom bar:**
- Show a floating bottom bar when cart is non-empty: "{N} items - Rs. {total}" with "View Cart" button
- "View Cart" opens a slide-up panel or modal showing cart items with quantities, line totals, and a "Checkout" button
- Each cart item shows: name, quantity +/-, unit price, line total, remove button

**Checkout flow:**
1. When "Checkout" is tapped, fetch wallet balance: `api.wallet()` to get current balance
2. Show order summary: items, total, wallet balance. If balance < total, show warning "Insufficient balance (Rs. {balance/100}). Please top up." and disable submit button.
3. "Place Order" button calls `api.placeCafeOrder(cart.map(c => ({ item_id: c.item.id, quantity: c.quantity })))`
4. While ordering, show spinner and disable button (prevent double-tap)
5. On success: clear cart, show confirmation screen with receipt_number, total, new balance. "Order another" button returns to menu.
6. On error: show error message from API response (out of stock, insufficient balance, etc.). Keep cart intact so user can adjust.

- Card bg `#222222`, border `#333333`
- Asphalt Black `#1A1A1A` for backgrounds
- Use Tailwind classes consistent with existing page

- Use `any` type anywhere
