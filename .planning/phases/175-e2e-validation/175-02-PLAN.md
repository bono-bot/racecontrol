Create `test/e2e/run-cross-sync.sh`. This script cannot automate the UI actions (they require browser interaction), but it provides:

1. A printed walkthrough — numbered steps for the tester to follow
2. Curl state-check commands run between browser steps to verify server-side state changed
3. A mechanism to mark each test PASS/FAIL by asking the tester for input (read -p)
4. Output appended to the existing E2E-TEST-RESULTS-{date}.md if it exists, or a new file

**Structure:**

```bash
#!/usr/bin/env bash
# run-cross-sync.sh — Cross-cutting real-time sync tests for Phase 175 E2E
# Requires: two browser windows open (POS at :3200, Kiosk at :3300)
# Usage: bash test/e2e/run-cross-sync.sh
# This script guides you through the 21 cross-cutting tests in sections 3.1-3.4

set -euo pipefail

SERVER="192.168.31.23"
DATE=$(date '+%Y-%m-%d')
REPORT="test/e2e/E2E-TEST-RESULTS-${DATE}.md"
PASS=0
FAIL=0
```

**Pre-flight:**
- Call check-health.sh — abort if services down
- Print setup instructions:
  - "Open POS in Browser A: http://192.168.31.23:3200"
  - "Open Kiosk in Browser B: http://192.168.31.23:3300"
  - "Log into POS with staff PIN"
  - "Have a test session ready (pod not in use)"
  - "Press Enter when ready..."
  - Wait for read -p

**For each of the 5 section 3.2 tests, script a sequence like:**

```
=== TEST 3.2.1: Start session on POS → Kiosk reflects occupied ===

Step 1: In Browser A (POS), go to /billing
Step 2: Click "Start" on an idle pod. Complete the start session modal.
Step 3: Note which pod number you started (e.g. Pod 3).

[Script curls fleet/health and shows current pod states]
echo "Current pod states (after you start session):"
curl -s http://${SERVER}:8080/api/v1/fleet/health | python3 -m json.tool 2>/dev/null || \
  curl -s http://${SERVER}:8080/api/v1/fleet/health

Step 4: In Browser B (Kiosk), verify Pod 3 shows as occupied (red/in-use).

read -p "Did the Kiosk show pod as occupied immediately? [pass/fail]: " result
```

Record result to PASS/FAIL counters and append to report.

Implement all 5 tests in section 3.2 this way.

**Section 3.1 (Responsiveness) — 5 tests:**
Print instructions for each (check at 1920x1080, verify sidebar fits, modal sizing). Prompt `read -p` for each.

**Section 3.3 (Error Handling) — 5 tests:**

**Section 3.4 (Edge Cases) — 6 tests:**

**End of script:**
Print total PASS/FAIL. Append section 3.x results to report file.
