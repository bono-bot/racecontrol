**Header / setup:**
```bash
#!/usr/bin/env bash
# run-e2e.sh — RaceControl E2E test runner
# Usage: bash test/e2e/run-e2e.sh [--pos-only|--kiosk-only|--api-only]
# Requires: curl, access to 192.168.31.23 (server must be running)
# Output: E2E-TEST-RESULTS-{date}.md in test/e2e/

set -euo pipefail

SERVER="192.168.31.23"
DATE=$(date '+%Y-%m-%d')
REPORT="test/e2e/E2E-TEST-RESULTS-${DATE}.md"
PASS=0
FAIL=0
SKIP=0
TIMEOUT=5
FILTER="${1:-}"
```

**Pre-flight check — call check-health.sh:**
Before running any tests, source or call check-health.sh. If it exits non-zero, print "ABORTED: services not healthy — run check-health.sh to diagnose" and exit 1.
```bash
echo "=== Pre-flight health check ==="
if ! bash C:/Users/bono/racingpoint/deploy-staging/check-health.sh; then
  echo "ABORTED: services not healthy. Fix before running E2E tests."
  exit 1
fi
echo ""
```

**Helper functions:**

1. `test_http_status(id, description, url, expected_status)` — curl the URL with -s -o /dev/null -w "%{http_code}", compare to expected (usually 200). Log PASS/FAIL with test ID and description. Append result line to $REPORT.

2. `test_api_json(id, description, url, grep_pattern)` — curl the URL, pipe response body through grep -q for the pattern. PASS if pattern found. FAIL otherwise. Useful for confirming JSON fields like `"status"` or `"pod_number"`.

3. `manual_test(id, description, expected)` — logs a MANUAL entry in the report with an unchecked checkbox `- [ ]`. Does not increment PASS/FAIL, increments SKIP.
