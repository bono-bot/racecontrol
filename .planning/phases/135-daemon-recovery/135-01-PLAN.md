Create C:\Users\bono\.claude\james_watchdog.ps1 using the Write tool. This is a PowerShell script (not a .bat — Write tool is fine for .ps1).

The script must:

1. Check daemon health via HTTP GET http://localhost:8766/relay/health with a 5-second timeout. A 200 response means the daemon is up. Anything else (connection refused, timeout, non-200) means it is down.

2. If the daemon is DOWN:
   a. Log a timestamped message to C:\Users\bono\.claude\comms-watchdog.log
   b. Send a crash notification to Bono via send-message.js:
      Start-Process "C:\Program Files\nodejs\node.exe" -WorkingDirectory "C:\Users\bono\racingpoint\comms-link" -ArgumentList "send-message.js", "[CRASH] CommsLink daemon down on James - restarting" -NoNewWindow -Wait -PassThru
      (Set env vars COMMS_PSK and COMMS_URL before calling)
   c. Start the daemon by running start-comms-link.bat:
      Start-Process "C:\Users\bono\racingpoint\comms-link\start-comms-link.bat" -WindowStyle Hidden
   d. Wait 10 seconds for the daemon to come up.
   e. Re-check health. If health returns 200:
      - Log "CommsLink daemon recovered" to the log file
      - Send recovery notification to Bono via send-message.js:
        "[RECOVERED] CommsLink daemon back online on James"
   f. If health still fails after 10s, log "CommsLink daemon failed to recover" and exit (do not loop — Task Scheduler triggers again in 2 min).

3. If the daemon is UP: log a brief "OK" line with timestamp and exit cleanly.

Implementation details:
- Use try/catch around the health check (Invoke-WebRequest throws on connection refused)
- Invoke-WebRequest with -UseBasicParsing -TimeoutSec 5 -ErrorAction SilentlyContinue
- Check $response.StatusCode -eq 200 to determine health
- Set $env:COMMS_PSK and $env:COMMS_URL before running send-message.js
- Log format: "YYYY-MM-DD HH:mm:ss IST — [status]"
- Log file: C:\Users\bono\.claude\comms-watchdog.log (append mode)
- Keep the script idempotent: running it when daemon is healthy must be a no-op (just log OK)

Do NOT use Start-Process -Wait for start-comms-link.bat (it never exits — it uses start /min which backgrounds). Just Start-Process it and sleep 10.

This Node.js script (ESM, same pattern as register-watchdog.js) registers a Task Scheduler task named "CommsLink-DaemonWatchdog" that:
- Runs C:\Users\bono\.claude\james_watchdog.ps1 via: powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\Users\bono\.claude\james_watchdog.ps1"
- Triggers: ONLOGON + repeat every 2 minutes indefinitely (using /sc MINUTE /mo 2 /du 9999:59 is NOT how schtasks works for repeat after logon; instead use /sc ONLOGON with a separate /sc MINUTE trigger OR use schtasks XML. The simplest approach: create the task with /sc ONLOGON for immediate start on boot, then add a MINUTE repeat trigger via a second schtasks call)

ACTUALLY: schtasks /create supports only one trigger type per command. To get "run on logon AND repeat every 2 min", use the XML approach or use /sc MINUTE /mo 2 alone (simpler — Task Scheduler runs it every 2 min, including after logon within the first 2 min).

Use /sc MINUTE /mo 2 (runs every 2 minutes, 24/7). This achieves detection-and-restart within 2 minutes of a crash, satisfying RECOV-01 (within 30s is for the restart action itself — the script restarts the daemon as soon as it detects it down; the watchdog polling interval is 2 min).
