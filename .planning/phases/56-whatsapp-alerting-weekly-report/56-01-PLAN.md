**3. Create crates/racecontrol/src/whatsapp_alerter.rs:**

This module implements:
- `pub async fn whatsapp_alerter_task(state: Arc<AppState>, mut error_rate_rx: broadcast::Receiver<()>)` — the main async task
- `struct P0State` — internal tracking state (NOT shared with AppState):
  ```rust
  struct P0State {
      all_pods_offline_since: Option<Instant>,
      last_all_pods_alert: Option<Instant>,
      error_rate_since: Option<Instant>,
      last_error_rate_alert: Option<Instant>,
      error_rate_last_signal: Option<Instant>,
  }
  ```
- `async fn send_whatsapp(config: &Config, message: &str)` — Evolution API call, identical to billing.rs L2243-2270 pattern. Uses `config.auth.evolution_url`, `config.auth.evolution_api_key`, `config.auth.evolution_instance` for Evolution API connection. Uses `config.alerting.uday_phone` for recipient number. 5-second timeout, best-effort (warn on failure, never panic/return error).
- `fn ist_now_string() -> String` — returns IST timestamp using `chrono::Utc::now().with_timezone(&chrono_tz::Asia::Kolkata).format("%d %b %Y %H:%M IST").to_string()`
- `async fn count_online_pods(state: &Arc<AppState>) -> (usize, usize)` — returns (online_count, total_count). Online = present in `state.agent_senders` map AND sender is not closed (`!sender.is_closed()`). Total = `state.pods.read().await.len()`.
- `async fn record_incident(db: &SqlitePool, alert_type: &str, pod_count: Option<i64>, description: &str)` — INSERT into alert_incidents table with UUID id (use `uuid::Uuid::new_v4().to_string()`), started_at = UTC now
- `async fn resolve_incident(db: &SqlitePool, alert_type: &str)` — UPDATE alert_incidents SET resolved_at = UTC now WHERE alert_type = ? AND resolved_at IS NULL (most recent unresolved)

The `whatsapp_alerter_task` function:
1. Checks `state.config.alerting.enabled` — if false, return immediately (log info "WA alerting disabled")
2. Checks `state.config.alerting.uday_phone.is_some()` — if None, return immediately (log warn "no uday_phone configured")
3. Subscribes to `state.bono_event_tx.subscribe()` for pod events
4. Runs a `tokio::select!` loop with three branches:
   - `bono_rx.recv()` — on PodOffline: sleep 2s debounce, then call `count_online_pods()`. If online==0 AND total>0 AND (last_all_pods_alert is None or elapsed > cooldown_secs): fire alert `"[RP ALERT] All Pods Offline - All {total} pods lost WS connection. {total} pods affected. {ist_now_string()}"`, set `all_pods_offline_since = Some(Instant::now())`, set `last_all_pods_alert = Some(Instant::now())`, call `record_incident(db, "all_pods_offline", total, "All pods lost WebSocket connection")`. On PodOnline: call `count_online_pods()`. If online==total AND total>0 AND `all_pods_offline_since.is_some()`: fire resolved `"[RP RESOLVED] All Pods Offline cleared. All {total} pods online. Duration: {minutes}m. {ist_now_string()}"` where minutes = `all_pods_offline_since.unwrap().elapsed().as_secs() / 60`, call `resolve_incident(db, "all_pods_offline")`, clear `all_pods_offline_since = None`.
   - `error_rate_rx.recv()` — on Ok(()): if last_error_rate_alert is None or elapsed > cooldown_secs: fire alert `"[RP ALERT] High Error Rate - Error rate threshold exceeded on racecontrol. {ist_now_string()}"`, set `error_rate_since = Some(Instant::now())`, set `last_error_rate_alert = Some(Instant::now())`, set `error_rate_last_signal = Some(Instant::now())`, call `record_incident(db, "error_rate", None, "Error rate threshold breach")`.
   - `tokio::time::sleep(Duration::from_secs(60))` — periodic check: if `error_rate_since.is_some()` AND `error_rate_last_signal` elapsed > 300 seconds (5 min): fire resolved `"[RP RESOLVED] High Error Rate cleared. No threshold breach for 5 minutes. Duration: {minutes}m. {ist_now_string()}"`, call `resolve_incident(db, "error_rate")`, clear `error_rate_since = None`, clear `error_rate_last_signal = None`.
5. Use `tracing::{info, warn}` for all logging with `target = "whatsapp_alerter"`.
