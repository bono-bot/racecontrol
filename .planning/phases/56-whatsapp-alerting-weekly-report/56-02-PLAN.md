**3. Create crates/weekly-report/src/main.rs:**

The binary does the following:
1. Reads DB path from env var `RACECONTROL_DB` (default: `C:\RacingPoint\racecontrol.db`)
2. Reads email script path from env var `EMAIL_SCRIPT` (default: `C:\RacingPoint\send_email.js`)
3. Reads recipient from env var `EMAIL_RECIPIENT` (default: `usingh@racingpoint.in`)
4. Opens SQLite in **read-only mode**: `sqlite:C:\RacingPoint\racecontrol.db?mode=ro`
5. Computes IST week boundaries: previous Monday 00:00 IST to previous Sunday 23:59 IST
   - IST = UTC+5:30, so Monday 00:00 IST = Sunday 18:30 UTC
   - Sunday 23:59 IST = Sunday 18:29 UTC (of following week)
   - Use chrono + chrono_tz to compute: `let now_ist = Utc::now().with_timezone(&Asia::Kolkata); let today = now_ist.date_naive(); let days_since_monday = today.weekday().num_days_from_monday(); let this_monday = today - chrono::Duration::days(days_since_monday as i64); let prev_monday = this_monday - chrono::Duration::days(7); let prev_sunday = this_monday - chrono::Duration::days(1);`
   - Convert to UTC for SQL: `prev_monday at 00:00 IST = prev_monday_naive.and_hms(0,0,0) - 5h30m` etc.
6. Queries:

**Total sessions:**
```rust
let total_sessions: (i64,) = sqlx::query_as(
```

**Total credits billed (paise):**

```rust
let total_paise: (i64,) = sqlx::query_as(

    ), 0) FROM billing_sessions
     WHERE started_at >= ? AND started_at < ?"
```

```rust
     WHERE sampled_at >= ? AND sampled_at < ?
```

**Incidents:**
