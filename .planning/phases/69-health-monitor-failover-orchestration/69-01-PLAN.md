**Methods:**
- `start()` â€” sets interval calling `#probe()` every PROBE_INTERVAL_MS
- `stop()` â€” clears interval

**Probe cycle (`#probe()`):**
1. LAN primary: HTTP GET `http://192.168.31.23:8080/api/v1/health` with 3s timeout
2. If primary fails, LAN secondary: HTTP GET `http://192.168.31.23:8090/ping` with 3s timeout
3. `lanOk = primary || secondary` â€” IMPORTANT: do NOT count each probe as separate failure
4. If `lanOk` is false, Tailscale fallback: HTTP GET `http://100.71.226.83:8090/ping` with 3s timeout
5. `cycleOk = lanOk || tailscaleOk`
6. If `cycleOk`: reset `consecutiveFailures` to 0, increment `consecutiveSuccesses`
7. If `!cycleOk`: reset `consecutiveSuccesses` to 0, increment `consecutiveFailures`
8. State transitions:
   - `consecutiveFailures >= DOWN_THRESHOLD && state !== 'down'` => state = `'down'`
   - `consecutiveFailures > 0 && state === 'healthy'` => state = `'degraded'`
   - `consecutiveFailures === 0 && state !== 'healthy'` => state = `'healthy'`
9. On state change: `console.log('[HEALTH] State: ${prev} -> ${next}')` and `this.emit('state_change', { from, to })`
10. On transition TO `'down'` (only once): `this.emit('server_down')`

**`#httpGet(url)` helper:** Use `node:http` (NOT fetch, NOT undici). Returns Promise<boolean>:
```javascript
const req = http.get(url, { timeout: PROBE_TIMEOUT_MS }, (res) => {
  resolve(res.statusCode >= 200 && res.statusCode < 500);

req.on('timeout', () => { req.destroy(); resolve(false); });
```

CRITICAL: The cycle result is ONE boolean per 5s tick. `consecutiveFailures` increments by 1 per tick, NOT per probe attempt. This ensures 12 failures = 60s, not 30s.
