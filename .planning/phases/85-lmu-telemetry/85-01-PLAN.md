Create `crates/rc-agent/src/sims/lmu.rs` following the iRacing adapter pattern closely but with rF2-specific differences:

**Struct: LmuAdapter**
Fields: pod_id(String), connected(bool), scoring_shm(Option<ShmHandle>) [cfg(windows)], telemetry_shm(Option<ShmHandle>) [cfg(windows)], last_lap_count(i16), last_session_type(i32), pending_lap(Option<LapData>), current_track(String), current_car(String), current_session_type(SessionType), first_read(bool).

No VarOffsets needed — rF2 uses fixed struct layout, not dynamic variable lookup.

**ShmHandle:** Reuse exact pattern from iRacing adapter — winapi OpenFileMappingW + MapViewOfFile wrapper with Drop impl (UnmapViewOfFile + CloseHandle). Need TWO handles: one for `$rFactor2SMMP_Scoring$`, one for `$rFactor2SMMP_Telemetry$`.

**connect():** Open both shared memory maps. If either fails, close the other and return Err with descriptive message mentioning rF2SharedMemoryMapPlugin. Set connected=true on success.

**read_telemetry():** Read from Telemetry buffer. Use torn-read guard (mVersionUpdateBegin at offset 0, mVersionUpdateEnd at offset 4 — if not equal, retry up to 3 times). Find player vehicle in Scoring buffer (scan vehicle array for m_is_player==1). Return TelemetryFrame with speed, throttle, brake, gear, rpm from telemetry data. On first_read, snapshot last_lap_count and set first_read=false.

**Struct offsets:** Compute from rF2Data.cs (C# StructLayout Sequential). The RESEARCH.md documents the key fields. Use named constants for offsets (e.g., `const SCORING_HEADER_SIZE: usize = 12;`). Read `rF2Data.cs` from the plugin repo to compute exact offsets. Document the source version in comments.

CRITICAL: The executor MUST read rF2Data.cs from TheIronWolfModding/rF2SharedMemoryMapPlugin to compute exact byte offsets for:
- rF2ScoringInfo size (to find where vehicle array starts)
- rF2VehicleScoring field offsets: mIsPlayer, mControl, mTotalLaps, mLastLapTime, mLastSector1, mLastSector2, mSector, mInPits
- rF2ScoringInfo field offsets: mSession, mGamePhase, mNumVehicles

Use `std::ptr::read_unaligned` for all reads (same as iRacing). All pointer arithmetic is unsafe.

**poll_lap_completed():** Take pending_lap via `.take()` — same as iRacing.

**Lap detection in read_telemetry():** When m_total_laps > last_lap_count AND !first_read AND m_last_lap_time > 0.0:
- Derive sectors: S1 = mLastSector1 * 1000, S2 = (mLastSector2 - mLastSector1) * 1000, S3 = (mLastLapTime - mLastSector2) * 1000
- Build LapData with sim_type: SimType::LeMansUltimate
- Store in pending_lap
- Update last_lap_count

**Session transition:** If mSession != last_session_type, reset: last_lap_count=0, first_read=true, pending_lap=None. Log the transition.

**read_is_on_track():** MUST be trait override inside `impl SimAdapter for LmuAdapter` (NOT inherent method only). Delegates to inherent helper read_is_on_track_from_shm(). Returns Some(true) when scoring buffer readable AND player vehicle found (m_is_player==1) AND mGamePhase >= 4 (Countdown or later). Non-windows: returns None.

**sector_times_ms():** Free function (not method). Takes (last_lap_s: f64, last_s1_s: f64, last_s2_cumul_s: f64). Returns (Option<u32>, Option<u32>, Option<u32>). Guard: if any input <= 0.0, return (None, None, None). Formula: S1=s1*1000, S2=(s2_cumul-s1)*1000, S3=(lap-s2_cumul)*1000.

**session_info():** Return Ok(None) — same as iRacing (session info populated from elsewhere).

**disconnect():** Set connected=false, drop ShmHandles (set to None). Log disconnection.

**Non-Windows stubs:** All shared memory operations return Err or None on non-Windows via `#[cfg(not(windows))]` blocks.

**Unit tests (6 tests in #[cfg(test)] mod tests):**
Follow iRacing test patterns exactly. Tests that don't need SHM (sector math, session transition, first-packet safety, lap event) work cross-platform by directly manipulating adapter fields.

Do NOT add `pub mod lmu;` to mod.rs — that is Plan 02's responsibility.
