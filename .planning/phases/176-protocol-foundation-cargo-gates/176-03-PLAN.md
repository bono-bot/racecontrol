on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

env:
  CARGO_TERM_COLOR: always

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Rust stable
        uses: dtolnay/rust-toolchain@stable

      - name: Cache cargo
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target
          key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}

      - name: Build all (default features)
        run: cargo build --workspace

      - name: Test rc-common
        run: cargo test -p rc-common

      - name: Build rc-agent (minimal features)
        run: cargo build -p rc-agent-crate --no-default-features

      - name: Build rc-sentry (minimal features)
        run: cargo build -p rc-sentry --no-default-features

      - name: Test rc-agent
        run: cargo test -p rc-agent-crate

      - name: Build rc-agent (default features - explicit)
        run: cargo build -p rc-agent-crate

      - name: Build rc-sentry (default features - explicit)

Add a new section to CLAUDE.md under the "### Deploy" standing rules section (after the existing deploy rules, before "### Comms"). Add this standing rule:

```markdown
- **Single-binary-tier policy (v22.0):** All pods run the SAME binary compiled with default features (full build). Feature selection is done at RUNTIME via feature flags (FF-01+), NOT at compile time per pod. The `--no-default-features` build exists for CI verification and future testing scenarios only — it is NEVER deployed to production pods. Do not create per-pod Cargo feature profiles, per-pod binaries, or pod-specific compile-time feature sets.
  _Why: Per-pod compile-time variants create a combinatorial explosion of untested binaries. 8 pods x N feature combinations = build/test/deploy nightmare. Runtime feature flags (v22.0 Phase 177+) provide the same capability with one tested binary._
```

This documents the policy that ROADMAP success criteria #5 requires: "The single-binary-tier policy is documented in rc-common or CLAUDE.md."
