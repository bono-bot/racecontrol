2. **metrics.rs** — Add `pub async fn query_best_recovery_action(db: &SqlitePool, pod_id: &str, sim_type: &str, failure_mode: &str) -> (String, f64)` following the exact pattern from query_dynamic_timeout(). SQL query:
   ```sql
   SELECT recovery_action_tried,
          COUNT(*) as total,
          SUM(CASE WHEN recovery_outcome='"Success"' THEN 1 ELSE 0 END) as successes
   FROM recovery_events
   WHERE pod_id = ? AND sim_type = ? AND failure_mode = ?
     AND created_at > datetime('now', '-30 days')
   GROUP BY recovery_action_tried
   ORDER BY (successes * 1.0 / total) DESC LIMIT 1
   ```
   Minimum 3 samples required — below that, return default `("kill_clean_relaunch", 0.0)`. Use `.fetch_all(db).await.unwrap_or_default()` (no unwrap).
