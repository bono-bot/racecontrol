Create `crates/rc-sentry-ai/src/playback.rs` with three endpoints:

**State:**
```rust
pub struct PlaybackState {
    pub nvr_client: NvrClient,
    pub cameras: Vec<CameraConfig>,
    pub db_path: String,
}
```

**Endpoint 1: GET /api/v1/playback/search**
Query params: `camera` (camera name), `start` (ISO datetime), `end` (ISO datetime)
- Look up `nvr_channel` from `cameras` vec by name. Return 400 if camera not found or has no nvr_channel.
- Call `nvr_client.search_files(channel, start, end)`.
- Return JSON array of NvrFileInfo objects.
- CORS enabled (same pattern as mjpeg.rs).

**Endpoint 2: GET /api/v1/playback/stream**
Query params: `file_path` (the file_path from search results)
- Call `nvr_client.stream_file(file_path)`.
- Stream the response body through to the client with `Content-Type: video/mp4` (Dahua .dav files are essentially raw H.264/MP4).
- Set `Content-Disposition: inline` so browser plays it.
- Use `Body::from_stream` to proxy the bytes without buffering the entire file.
- Set appropriate CORS headers.

**Endpoint 3: GET /api/v1/playback/events**
Query params: `day` (YYYY-MM-DD format)
- Open SQLite connection to db_path using `spawn_blocking` (same pattern as attendance/routes.rs).
- Call `attendance::db::get_attendance_for_day(&conn, &day)`.
- Return JSON array of AttendanceEntry objects.
- This is separate from the existing attendance/history endpoint because it returns ALL events for the day (for timeline markers), not just filtered ones.

**Router:**
```rust
pub fn playback_router(state: Arc<PlaybackState>) -> axum::Router {
    let cors = CorsLayer::new()
        .allow_origin(Any)
        .allow_methods([axum::http::Method::GET])
        .allow_headers(Any);

    axum::Router::new()
        .route("/api/v1/playback/search", get(search_handler))
        .route("/api/v1/playback/stream", get(stream_handler))
        .route("/api/v1/playback/events", get(events_handler))
        .with_state(state)
        .layer(cors)
}
```

Use `axum::extract::Query` with serde Deserialize structs for query params. Follow the same error handling pattern as attendance/routes.rs (return StatusCode + Json error on failure).
