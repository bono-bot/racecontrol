<objective>
Refactor rc-sentry's crash handler from a flat Tier-1-then-restart sequence into a graduated 4-tier recovery flow with spawn verification and recovery event reporting. This is the core upgrade that makes rc-sentry intelligent instead of blind.

Purpose: Replace the current blind restart loop with a graduated response that verifies spawned processes actually started, checks pattern memory before restarting, tracks server reachability to prevent false MAINTENANCE_MODE lockouts, and reports every recovery attempt to the server for cross-machine visibility.

Output: Modified tier1_fixes.rs with graduated handle_crash(), spawn verification at 500ms/10s, server_reachable check, recovery event HTTP reporter. Modified main.rs crash handler thread to use the new flow.
</objective>

```rust
fn post_recovery_event(event: &rc_common::recovery::RecoveryEvent) {
    let body = match serde_json::to_string(event) {
        Ok(b) => b,
        Err(e) => {
            tracing::warn!(target: LOG_TARGET, "failed to serialize recovery event: {}", e);
            return;
        }
    };
    let request = format!(
        "POST /api/v1/recovery/events HTTP/1.0\r\n\
         Host: 192.168.31.23\r\n\
         Content-Type: application/json\r\n\
         Content-Length: {}\r\n\
         Connection: close\r\n\
         \r\n{}",
        body.len(), body
    );
    let addr: std::net::SocketAddr = match SERVER_ADDR.parse() {
        Ok(a) => a,
        Err(_) => return,
    };
    let stream = match std::net::TcpStream::connect_timeout(&addr, SERVER_CONNECT_TIMEOUT) {
        Ok(s) => s,
        Err(e) => {
            tracing::warn!(target: LOG_TARGET, "recovery event POST failed (connect): {}", e);
            return;
        }
    };
    let mut stream = stream;
    if let Err(e) = stream.write_all(request.as_bytes()) {
        tracing::warn!(target: LOG_TARGET, "recovery event POST failed (write): {}", e);
    }
    // Don't read response — fire-and-forget
}
```
