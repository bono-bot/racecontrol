    - [ ] Rollback plan prepared (know how to undo)

    ## racecontrol (Server .23, port 8080)
    ### Deploy
    ```bash
    bash deploy-staging/deploy.sh racecontrol
    ```
    This: builds release binary, SCPs to server, taskkill, schtasks restart, health check.

    ### Manual Steps (if deploy.sh unavailable)
    1. cargo build --release --bin racecontrol
    2. scp target/release/racecontrol.exe ADMIN@192.168.31.23:C:/RacingPoint/racecontrol-new.exe
    3. ssh ADMIN@192.168.31.23 "taskkill /F /IM racecontrol.exe & move /Y C:\RacingPoint\racecontrol-new.exe C:\RacingPoint\racecontrol.exe & schtasks /Run /TN StartRCTemp"
    4. sleep 5 && curl http://192.168.31.23:8080/api/v1/health

    ### Rollback
    ```bash
    ssh ADMIN@192.168.31.23 "copy C:\RacingPoint\racecontrol-prev.exe C:\RacingPoint\racecontrol.exe & schtasks /Run /TN StartRCTemp"
    ```

    ### Deploy (RCAGENT_SELF_RESTART sentinel — standing rule)
    1. cargo build --release --bin rc-agent
    2. Copy to deploy-staging/rc-agent.exe
    3. Start HTTP server: python -m http.server 9998 --directory deploy-staging
    4. For each pod, via relay exec:
       curl.exe -s -o C:\RacingPoint\rc-agent-new.exe http://192.168.31.27:9998/rc-agent.exe
    5. Send RCAGENT_SELF_RESTART sentinel — rc-agent calls relaunch_self() and swaps binary
    6. Verify: GET http://<pod-ip>:8090/health — check build_id

    Always canary Pod 8 first, then fleet.
