```json
[{
```

```json
{
  "grid_mode": "3x3",
  "camera_order": [1, 2, 3, ...],
  "zone_filter": null
}
```

GET /api/v1/cameras/nvr/:channel/snapshot returns JPEG image (cached, ~1ms response).

PUT /api/v1/cameras/layout accepts CameraLayout JSON, writes atomically to camera-layout.json.

**JavaScript (inside script tag):**

All DOM construction MUST use createElement -- never innerHTML (security hook requirement per CONTEXT.md).

1. **State variables:**
   ```
   var cameras = [];
   var tileElements = {};  // keyed by nvr_channel
   var imgElements = {};   // keyed by nvr_channel
   var dotElements = {};   // keyed by nvr_channel
   var currentMode = '3x3';
   var timer = null;
   ```

2. **fetchCameras()** -- `fetch('/api/v1/cameras')` then `.json()`. Sort result by `display_order`. Store in `cameras`. Then call `buildGrid()`. On error, set status to "API error".

3. **fetchLayout()** -- `fetch('/api/v1/cameras/layout')` then `.json()`. If `grid_mode` is returned, set `currentMode` to it and apply the CSS class. If `camera_order` has values, reorder `cameras` array to match (cameras not in the order list go at the end, maintaining their display_order). Called after fetchCameras completes.

4. **buildGrid()** -- Clear grid. For each camera in `cameras` array:
   - Create div.cam, set `data-channel` attribute to `nvr_channel`.
   - Create div.cam-label with two children: span for display_name text, span.dot for status.
   - Create img element with alt=display_name, set cursor:pointer.
   - If camera has no nvr_channel (null), skip snapshot loading for it.
   - Add click listener on img that calls `openFullscreen(camera)`.
   - Store references in tileElements, imgElements, dotElements by nvr_channel.
   - Append to grid.
   - If camera.status is "offline" or "disconnected": add "offline" class to .cam div, create and append an "OFFLINE" text span.

5. **setLayoutMode(mode)** -- Remove all grid-* classes from grid element. Add `grid-${mode}` class. Update `currentMode`. Update active state on layout buttons. This is CSS class swap only -- NO DOM rebuild, tiles stay in DOM. Save mode by calling `saveLayout()`.

6. **Layout button listeners:** `document.querySelectorAll('.layout-btn').forEach(btn => btn.addEventListener('click', () => setLayoutMode(btn.dataset.mode)))`.

7. **refresh()** -- For each camera with a valid nvr_channel:
   - Create `new Image()`.
   - On load: set `imgElements[ch].src = this.src`, update dot class based on camera status (use "dot dot-live").
   - On error: add stale class, set dot to "dot dot-offline".
   - Set src to `/api/v1/cameras/nvr/${ch}/snapshot?t=${Date.now()}`.
   Track ok/fail counts and update status span: `"${ok}/${cameras.length} online"`.

8. **startLoop()** -- `clearInterval(timer); refresh(); timer = setInterval(refresh, parseInt(rateEl.value))`. Rate select change listener calls `startLoop()`.

9. **saveLayout()** -- `fetch('/api/v1/cameras/layout', { method: 'PUT', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({ grid_mode: currentMode, camera_order: cameras.map(c => c.nvr_channel).filter(Boolean), zone_filter: null }) })`. Fire-and-forget (no await needed for UX).

10. **Status mapping logic:** Map API status to dot class:
    - "connected" -> "dot dot-live" (green)
    - "reconnecting" -> "dot dot-stale" (yellow)
    - "disconnected" or "offline" -> "dot dot-offline" (red)

11. **Initialization:** Call `fetchCameras().then(fetchLayout).then(startLoop)`. Chain ensures cameras are loaded and ordered before polling starts.

12. **Fullscreen placeholder:** `openFullscreen(camera)` and `closeFullscreen()` are stub functions for now -- plan 03 will implement WebRTC. For now, openFullscreen shows the fullscreen div with the snapshot image (same as current behavior). closeFullscreen hides it. Escape key and click close it.

**CRITICAL constraints:**
- Single HTML file, no external JS imports (except Montserrat font from Google Fonts)
- All DOM via createElement, never innerHTML
- Brand colors: #E10600 (red), #1A1A1A (black), #222222 (card), #333333 (border)
- Grid gap: 2px (per user decision)
- Transition on grid-template-columns for smooth layout switching (UIUX-04)
- No .unwrap() equivalent -- all fetch calls have .catch() error handling
