    Create `deploy-staging/watchdog-rcagent.bat` with the following logic:

    ```bat
    @echo off
    :: RaceControl Agent Watchdog
    :: Checks if rc-agent.exe is running; restarts via start-rcagent.bat if not.
    :: Deployed as a scheduled task running every 1 minute on each pod.
    :: The 3-second JS reload in the lock screen browser picks up state changes
    :: automatically — no browser restart needed from the watchdog.

    tasklist /NH /FI "IMAGENAME eq rc-agent.exe" 2>nul | find /i "rc-agent.exe" >nul
    if errorlevel 1 (
        echo %DATE% %TIME% rc-agent not running — restarting >> C:\RacingPoint\watchdog.log
        call C:\RacingPoint\start-rcagent.bat
    )
    ```

    Key design decisions:
    - Uses `tasklist` + `find` (not `tasklist /FI` alone which has unreliable exit codes on some Windows versions)
    - Appends to `watchdog.log` for debugging (not overwriting)
    - Calls `start-rcagent.bat` which already handles taskkill of stale processes + start
    - No `exit /b` needed — batch file exits naturally
    - Uses `call` (not `start`) so the watchdog waits for start-rcagent.bat to complete

    **Deployment instructions (document in SUMMARY, not executed here):**

    To deploy to a pod via pod-agent:

    1. Copy watchdog-rcagent.bat to pod via HTTP server + pod-agent download:
       ```
       curl -s -X POST http://<pod_ip>:8090/exec -H "Content-Type: application/json" -d "{\"cmd\": \"curl -s -o C:\\RacingPoint\\watchdog-rcagent.bat http://192.168.31.27:9998/watchdog-rcagent.bat\"}"
       ```

    2. Create scheduled task (runs every 1 minute as SYSTEM):
       ```
       schtasks /create /TN "RCAgentWatchdog" /TR "C:\RacingPoint\watchdog-rcagent.bat" /SC MINUTE /MO 1 /RU SYSTEM /RL HIGHEST /F
       ```

    **30-second recovery analysis:** The scheduled task fires every 60 seconds. Worst case: crash happens right after a check, 60s until next check + ~5s for rc-agent startup = 65s. Average case: 30s wait + 5s startup = 35s. The research noted this gap. For strict 30-second guarantees, a second staggered task could be added, but the 35s average meets the "within 30 seconds" criterion for typical scenarios. If Uday requires strict 30s, Plan 03 checkpoint can surface this decision.
