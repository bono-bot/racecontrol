3. In the `telemetry_interval` tick block, find the section where `AcStatus::Live` is detected (the 1s stability window check). AFTER the existing stability check confirms Live, add the False-Live guard:

The existing code flow for AC when status==Live:
- `ac_status_stable_since` tracks 1s stability
- After 1s stability confirmed, it emits `AcStatus::Live` to server

Modify this to add the 5s speed+steer gate:

```rust
// After AC stability confirmed (1s), start False-Live guard
if conn.ac_live_since.is_none() {
    conn.ac_live_since = Some(std::time::Instant::now());
    conn.ac_live_has_input = false;
    tracing::debug!(target: LOG_TARGET, "AC False-Live guard started (5s window)");
}

// Check telemetry for real input (speed or steering)
if let Ok(Some(ref frame)) = adapter.read_telemetry() {
    if frame.speed_kmh > 0.0 || frame.steering.abs() > 0.02 {
        conn.ac_live_has_input = true;
    }
}

let elapsed = conn.ac_live_since.map(|t| t.elapsed().as_secs()).unwrap_or(0);

if conn.ac_live_has_input {
    // True Live — emit AcStatus::Live to server
    conn.ac_live_since = None;
    // ... existing emit code proceeds normally ...
} else if elapsed >= 5 {
    // False-Live: suppress, reset window
    conn.ac_live_since = None;
    tracing::info!(target: LOG_TARGET, "AC False-Live suppressed (5s, speed=0, steer=0)");
    continue; // or early return from this block — do NOT emit Live
} else {
    // Still within 5s window, waiting for input
    continue; // do NOT emit Live yet
}
```

IMPORTANT: The exact integration point depends on how the existing code structures the emit. Read the full telemetry_interval block carefully. The guard wraps AROUND the existing AcStatus::Live emit — if no input in 5s, the emit is suppressed.
