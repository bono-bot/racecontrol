<task type="auto">
  <name>Task 2: Rust broadcast endpoint in cafe_marketing.rs</name>
  <files>crates/racecontrol/src/cafe_marketing.rs, crates/racecontrol/src/api/routes.rs</files>
  <action>
Create `crates/racecontrol/src/cafe_marketing.rs` with a single public handler:

```
POST /api/v1/cafe/marketing/broadcast
Auth: admin token required (same as other admin routes)
Body: { "message": string, "promo_name"?: string }
Response: { "attempted": usize, "sent": usize, "skipped_cooldown": usize, "skipped_no_phone": usize }
```

Implementation details:
1. Query all drivers: `SELECT id, name, phone FROM drivers WHERE phone IS NOT NULL AND phone != ''`
2. Rate limit: in-memory `LazyLock<Mutex<HashMap<String, Instant>>>` keyed by driver_id, 24h cooldown. Skip drivers whose last broadcast was within 24h. This is the same debounce pattern used in `send_security_alert` in whatsapp_alerter.rs.
3. For each non-cooldown driver, call Evolution API: `{evo_url}/message/sendText/{evo_instance}` with `{"number": phone, "text": message}` and `apikey` header — same pattern as `send_whatsapp` in whatsapp_alerter.rs. Use `reqwest::Client` with 5s timeout (no `.unwrap()` — use `?` with proper error mapping or warn+continue on per-driver failures).
4. Collect counts: attempted (total drivers with phones), sent (Evolution returned 2xx), skipped_cooldown (cooldown active), skipped_no_phone (no phone).
5. Return JSON summary with StatusCode::OK.
6. If Evolution API config is not present in config, return 503 with `{"error": "WhatsApp not configured"}`.

No `.unwrap()` anywhere. All individual driver send failures must warn+continue (never abort the whole broadcast).

Register the route in `crates/racecontrol/src/api/routes.rs`:
- Add `mod cafe_marketing;` (or `use crate::cafe_marketing;`) at the top where other cafe modules are declared
- Add route under the admin-protected router: `POST /api/v1/cafe/marketing/broadcast` → `cafe_marketing::broadcast_promo`
- This route needs admin auth (same middleware layer as `/cafe/promos`)
