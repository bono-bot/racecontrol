<objective>
Wire the rc-core API routes and WebSocket handlers so PWA requests for assist/FFB changes reach the agent and confirmations flow back. Critically, implement a per-pod assist state cache in AppState so that GET /assist-state returns actual values immediately (not just a 'queried' acknowledgment), satisfying the locked decision: "Drawer shows actual pod state when opened."

Purpose: Without this, the PWA has no way to communicate assist/FFB changes to the agent, and cannot show current pod state when the controls drawer opens. The core acts as the relay between PWA HTTP requests and agent WebSocket messages, with caching to bridge the async gap.
Output: New/updated API routes in rc-core, WebSocket handler arms for new AgentMessage variants, per-pod assist state cache.
</objective>

        State(state): State<AppState>,
        Path(pod_id): Path<String>,
        Json(body): Json<serde_json::Value>,
    ) -> Json<serde_json::Value> {
        let assist_type = body.get("assist_type")
            .and_then(|v| v.as_str())
            .unwrap_or("");
        let enabled = body.get("enabled")
            .and_then(|v| v.as_bool())
            .unwrap_or(false);

        // Validate assist_type is one of the supported types
        if !["abs", "tc", "transmission"].contains(&assist_type) {
            return Json(json!({ "error": "Invalid assist_type. Supported: abs, tc, transmission" }));
        }

        // Stability control intentionally excluded per user decision (no runtime mechanism in AC)

        if let Err(e) = state.send_to_pod(&pod_id, CoreToAgentMessage::SetAssist {
            assist_type: assist_type.to_string(),

        }).await {
            tracing::error!("Failed to send SetAssist to {}: {}", pod_id, e);
            return Json(json!({ "error": format!("Pod not connected: {}", e) }));
        }

        Json(json!({ "ok": true, "assist_type": assist_type, "enabled": enabled }))
    }

    Initialize it as empty in the AppState constructor.

    Then the handler reads from this cache:
    ```rust
    async fn get_pod_assist_state(
        State(state): State<AppState>,
        Path(pod_id): Path<String>,
    ) -> Json<serde_json::Value> {
        // Read cached state immediately
        let cached = {
            let cache = state.assist_cache.read().unwrap_or_else(|e| e.into_inner());
            cache.get(&pod_id).cloned()
        };

        // Also send QueryAssistState to agent for background refresh
        // (next time PWA opens the drawer, cache will be even fresher)
        if let Err(e) = state.send_to_pod(&pod_id, CoreToAgentMessage::QueryAssistState).await {
            tracing::warn!("Failed to send QueryAssistState to {}: {}", pod_id, e);
        }

        match cached {
            Some(s) => Json(json!({
                "ok": true,
                "abs": s.abs,
                "tc": s.tc,
                "auto_shifter": s.auto_shifter,
                "ffb_percent": s.ffb_percent,
            })),
            None => {
                // No cached state yet (pod never reported state).
                // Return defaults -- the background QueryAssistState will populate the cache.
                Json(json!({
                    "ok": true,
                    "abs": 0,
                    "tc": 0,
                    "auto_shifter": true,
                    "ffb_percent": 70,
                }))
            }
        }
    }
    ```

    Add match arms in the WebSocket message handler for the 3 new AgentMessage variants from Plan 01. Follow the existing pattern used by FfbZeroed and GameCrashed. CRITICAL: each handler must update the AppState assist cache so that GET /assist-state returns real values.

    The `state` variable (AppState) is accessible in the WS handler context. Use the `assist_cache` field added in Task 1.

    **AssistChanged handler:**
    ```rust
    AgentMessage::AssistChanged { pod_id, assist_type, enabled, confirmed } => {
        tracing::info!(
            "Pod {} assist changed: {} = {} (confirmed: {})",
            pod_id, assist_type, enabled, confirmed
        );
        // Update pod activity timestamp
        if let Some(pod) = state.pods.get(&pod_id) {
            pod.update_activity();
        }
        // Update assist cache with the changed value
        {
            let mut cache = state.assist_cache.write().unwrap_or_else(|e| e.into_inner());
            let entry = cache.entry(pod_id.clone()).or_default();
            match assist_type.as_str() {
                "abs" => entry.abs = if enabled { 1 } else { 0 },
                "tc" => entry.tc = if enabled { 1 } else { 0 },
                "transmission" => entry.auto_shifter = enabled,
                _ => {}
            }
        }
    }
    ```

    **FfbGainChanged handler:**
    ```rust
    AgentMessage::FfbGainChanged { pod_id, percent } => {
        tracing::info!("Pod {} FFB gain changed to {}%", pod_id, percent);
        if let Some(pod) = state.pods.get(&pod_id) {
            pod.update_activity();
        }
        // Update FFB percent in assist cache
        {
            let mut cache = state.assist_cache.write().unwrap_or_else(|e| e.into_inner());
            let entry = cache.entry(pod_id.clone()).or_default();
            entry.ffb_percent = percent;
        }
    }
    ```

    **AssistState handler (full state update from QueryAssistState response):**
    ```rust
    AgentMessage::AssistState { pod_id, abs, tc, auto_shifter, ffb_percent } => {
        tracing::info!(
            "Pod {} assist state: ABS={} TC={} auto_shifter={} FFB={}%",
            pod_id, abs, tc, auto_shifter, ffb_percent
        );
        if let Some(pod) = state.pods.get(&pod_id) {
            pod.update_activity();
        }
        // Replace entire cached state for this pod with fresh data from agent
        {
            let mut cache = state.assist_cache.write().unwrap_or_else(|e| e.into_inner());
            cache.insert(pod_id.clone(), CachedAssistState {
                abs,
                tc,
                auto_shifter,
                ffb_percent,
            });
        }
    }
    ```

    NOTE: Import `CachedAssistState` from wherever it was defined in Task 1 (likely `crate::api::routes::CachedAssistState` or `crate::state::CachedAssistState`).

    These match arms ensure the code compiles with the new enum variants AND actively maintain the assist cache. This is the key link that connects the agent's confirmed state to the PWA's GET /assist-state response -- without this caching, the PWA would get stale or empty values when opening the controls drawer.
