Wire the content manifest through the full system: core-side caching, catalog filtering, API integration, launch validation, and agent-side sending.

Purpose: This plan connects the scanner from Plan 01 to the rest of the system so that customers only see content installed on their pod, invalid session types are hidden, and launch-time validation prevents broken combos from reaching the agent.

Output: Per-pod manifest cache in AppState, filtered catalog API, launch validation gate at both launch_game() and customer_book_session(), and agent sending manifest after Register.
