    - Constructor accepts { nowFn } for DI (default Date.now)
    - recordRestart() increments restart count
    - recordDisconnect(ts) records disconnect timestamp
    - recordReconnect(ts) increments reconnection count and tracks longest disconnect
    - snapshot(nowTs) returns { uptimePercent, restartCount, reconnectionCount, longestDisconnectMs, periodStartTs }
    - uptimePercent = ((totalMs - totalDisconnectMs) / totalMs) * 100, rounded to 1 decimal
    - If no disconnects, uptimePercent is 100
    - Multiple disconnect/reconnect cycles: longestDisconnectMs is the max of all gaps
    - reset(nowTs) zeroes counters and sets new period start -- subsequent snapshot shows fresh data
    - snapshot() before reset() captures the period data (snapshot-then-reset pattern)

    DailySummaryScheduler (test/daily-summary.test.js):
    - Constructor accepts { nowFn, setTimeoutFn, sendWhatsAppFn, sendEmailFn, accumulator, udayNumber, sendEmailPath, execFileFn }
    - start() schedules the next window via setTimeout
    - Emits 'summary_due' event with { window: 'morning'|'evening' } when timer fires
    - msUntilNextWindow() computes ms from now to next 9:00 AM IST or 11:00 PM IST (whichever is sooner)
    - If current IST time is 08:00, next window is 9:00 AM IST (1 hour away)
    - If current IST time is 10:00, next window is 11:00 PM IST (13 hours away)
    - If current IST time is 23:30, next window is 9:00 AM IST (next day, 9.5 hours away)
    - Re-arms after each fire (chained setTimeout, not setInterval)
    - stop() clears the pending timer
    - formatWhatsApp(snapshot, podStatus) returns emoji-prefixed one-liner string
    - formatEmail(snapshot, podStatus, periodStart, periodEnd) returns multi-line detailed string with tables
    - sendSummary(window, podReport) calls accumulator.snapshot(), formats, sends via WhatsApp + email, then calls accumulator.reset()
