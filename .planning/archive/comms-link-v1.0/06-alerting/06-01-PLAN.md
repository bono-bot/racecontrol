    - canSend() returns true when no alerts have been sent yet
    - canSend() returns false within the suppression window after recordSent()
    - canSend() returns true after the window expires
    - reset() makes canSend() return true immediately (used on recovery)
    - Constructor accepts { windowMs, nowFn } for DI testability

    sendEvolutionText({ url, instance, apiKey, number, text }):
    - Returns Promise resolving to { ok, status, body }
    - Accepts injectable transportFn for testing (default uses node:http/https)
    - Constructs correct URL path: /message/sendText/{instance}
    - Sets Content-Type: application/json and apikey headers
    - Sends JSON body with { number, text }
    - Returns { ok: false, status: 0, body: errorMessage } on network error
    - 10s timeout on request

    AlertManager:
    - Constructor accepts { sendFn, udayNumber, cooldownMs, nowFn } (DI for sendFn)
    - Constructor validates udayNumber is present; logs warning and disables if missing
    - handleJamesDown({ timestamp, lastHeartbeat }) sends WhatsApp with format: emoji + "James DOWN" + time + crash info
    - handleJamesDown suppressed by AlertCooldown (canSend() false = no message sent)
    - handleRecovery({ crashCount, downtimeMs, restartCount }) sends WhatsApp with format: emoji + "James UP" + time + downtime + restarts
    - handleRecovery resets the AlertCooldown (next down-alert fires immediately)
    - handleJamesDown tracks downSince timestamp for downtime context
    - Emits 'alert_sent' event with { type, number, text } for observability
    - Emits 'alert_suppressed' event when cooldown blocks a send
    - WhatsApp message format matches CONTEXT.md locked decisions:
      - Down: "🚨 James DOWN HH:MM (restart attempt N, cooldown Xmin)"
      - Up: "✅ James UP HH:MM (down Xmin, N restarts)"
