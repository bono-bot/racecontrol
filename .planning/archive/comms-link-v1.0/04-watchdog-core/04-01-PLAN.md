function detectClaude() {
  return new Promise((resolve) => {
    execFile('tasklist', ['/NH', '/FI', 'IMAGENAME eq claude.exe'], { timeout: 5000 },
      (err, stdout) => {
        if (err) { resolve(null); return; }

        if (output.includes('claude.exe')) resolve(true);
        else if (output.includes('no tasks are running')) resolve(false);
        else resolve(null);
      });
  });
}

<feature>
  <name>ClaudeWatchdog: crash detection, zombie cleanup, auto-restart</name>
  <files>james/watchdog.js, test/watchdog.test.js</files>
  <behavior>
    WD-01 (detection):
    - Test: When detectFn returns false (Claude not running) and watchdog is not already restarting, emit 'crash_detected' event with timestamp
    - Test: When detectFn returns true (Claude running), do nothing (no events emitted)
    - Test: When detectFn returns null (error/timeout), treat as running (do not restart on detection failure)
    - Test: Polling interval fires every pollMs milliseconds (default 3000)
    - Test: First poll fires immediately on start() (not waiting for interval)

    WD-02 (zombie kill + restart):
    - Test: After crash detected, killFn is called before spawnFn (order matters)
    - Test: After killFn completes, a 2-second delay occurs before spawnFn (OS cleanup time)
    - Test: spawnFn is called with the path returned by findExeFn
    - Test: If findExeFn returns null (no exe found), emit 'restart_failed' with reason, do not call spawnFn
    - Test: After successful spawn, emit 'restart_success' with { pid, exePath }
    - Test: After successful spawn, wait 3 seconds then verify Claude is running via detectFn; if not running, emit 'restart_failed'

    Restart guard:
    - Test: While a restart is in progress (#restarting flag), poll cycles skip detection entirely (no double-restart)
    - Test: #restarting flag is cleared after restart sequence completes (success or failure)

    Version discovery (findClaudeExe):
    - Test: Reads CLAUDE_CODE_BASE directory, sorts version subdirectories by semver descending, returns path to claude.exe in the latest
    - Test: Returns null if no version directories exist
    - Test: Returns null if latest directory has no claude.exe

    Lifecycle:
    - Test: start() begins polling, stop() clears interval
    - Test: Calling start() twice clears previous interval (no double-polling)
    - Test: stop() is idempotent (calling stop() when already stopped does nothing)
  </behavior>
  <implementation>
    Create `james/watchdog.js` exporting `ClaudeWatchdog` class extending EventEmitter:

    Constructor accepts `{ pollMs, detectFn, killFn, spawnFn, findExeFn }` with sensible defaults:
    - `pollMs`: 3000 (3 seconds, detection within 3-6s satisfies WD-01's 5s target)
    - `detectFn`: Default uses execFile('tasklist', ['/NH', '/FI', 'IMAGENAME eq claude.exe']) -- adapt from system-metrics.js detectClaude()
    - `killFn`: Default uses execFile('taskkill', ['/F', '/T', '/IM', 'claude.exe']) -- errors are expected (no processes to kill) and silently resolved
    - `spawnFn`: Default uses spawn(exePath, [], { detached: true, stdio: 'ignore' }) + child.unref(), returns child.pid
    - `findExeFn`: Default reads `${process.env.LOCALAPPDATA}/Packages/Claude_pzs8sxrjxfjjc/LocalCache/Roaming/Claude/claude-code/` directory, sorts version dirs by semver descending, returns `join(dir, 'claude.exe')` if exists, null otherwise

    Private fields: `#interval`, `#pollMs`, `#restarting`, `#detectFn`, `#killFn`, `#spawnFn`, `#findExeFn`

    Methods:
    - `start()`: Clear existing interval, run #poll() immediately, set interval
    - `stop()`: Clear interval, set to null (idempotent)
    - `async #poll()`: If #restarting, return early. Call detectFn. If false, begin restart sequence. If true or null, do nothing.
    - `async #restart()`: Set #restarting=true. Emit 'crash_detected'. Call killFn. Wait 2s (setTimeout). Call findExeFn. If null, emit 'restart_failed' { reason: 'exe_not_found' }, clear #restarting, return. Call spawnFn(exePath). Emit 'restart_success' { pid, exePath }. Wait 3s. Call detectFn to verify. If not running, emit 'restart_failed' { reason: 'process_died_after_spawn' }. Clear #restarting.

    Also export `findClaudeExe` as a standalone named export for reuse by watchdog-runner.js.

    Events emitted: 'crash_detected' { timestamp }, 'zombies_killed' { timestamp }, 'restart_success' { pid, exePath, timestamp }, 'restart_failed' { reason, timestamp }
  </implementation>
</feature>
