<feature>
  <name>LogbookWatcher: File polling, hash comparison, echo suppression, atomic write</name>
  <files>james/logbook-watcher.js, test/logbook-watcher.test.js</files>
  <behavior>
    - Poll reads file content every pollMs (default 5000ms) using injected readFileFn
    - First poll sets initial hash, does NOT emit 'changed' event
    - Subsequent poll with different hash emits 'changed' with { hash, content, timestamp }
    - Subsequent poll with same hash does NOT emit 'changed'
    - When #writing flag is true, poll() skips (returns without reading) -- echo suppression
    - suppressNextCycle() sets #writing = true
    - resumeDetection(newHash) sets #lastHash = newHash and clears #writing = false
    - lastHash getter returns current hash (null before first poll)
    - start() calls poll immediately, then sets interval; stop() clears interval
    - start() after stop() works (re-entrant)
    - atomicWrite(targetPath, content) writes to targetPath + '.tmp' then renames -- uses injected writeFileFn and renameFn for testing
    - readFile error emits 'error' event and does not crash
  </behavior>
  <implementation>
    Create LogbookWatcher class extending EventEmitter with private fields (#filePath, #pollMs, #interval, #lastHash, #writing, #readFileFn, #writeFileFn, #renameFn).

    Constructor accepts { filePath, pollMs, readFileFn, writeFileFn, renameFn } with DI defaults to real fs/promises functions.

    Export a standalone atomicWrite function that takes (targetPath, content, writeFileFn, renameFn) for reuse in wiring code.

    Use createHash('sha256').update(content).digest('hex') for hashing.

    Follow project conventions: ESM, Object.freeze where applicable, private class fields, DI for all I/O.
  </implementation>
</feature>

<feature>
  <name>Logbook merge: append detection, auto-merge, conflict flagging</name>
  <files>shared/logbook-merge.js, test/logbook-merge.test.js</files>
  <behavior>
    - getAppendedLines(baseContent, newContent): returns appended string if newContent is baseContent + new lines, null otherwise
    - getAppendedLines with identical content returns null (no append)
    - getAppendedLines with newContent shorter than baseContent returns null
    - getAppendedLines with newContent that modifies existing lines returns null
    - getAppendedLines trims trailing whitespace before comparison
    - mergeAppends(baseContent, localAppend, remoteAppend): returns baseContent.trimEnd() + localAppend + remoteAppend
    - detectConflict(baseContent, localContent, remoteContent): returns { canMerge: true, merged } when both sides are pure appends, returns { canMerge: false } when either side has non-append changes
    - detectConflict with one side unchanged returns { canMerge: true, merged: changedSide }
  </behavior>
  <implementation>
    Create shared/logbook-merge.js with pure functions (no side effects, no I/O). All functions are exported.

    detectConflict orchestrates: calls getAppendedLines for both local and remote against base. If both are pure appends, calls mergeAppends. If either is not, returns canMerge: false.

    Edge cases: empty base content, empty appends, trailing newline differences.

    Follow project conventions: ESM exports, no classes needed (pure functions).
  </implementation>
</feature>
