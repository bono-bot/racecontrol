<feature>
  <name>EscalatingCooldown + ClaudeWatchdog Self-Test</name>
  <files>james/watchdog.js, test/watchdog.test.js</files>
  <behavior>
    EscalatingCooldown (standalone class, exported):
    - constructor accepts { steps?: number[], nowFn?: () => number } with defaults [5000, 15000, 30000, 60000, 300000] and Date.now
    - delay getter: returns 0 when attemptCount is 0, otherwise steps[attemptCount-1] clamped to last element
    - attemptCount getter: returns current attempt count
    - ready(): returns true when attemptCount is 0 OR (nowFn() - lastAttemptTime) >= delay
    - recordAttempt(): increments attemptCount, records lastAttemptTime = nowFn()
    - reset(): sets attemptCount to 0, lastAttemptTime to 0

    Test cases for EscalatingCooldown:
    - ready() returns true initially (no attempts yet)
    - After recordAttempt(), ready() returns false if delay not elapsed
    - After recordAttempt(), ready() returns true after delay elapsed
    - delay returns correct step for each attempt (5s, 15s, 30s, 60s, 5min)
    - delay clamps to last step (5min) for attempts beyond step count
    - reset() makes ready() return true again and delay return 0
    - attemptCount tracks correctly through record and reset

    ClaudeWatchdog cooldown integration:
    - Constructor accepts optional cooldown option (EscalatingCooldown instance for DI)
    - Creates default EscalatingCooldown if not provided
    - #poll() checks cooldown.ready() before restart -- returns early if not ready
    - #poll() calls cooldown.recordAttempt() before restart when crash detected
    - #restart() emits 'self_test_passed' { pid, exePath, timestamp } when 3s verify succeeds
    - #restart() emits 'self_test_failed' { reason: 'process_died_after_spawn', timestamp } when 3s verify fails
    - Cooldown is NOT reset inside ClaudeWatchdog -- reset is the responsibility of the event consumer (Plan 02 runner)

    Test cases for ClaudeWatchdog cooldown:
    - Poll skips restart when cooldown.ready() returns false
    - Poll calls cooldown.recordAttempt() before restart
    - self_test_passed emitted when post-spawn detect returns true
    - self_test_failed emitted when post-spawn detect returns non-true
    - restart_failed still emitted alongside self_test_failed (backward compat)
    - Existing 75 tests continue to pass (no regressions)
  </behavior>
  <implementation>
    1. Add EscalatingCooldown class at the top of watchdog.js (before ClaudeWatchdog):
       - Private fields: #steps, #attemptCount = 0, #lastAttemptTime = 0, #nowFn
       - Constructor: { steps = [5000, 15000, 30000, 60000, 300000], nowFn = Date.now }
       - get delay(): attemptCount === 0 ? 0 : steps[min(attemptCount-1, steps.length-1)]
       - get attemptCount(): return #attemptCount
       - ready(): attemptCount === 0 ? true : (nowFn() - lastAttemptTime) >= delay
       - recordAttempt(): #attemptCount++, #lastAttemptTime = nowFn()
       - reset(): #attemptCount = 0, #lastAttemptTime = 0
       - Export the class

    2. Modify ClaudeWatchdog constructor to accept optional `cooldown` option:
       - this.#cooldown = options.cooldown ?? new EscalatingCooldown()
       - Add #cooldown private field

    3. Modify #poll():
       - After #restarting guard, add: if (!this.#cooldown.ready()) return;
       - Before await this.#restart(), add: this.#cooldown.recordAttempt();

    4. Modify #restart() -- at the 3s post-spawn verify step:
       - If verified === true: emit 'self_test_passed' { pid, exePath, timestamp: Date.now() }
       - If verified !== true: emit 'self_test_failed' { reason: 'process_died_after_spawn', timestamp: Date.now() }
       - Keep existing 'restart_failed' emit for backward compatibility when verified !== true

    5. IMPORTANT: Do NOT call cooldown.reset() inside ClaudeWatchdog. The consumer (watchdog-runner.js in Plan 02) listens for self_test_passed and calls cooldown.reset() externally. This keeps ClaudeWatchdog unaware of the runner's reset policy. However, since the cooldown is injected via constructor, and the runner will need access to it, expose a getter: get cooldown() { return this.#cooldown; }
  </implementation>
