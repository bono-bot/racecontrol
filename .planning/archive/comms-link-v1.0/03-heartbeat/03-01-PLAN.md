<objective>
Add application-level heartbeat with system metrics from James to Bono, and server-side DOWN detection.

Purpose: Bono must be able to detect within 45 seconds when James is down. Each heartbeat carries Claude Code process status and system health (CPU, memory, uptime) so Bono knows not just whether James is alive, but what state the machine is in. This is the foundation for alerting (Phase 6) -- without heartbeat, Bono cannot trigger WhatsApp alerts to Uday.

Output: Three new modules (system-metrics.js, heartbeat-sender.js, heartbeat-monitor.js), two test files, and wiring in both entry points. TDD approach -- tests first, then implementation.
</objective>
