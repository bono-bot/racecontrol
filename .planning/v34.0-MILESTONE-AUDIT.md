## Cross-Phase Integration Gaps

### Gap 1 (P1): No Metric Producers
`MetricsSender` channel created in main.rs but never used. `_metrics_tx` has underscore prefix. No code calls `record_sample()`. TSDB structurally complete but empty at runtime. Cascades to all downstream: query API returns empty, dashboard shows no data, alerts never fire, Prometheus returns empty body.

**Fix:** Wire producers in main.rs that periodically send MetricSample values (pod health scores, WS connections, billing revenue) via the channel.

### Gap 2 (P1): Dashboard Stub Not Replaced
Phase 287 dashboard calls stub functions returning fake sine-wave data. Three `// TODO: Replace with real API call when Phase 286 ships` markers in `racingpoint-admin/src/lib/api/metrics.ts`. Also has API contract mismatches between TypeScript interfaces and Rust response structs.

**Fix:** Replace stubs with rcFetch calls + reconcile field name/type mismatches.

### Gap 3 (P2): Integration Tests Broken
Pre-existing `BillingTimer { nonce }` compile error in `tests/integration.rs`. 740 lib tests pass, integration suite doesn't compile. Pre-dates v34.0.
