## Overview

Make autonomous action loops observable and queryable with time-series depth -- answer "what happened last Tuesday at 8pm" without grepping JSONL logs. SQLite TSDB with rollups, query API, Next.js dashboard, Prometheus export, WhatsApp threshold alerts, metric producers, and dashboard API wiring.

## Milestone Summary

**Key Decisions:**
- SQLite TSDB over Prometheus/InfluxDB (venue-scale, 8 pods, extend-don't-replace)
- Custom Next.js dashboard over Grafana (maintainable, branded)
- WhatsApp threshold alerts over Slack/email (Uday uses WhatsApp)
- Single metrics_rollups table with resolution discriminator (not separate hourly/daily tables)
- Any-pod threshold: alert fires if ANY pod exceeds threshold
- Prometheus endpoint public (no auth) for zero-friction scraping

**Issues Resolved:**
- Integration audit found MetricsSender channel had no producers (Phase 290 gap closure)
- Dashboard stub functions never replaced with real API calls (Phase 291 gap closure)
- API contract mismatches between TypeScript and Rust response structs (fixed in Phase 291)

**Issues Deferred:**
- integration.rs compile error (BillingTimer missing nonce field) — pre-existing, not v34.0
- Live round-trip verification (requires running server with real data)

**Technical Debt:**
